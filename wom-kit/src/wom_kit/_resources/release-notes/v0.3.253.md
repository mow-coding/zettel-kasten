# v0.3.253 - Optional MOW Harness Compatibility Boundary

v0.3.253 makes WOM's support boundary for the optional MOW Harness explicit
without making either system depend on the other.

## What Changed

- A new compatibility guide separates WOM archive authority from MOW Harness
  coordination and installer authority.
- The guide connects the existing `/collab/` and `/.mow-harness/` local-only
  protections to onboarding, project intake, public navigation, and the
  capability matrix.
- Focused regression tests protect namespace isolation, the external source
  alpha status, preview-before-write guidance, and separate activation.
- The public status distinguishes stable file-based Harness `v0.1.23` from an
  external CLI that remains a source alpha and is not published to npm.

## Why It Matters

WOM should support a structured multi-agent operating room without treating
its prompts, mailboxes, or installer metadata as human archive knowledge. The
same boundary prevents the opposite mistake: WOM must not invent a second MOW
installer, silently approve an external write, or imply that updated files
activate a Harness session.

Durable insight produced during coordinated work still enters WOM only through
the ordinary reviewed artifact and receipt paths.

## Compatibility And Safety Boundary

- No archive migration is required.
- No WOM command, schema, receipt, archive write, model call, provider call,
  network call, database call, or UI is added.
- WOM does not install, update, activate, remove, or invoke MOW Harness.
- Existing archives already ignore `/collab/` and `/.mow-harness/`; this release
  documents and regression-protects that behavior.
- A real MOW legacy-adoption apply and later Harness activation remain separate
  human-approved external operations.

## Verification

Focused tests verify the public guide, navigation links, capability boundary,
and existing generated-archive ignore patterns. The full WOM-kit test suite,
package-resource and clean-wheel checks, public privacy scan, and release
readiness gate remain mandatory.
