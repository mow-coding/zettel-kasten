# v0.3.243 - Progressive AI Runtime Skill

v0.3.243 turns WOM-kit's monolithic archive runtime manual into a
standards-compatible, progressively disclosed Agent Skill package.

## What Changed

- Added YAML `name` and `description` metadata to the 97-line, 555-word
  `wom-archive/SKILL.md` entry document.
- Added five focused references for startup/update, memory/revision,
  capture/publication, foreign sharing/trust, and result/human-language work.
- Preserved the complete v0.3.242 operator contract as a searchable advanced
  reference instead of deleting accumulated safety guidance.
- Added `check_runtime_skill.py` and made it part of release readiness.
- Updated documentation tests to validate the complete skill package.
- Included every skill reference in the deterministic packaged-resource
  mirror and clean-wheel verification.

## Safety Boundary

- No archive migration or archive write.
- No user-data discovery or read during installation.
- No AI-host configuration or skill-directory write.
- No provider, object-store, database, model, or credential call.
- No generated graph or index becomes canonical.
- Reviewed local zet, objet, relation, and receipt records remain authoritative.

## Verification

Release verification covers the runtime-skill checker, focused tests, the full
WOM-kit suite, package-resource synchronization, clean-wheel build/install,
all four command entrypoints, disposable archive onboarding, strict Doctor,
public privacy hygiene, and release readiness.

## Upgrade

No archive migration is required. Update WOM-kit and restart the AI operator
process. Automatic installation into a host such as Codex or Claude remains a
later, separately approval-gated boundary.
