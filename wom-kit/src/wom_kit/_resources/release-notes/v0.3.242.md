# v0.3.242 - Self-Contained Python Tool Wheel

v0.3.242 turns WOM-kit's existing Python package metadata into a usable
installation artifact. A wheel no longer depends on a neighboring repository
checkout for schemas, templates, base rules, or release identity.

## What Changed

- The wheel now includes the runtime schema, template, zettel-kasten, and
  current-release resource set.
- A deterministic resource manifest records each packaged file's source path,
  package path, byte length, and SHA-256.
- Runtime path resolution prefers the editable repository source during
  development and falls back to the packaged mirror after installation.
- `sync_package_resources.py --check` blocks source/mirror drift.
- `check_wheel_install.py` builds from a clean copy, inspects the wheel,
  installs it in a fresh virtual environment, verifies all four entrypoints,
  creates a disposable archive through preview and approval, and runs strict
  Doctor.
- The verified wheel is attached to this GitHub release. Isolated `uv tool`
  installation is recommended; a dedicated `pip` virtual environment remains
  available as a compatibility path.

## Upgrade And Compatibility

No archive migration is required. Existing source-checkout and project-mirror
workflows remain compatible. Restart the AI operator process after changing the
installed tool.

This release does not publish WOM-kit to PyPI. `pip install wom-kit` is not yet
an official distribution command. Use the exact wheel URL documented in
`wom-kit/docs/python-tool-install.md`.

## Safety And Claim Boundary

- Installing the package does not create, discover, read, or modify an archive.
- No provider, object store, external database, model, credential store, or
  network call is added to WOM runtime behavior.
- Packaged resources are mirrors, not a second editable source of truth.
- The resource manifest proves package/source byte identity at build time; it
  does not prove user archive health or factual truth.
- No generated graph or index becomes canonical.
- PyPI name ownership and trusted publishing remain unperformed.

## Verification

- Package resource tests check 84 source-bound files and exact hashes.
- A clean wheel contained 98 files and all 84 manifested runtime resources.
- Fresh-environment `archive version`, onboarding preview, approved disposable
  onboarding, required-file checks, and strict Doctor passed.
- The repository test suite plus public privacy and release-readiness gates must
  pass before publication.
