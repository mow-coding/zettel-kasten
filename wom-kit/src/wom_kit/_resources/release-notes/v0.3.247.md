# v0.3.247 - Runtime Artifact Primacy

v0.3.247 closes the gap between WOM's public artifact-first philosophy and the
instructions actually read by an AI that operates a local archive.

## What Changed

- The installed `wom-archive` Agent Skill now treats time-situated artifacts
  and chronology as primary evidence.
- Its universal safety contract says `canonical` means the current
  human-reviewed archive state, not objective or timeless truth.
- Matching names or labels never authorize a silent identity merge.
- Nodes, ties, edges, indexes, embeddings, and graph projections remain
  reviewable claims, reading routes, or regenerable aids beneath artifacts.
- The focused reading reference tells the AI to compare artifact, time, scope,
  and provenance before describing identity or human change.
- The focused capture reference blocks label-based identity reuse or merge as
  an unstated drafting shortcut.
- The complete operator contract and personal, company, and family archive
  `AGENTS.md` templates carry the same rule.
- Runtime-skill validation and focused regression tests protect the contract.

## Why It Matters

A public philosophy document cannot prevent a runtime AI from cleaning human
memory into a false graph if that AI never reads the document. The rule must be
present at the decision point: while reading, drafting, linking, revising, and
explaining the archive.

WOM still permits inference. It requires the inference to remain visibly
bounded by the artifacts and their chronology rather than replacing them.

## Safety Boundary

- No archive format, schema, or stored zet changes.
- No entity resolver or automatic identity merge.
- No model, embedding, graph, provider, database, or object-store call.
- No new CLI, MCP, write authority, migration, or UI.
- Existing archives receive the updated rules only when their managed Agent
  Skill or generated archive instructions are explicitly updated.

## Verification

Release verification covers the runtime doctrine regression tests, runtime
skill package validation, the full WOM-kit suite, deterministic package
resources, clean-wheel installation, public link hygiene, Korean product
language hygiene, public privacy hygiene, and release readiness.
