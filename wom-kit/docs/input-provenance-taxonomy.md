# Input Provenance Taxonomy

Status: v0.3.152 input provenance taxonomy checkpoint

WOM now has a read-only input provenance taxonomy for AI operators.

This helps an AI avoid claiming that a tool discovered or verified an input
when the input was actually supplied by a caller, generated by an AI, inferred
from the environment, or only present in a fixture.

## Command

```powershell
archive input-provenance-taxonomy <archive-root> `
  --dry-run `
  --format json
```

Aliases:

```powershell
archive provenance-taxonomy <archive-root> --dry-run
archive caller-input-taxonomy <archive-root> --dry-run
```

## Provenance Classes

- `tool_discovered`: the command found the input from archive metadata, an index, or a local manifest.
- `receipt_verified`: the input came from a parsed receipt that matched the current archive.
- `human_selected`: a human selected the input, but the tool may not have independently verified the target.
- `caller_supplied`: the invoking AI, script, or shell supplied the input argument.
- `ai_generated`: an AI generated the label, path, identifier, or mapping.
- `fixture_supplied`: the input came from a local fixture or sanitized sample.
- `environment_inferred`: the input was inferred from cwd, environment variables, path conventions, or local defaults.
- `unknown`: the command cannot tell where the input came from.

## AI Operator Rule

`caller_supplied`, `ai_generated`, `fixture_supplied`, `environment_inferred`,
and `unknown` are not verified source truth.

If an input is caller-supplied, the AI should say it was supplied to the command,
not discovered by the command. If an input is AI-generated, the AI should require
validation before minting, linking, fetching, or publishing.

## Safety Boundary

The command:

- reads no archive body text,
- echoes no input values,
- echoes no source values,
- calls no providers,
- checks no network,
- echoes no local absolute paths, tokens, or secret values,
- writes nothing.
