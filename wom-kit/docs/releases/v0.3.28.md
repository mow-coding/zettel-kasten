# WOM-kit v0.3.28

Date: 2026-06-15

v0.3.28 adds a read-only credential vault onboarding planner.

## Added

- CLI `archive credential-vault-onboarding-plan --dry-run`.
- CLI aliases `archive credential-vault-onboarding --dry-run` and
  `archive secret-vault-onboarding-plan --dry-run`.
- MCP `credential_vault_onboarding_plan`.
- `wom-kit/docs/credential-vault-onboarding-plan.md`.

## Behavior

The planner helps a human choose how an external vault/keyring/password manager
should fit WOM without making WOM the vault.

It can describe:

- scenario-based store choice,
- selected store id,
- store class,
- store kind,
- future adapter kind,
- WOM ref prefix to record,
- human setup steps,
- WOM compatibility steps,
- broker-plan summary when a safe credential id is supplied,
- adapter-readiness summary when a safe credential id is supplied.

## Boundaries

This release does not open KeePassXC, Bitwarden, 1Password, Chrome or Edge
password stores, Windows Credential Manager, macOS Keychain, Linux keyrings,
environment variables, plaintext secret files, provider APIs, OAuth flows,
secret importers, adapter runners, approval writers, manifest writers, or audit
receipt writers.

It writes no files and includes no secret values, exact credential refs,
usernames, email addresses, tokens, local paths, provider URLs, passwords, or
API keys.

## Relationship To Previous Layers

The intended safe sequence is:

```text
credential-store-recommendation
-> credential-vault-onboarding-plan
-> credential-ref-plan
-> credential-ref-inventory
-> credential-access-broker-plan
-> credential-access-approval-plan
-> credential-adapter-readiness-plan
-> credential-adapter-manifest-plan
-> credential-adapter-audit-plan
```

v0.3.28 adds the human-facing onboarding step in that sequence. Live secret
storage, retrieval, migration, rotation, and provider use remain future work.
