# v0.3.151 - Operation Status Taxonomy

v0.3.151 adds a read-only taxonomy for AI-visible operation status.

## What Changed

- Added `archive operation-status-taxonomy`.
- Added aliases `archive status-taxonomy` and `archive partial-success-taxonomy`.
- Added a machine-readable taxonomy for:
  - `succeeded`,
  - `preview`,
  - `written`,
  - `no_change`,
  - `partial`,
  - `truncated`,
  - `blocked`,
  - `failed`.
- Marked `partial`, `truncated`, `blocked`, and `failed` as not-success classes.
- Added AI operator checks for outputs with truncation, omitted counts,
  incomplete counts, dry-run mode, and privacy guards.

## Safety Boundary

The command:

- reads no archive body text,
- echoes no zettel body text,
- echoes no source values,
- calls no providers,
- checks no network,
- echoes no local absolute paths, tokens, or secret values,
- writes nothing.

## Still Future
