# v0.2.47 Release Notes

Date: 2026-05-26

## Summary

v0.2.47 adds the ZET closed sharing model baseline.

This is a documentation and sanitized-example release. It clarifies that the base zettel-kasten system uses GitHub, object storage, and DB relationships, while the future ZET layer is a closed sharing/SNS/SaaS layer above that base.

## Added

- `wom-kit/docs/zet-closed-sharing-model-baseline.md`
- `wom-kit/examples/zet-closed-sharing/README.md`
- `wom-kit/examples/zet-closed-sharing/closed-sharing-update.example.json`

## Clarified

- GitHub can be part of the base system or an implementation substrate, but ZET is not defined as a GitHub-based SNS.
- WordPress is one possible user-selected projection surface, not the product core.
- `zet` is the organized text unit.
- `objet` is WOM product language for source/original material stored outside Git.
- Attestation in the closed SNS flow is receiver-side verification/review before any neighbor feed update, mirroring, or re-projection.

## Compatibility

No private archive migration is required.

No CLI or MCP behavior changed in this release. Existing v0.2.46 projection-plan previews remain unchanged.

## Safety

v0.2.47 does not:

- call providers,
- publish to WordPress,
- write projection records or projection receipts,
- implement real ZET transport,
- automatically update neighbor feeds,
- mint, trust, import, accept, attest, sign, anchor, apply, or transport anything,
- introduce Redis, queues, background workers,
- implement payments, staking, consensus, blockchain, WOM coin, or token mechanics,
- train models, run backpropagation, or enable full-auto execution.
