# WOM-kit v0.3.19

Date: 2026-06-14

v0.3.19 adds the first safe planning layer for treating email mailboxes as WOM
source worlds.

## Added

- Source type `imap_mailbox`.
- CLI `archive imap-mailbox-plan --dry-run`.
- MCP `imap_mailbox_plan`.
- `wom-kit/docs/imap-mailbox-source.md`.
- Tests for CLI output, MCP structured output, source registration, fail-closed
  scanning, dry-run-only behavior, and private-value redaction.

## Behavior

The plan supports provider presets for:

- Gmail,
- Naver,
- generic IMAP.

It accepts safe refs for the account, username, credential, and mailbox. It does
not accept raw emails, usernames, app passwords, OAuth tokens, provider URLs, or
local paths.

After review, an `imap_mailbox` source can be registered with `add-source`.
`scan-source` still fails closed for live IMAP in this release.

## Boundaries

This release does not connect to IMAP, login, select mailboxes, read message
headers, read message bodies, read attachments, send mail, delete mail, change
message flags, start OAuth, store credentials, capture `.eml` files, extract
derived text, draft zets, mint zets, or write files from the planning command.

## Small UX Fix

Zettel path commands now give one clearer message when a user passes an absolute
path or a path outside `inbox/` or `zettels/`: use an archive-relative path or
`--zettel-id`.
