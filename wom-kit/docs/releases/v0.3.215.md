# v0.3.215 - project version update

v0.3.215 replaces manual source-mirror checkout and pin editing with one
reviewed transaction. It is additive and requires no archive migration.

## What Changed

- Added CLI `project-version-update`, with `version-update` and
  `update-wom-kit` aliases.
- Added local/no-write dry-run and reviewer-attributed approval.
- Added atomic configured-origin main plus exact-tag fetch without forced tag
  overwrite.
- Added annotated-tag, main-ancestry, and package/pyproject/root-shim version
  verification before checkout.
- Added clean-worktree, symlink, pin, downgrade, lock, and ignored-file collision
  guards.
- Added detached exact-tag checkout, recognized pin alignment, schema-backed
  receipt, and rollback of checkout/pins/partial receipt state.
- Added content-free progress and restart-required follow-up guidance.

## Honest Boundary

The command verifies the configured origin and its fetched main history. It does
not verify a cryptographic tag signature. Git transport may use the configured
credential helper, but WOM-kit reads and echoes no credential value, remote URL,
raw Git stderr, absolute path, or unknown dirty filename.

The updating Python process remains loaded at its starting version. A new
process must run `archive version` before the target runtime can be claimed
active. Because the command first exists in v0.3.215, older clients need one
final existing/manual verified bootstrap update to this release.

## Compatibility And Safety

Archive zets, objets, manifests, source materials, providers, maps, and external
databases are untouched. A per-project lock prevents concurrent updates. Any
post-checkout failure attempts full local rollback; fetched refs may remain as
non-canonical discovery state.
