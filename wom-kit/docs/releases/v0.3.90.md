# v0.3.90 - Saved View Health

Date: 2026-06-17

## Added

- CLI `archive view-health <archive-root> --dry-run`.
- MCP `view_health`.
- Public docs for saved view health diagnostics.

## Why

Real-use feedback showed that saved AI navigation views can go stale when
their facet filters no longer match the actual indexed facet distribution.

This release adds a read-only way to see which saved views are active, empty,
or blocked before editing `views/*.yml`.

## Behavior

- Reads saved view definitions under `views/*.yml`.
- Reads the generated local SQLite index.
- Reports saved view hit counts.
- Shows per-filter matching counts and observed facet value samples for the
  keys used by saved views.
- Blocks clearly when the archive index is missing.

## Safety

The new path writes nothing, runs no index rebuild, rewrites no view files or
zettel facets, calls no providers, reads no object bytes, and echoes no zettel
bodies, zettel titles, absolute local paths, provider URLs, account ids,
emails, tokens, or secret values.

Facet values are metadata and may be shown as distribution samples, with unsafe
values redacted.
