# WOM-kit v0.3.31

Date: 2026-06-15

v0.3.31 adds the first local credential approval receipt writer.

## Added

- CLI `archive credential-access-approval --approve --reviewed-by <actor>`.
- CLI preview `archive credential-access-approval-plan --dry-run` remains
  available.
- Policy-check receipt verification through `archive credential-policy-check
  --approval-receipt <path> --dry-run`.

## Behavior

The approval writer records one non-secret JSON receipt under:

```text
receipts/credentials/access-approvals/
```

The receipt records safe metadata such as:

- credential id,
- credential kind,
- provider,
- ref store and ref prefix,
- action kind,
- store kind,
- consumer,
- reviewer,
- decision,
- receipt path.

The policy check can verify that receipt before a future live adapter runs.

## Boundaries

This release does not read, print, store, or infer the real secret value. It
does not echo exact credential refs, usernames, email addresses, local paths,
provider URLs, passwords, app passwords, OAuth tokens, API keys, or vault
contents.

It does not open KeePassXC, Bitwarden, 1Password, browser password stores,
Windows Credential Manager, macOS Keychain, Linux keyrings, environment
variables, plaintext secret files, provider APIs, OAuth flows, or live adapter
runners.

Even after a receipt verifies, `credential-policy-check` keeps
`live_execution_allowed_now` false. Live secret execution remains future work.

## Relationship To Previous Layers

The intended safe sequence is now:

```text
credential-store-recommendation
-> credential-vault-onboarding-plan
-> credential-plaintext-migration-plan
-> credential-ref-plan
-> credential-ref-inventory
-> credential-access-broker-plan
-> credential-access-approval-plan / credential-access-approval --approve
-> credential-policy-check --approval-receipt <path>
-> future adapter execution
-> credential-adapter-audit-plan
```

v0.3.31 creates the durable, non-secret approval receipt that a future minimal
KeePassXC adapter can require before doing any local secret-store work.
