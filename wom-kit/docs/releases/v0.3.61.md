# v0.3.61 - IMAP Adapter Execution Contract

Released: 2026-06-16

This release adds the read-only execution contract for a future live IMAP
adapter.

## Added

- `archive imap-mailbox-adapter-execution-contract <archive-root> --dry-run
  --format json`.
- Aliases `archive imap-mailbox-adapter-execution-plan` and
  `archive mailbox-adapter-execution-contract`.
- A contract wrapper around IMAP adapter preflight that becomes ready only when
  the manifest, approval receipt, selection plan, and audit preview are ready.
- Explicit future output rules for opaque candidate refs, counts-only receipts,
  and no AI-visible mail content.

## Safety Boundary

The command opens no IMAP connection, attempts no login, selects no mailbox,
lists no messages, reads no headers, bodies, or attachments, retrieves no
credential values, opens no vaults/keyrings/browser stores, calls no providers,
writes no receipts, and writes no files.

## Still Future Work

This release does not implement live IMAP login, mailbox selection, message
capture, attachment capture, derived-text capture, execution receipt writing,
credential secret retrieval, or OAuth.
