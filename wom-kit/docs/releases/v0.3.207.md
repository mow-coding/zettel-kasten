# v0.3.207 - compact node reading and contiguous coverage

v0.3.207 reduces archive-wide first-pass overhead and distinguishes page-end
from proven contiguous completion. It is additive and needs no archive
migration.

## What Changed

- Added CLI/MCP `projection=reading`, preserving node identity, title/kind,
  updated time, abstract state, facets, tie summary, and every safe edge while
  omitting full-only item fields.
- Added optional `coverage_mode=strict`, starting at cursor zero and requiring
  the prior page's checksum continuation token at every later cursor.
- Added contiguous-prefix count and chain hash, stateless continuation token,
  and `archive_wide_coverage_claim_ready`.
- Strict mode blocks cursor skips, projection/status mismatch, malformed or
  tampered tokens, expected-snapshot conflicts, and changed snapshots.
- Updated runtime context, AI start-here, archive templates, and runtime skill
  to use compact strict reading and require claim readiness.

## Scale Evidence

A temporary synthetic 10,000-zet run with 120-character abstracts, one simple
edge per zet, and an 8,000-token page budget completed 179 strict reading pages
in 52.4 seconds on the development Windows host. It returned 10,000 unique ids,
zero over-budget pages (largest estimate 7,925), and archive-wide claim
readiness. Compact reading items estimated 1,414,699 tokens versus 2,064,699
for full items on the same fixture.

These values are local observations and a transparent character heuristic, not
an SLA or provider-reported token usage.

## Compatibility And Safety

Defaults remain `projection=full` and `coverage_mode=page`. Existing callers
and archives require no migration. Page-mode `complete` retains its old
page-end meaning but is not contiguous proof.

The continuation token contains only safe snapshot/cursor/projection/order and
chain metadata, is never persisted, and is not a signature, attestation,
credential, or receipt. The catalog still reads no zet bodies, objet bytes,
providers, or secrets and requires no generated index.
