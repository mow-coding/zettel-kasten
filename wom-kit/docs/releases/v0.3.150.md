# v0.3.150 - Approval Handoff Lifecycle

v0.3.150 gives AI operators a first-class human approval handoff surface.

## What Changed

- Added read-only `archive approval-handoff-plan`.
- Added aliases `archive handoff-plan` and `archive human-approval-handoff-plan`.
- Added approval-gated `archive approval-handoff-record`.
- Added aliases `archive handoff-record` and `archive human-approval-handoff-record`.
- Approved records write:
  - `ops/approval-handoffs/<handoff-id>.yml`,
  - a receipt under `receipts/approval-handoffs/`.
- Added explicit lifecycle statuses:
  - `needs_review`,
  - `approved_once`,
  - `denied`,
  - `superseded`,
  - `resolved`.
- Added operation kinds for sensitive handoff categories, including private
  material reads, archive writes, external provider actions, releases,
  credential access, and derived artifact updates.

## Safety Boundary

The commands:

- execute no underlying operation,
- read no private material,
- call no providers,
- check no network,
- read no local secrets,
- echo no target ref values,
- echo no requested action values,
- echo no local absolute paths, tokens, or secret values.

## Still Future

- Command-specific adapters that require a matching approval handoff receipt.
- A shared operation status taxonomy across every JSON command.
- Automatic stale-approval detection when target metadata changes.
- A compact AI answer template for showing handoff state to a beginner user.
