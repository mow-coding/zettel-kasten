# v0.2.34 Foreign Block Quarantine Decision Preview

Release date: 2026-05-25

v0.2.34 adds a read-only decision preview for one existing foreign block quarantine case.

This is still not a decision write, not import, not trust, not attestation, not acceptance, and not ZET transport. It is a decision aid only.

## Added

- `archive quarantine-decision <archive-root> --case-id <safe-id> --dry-run --format json`
- optional `--decision-intent auto|keep_quarantined|reject_and_keep_record|eligible_for_attestation_review|needs_more_review`
- optional preview context `--reviewer` and `--review-note`
- read-only MCP `foreign_block_quarantine_decision_check`
- decision previews for:
  - `keep_quarantined`,
  - `reject_and_keep_record`,
  - `eligible_for_attestation_review`,
  - `needs_more_review`.

## Safety Boundary

The decision preview:

- reads one existing quarantine case JSON and matching quarantine receipt,
- keeps `trust_state: untrusted_foreign`,
- emits archive-relative paths only,
- blocks unsafe case ids, malformed case JSON, trust-state contradictions, receipt contradictions, unsafe reviewer values, and unsafe review notes,
- summarizes review-note presence without echoing note content,
- always returns `would_change: []`.

The decision preview does not:

- write files,
- write a decision,
- record approval,
- read the original foreign artifact,
- import the foreign block,
- mark the foreign block trusted,
- create an attestation,
- mint, anchor, delegate, sign, or execute anything,
- call provider APIs.

## MCP

MCP exposes only `foreign_block_quarantine_decision_check` for this workflow. It is read-only and requires `dry_run: true`.

MCP does not expose quarantine decision apply, write, accept, import, trust, attestation, receipt write, mint, anchor, delegate, signing, provider sync, ZET transport, or full-auto tools.

## Compatibility

No private archive migration is required.

Existing v0.2.32 quarantine cases and v0.2.33 review indexes remain compatible. The new preview depends on existing quarantine case and receipt files but mutates neither.
