# v0.3.39 - Object Storage Recommendation Matching

Date: 2026-06-15

v0.3.39 adds a read-only object storage recommendation matcher.

## Added

- CLI `archive object-storage-recommendation <archive-root> --dry-run`.
- Aliases:
  - `archive object-storage-match`
  - `archive objet-storage-recommendation`
- New document:
  - `wom-kit/docs/object-storage-recommendations.md`

## Behavior

The command maps human scenarios to existing WOM-kit object-storage setup
provider ids and returns a next `archive object-storage --dry-run` command
shape.

Supported scenarios:

- `personal_low_ops`
- `s3_compatible`
- `backup_cost_sensitive`
- `aws_native`
- `google_cloud_native`
- `generic_provider`

## Safety

The command is read-only. It:

- writes no files,
- calls no provider APIs,
- performs no live price lookup,
- checks no bucket availability,
- creates no buckets,
- uploads or downloads no files,
- reads no object bytes,
- creates no presigned URLs,
- starts no OAuth,
- reads no secret values,
- echoes no provider URLs, local paths, object filenames, tokens, or secret
  values.
