# v0.3.179 - remint diagnostic-only redaction

v0.3.179 adds a redacted dry-run JSON view for `archive remint-reconcile`. It is additive:
existing `--format json` output remains unchanged for compatibility, and no migration is
required.

## The problem this fixes

v0.3.176 correctly made `body_diff_diagnostic` content-free: the diagnostic contains only fixed
category labels, byte offsets, and length deltas. But the ordinary `remint-reconcile --format
json` payload still includes `current_canonical_text` and detailed frontmatter value changes for
human review.

That is useful when a reviewer is deciding whether to approve a reconcile, but it is too noisy
for an operator who only wants to inspect the body-diff category and avoid copying canonical
body text into logs or handoff transcripts.

## The fix

`archive remint-reconcile` now accepts:

```bash
archive remint-reconcile <archive-root> --zettel-id <id> --dry-run --diagnostic-only --format json
```

The resulting JSON includes:

- `drift_class`, `classification_basis`, and `content_change_ack_required`;
- `body_changed` and `body_diff_diagnostic`;
- blockers, warnings, next safe actions, and write status;
- `frontmatter_field_change_count` and `frontmatter_field_change_fields`.

It omits:

- `current_canonical_text`;
- the full `frontmatter_field_changes` list with values.

The output also includes `diagnostic_only: true` and an `omitted_fields` list so callers can tell
that they are looking at the redacted projection, not the full review payload.

## Approval remains review-visible

`--diagnostic-only` is refused with `--approve`. Reconcile approval is a human review action:
the operator must still see the current on-disk content before recomputing receipt hashes.

The ordinary `remint-reconcile --format json` output is unchanged for scripts that already rely
on the full review payload.

## Tests

Added focused coverage that:

- a diagnostic-only JSON dry-run omits a distinctive body marker;
- a diagnostic-only JSON dry-run omits a distinctive frontmatter value marker;
- the redacted payload preserves drift/body-diff diagnostics and frontmatter field names;
- `--diagnostic-only` refuses `--approve`;
- `--diagnostic-only` refuses text output.

## Compatibility

Additive; **no migration required**. Existing remint reconcile classifications, apply behavior,
receipt writes, and default JSON/text outputs are unchanged.
