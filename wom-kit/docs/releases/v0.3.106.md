# v0.3.106 - Runtime AI Guide Handoff

Date: 2026-06-17

## What Changed

`archive runtime-context <archive-root> --format json` now returns
machine-readable guide handoff fields inside `canonical_entrypoints`:

- `ai_runtime_order`
- `recommended_first_commands`
- `material_link_routes`

These fields tell terminal-capable AI runtimes to:

1. run `runtime-context`,
2. read `canonical_entrypoints`,
3. read `AGENTS.md` when present,
4. run `ai-response-concept-guide`,
5. choose the correct read-only Notion material-link route.

## Why

The previous docs already said that an AI should start from `runtime-context`,
then local instructions, then the concept guide. But the runtime JSON did not
carry that sequence as structured data. That meant a capable AI could still miss
the guide unless the human explicitly pointed to it.

This release moves the handoff into the product surface itself.

## Material-Link Routes

The new `material_link_routes` list points to the existing read-only paths:

- `notion-objet-import-clue-audit`
- `notion-objet-source-map-link-plan`
- `notion-objet-link-index`
- `notion-objet-link-plan`

Each route is labeled as non-writing and provider-free.

## Safety Boundary

This release is still read-only. It does not:

- read zettel bodies,
- read source bytes,
- write files,
- call providers,
- read secrets,
- start OAuth,
- write material links,
- run live IMAP adapters.

It only returns archive-relative commands, guide order, and safe routing labels.
