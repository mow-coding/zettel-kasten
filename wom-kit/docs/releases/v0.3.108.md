# v0.3.108 - Edge Batch Scale And Rollback

Date: 2026-06-18

## What Changed

This release fixes the same large-manifest pattern in the edge writer that
v0.3.107 fixed in source-map planning.

- `archive zettel-edge-batch` now preloads `objects/manifests/files.jsonl` once
  when batch rows target manifested objets.
- The batch dry-run and approve paths reuse that object-id index instead of
  reloading the full manifest for each objet target.
- `archive revert-edge` can remove one approved edge by replaying its
  `receipts/edges/*.zettel-edge.json` receipt in reverse.
- `archive revert-batch` can remove every edge listed in a
  `receipts/edges/batches/*.zettel-edge-batch.json` receipt.

## Why

Real material-link use confirmed that source-map planning could now produce the
desired zet-to-objet edges, but the next write step still had a scale trap:
large batches of objet targets could repeatedly reload the object manifest.

The same pilot also showed that successful approved writes need a matching
approved rollback command. A human should not have to manually edit many zets
after a reviewed batch write.

## Rollback Model

Rollback is receipt-based and audit-preserving:

- original edge write receipts are not deleted,
- original batch write receipts are not deleted,
- `revert-edge` writes `receipts/edges/reverts/*.zettel-edge-revert.json`,
- `revert-batch` writes per-edge revert receipts plus
  `receipts/edges/batches/reverts/*.zettel-edge-batch-revert.json`,
- batch revert restores touched files if a later item fails.

## Verification

Added regression coverage for:

- object-target batch plans loading the object manifest once,
- `revert-edge` dry-run from an edge receipt,
- `revert-batch` dry-run and approval from a batch receipt,
- original write receipt preservation after rollback.

## Safety Boundary

This release stays local and approval-gated. It does not:

- call providers,
- start OAuth,
- expose an MCP write or revert tool,
- read real source exports,
- read zettel body text,
- read object bytes,
- update object manifests,
- delete original edge or batch receipts,
- echo provider URLs, local absolute paths, page titles, account ids, emails,
  tokens, or secret values.
