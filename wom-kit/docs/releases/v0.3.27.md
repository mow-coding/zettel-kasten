# WOM-kit v0.3.27

Date: 2026-06-15

v0.3.27 adds a read-only credential adapter audit receipt preview.

## Added

- CLI `archive credential-adapter-audit-plan --dry-run`.
- CLI aliases `archive credential-adapter-audit --dry-run` and
  `archive secret-adapter-audit-plan --dry-run`.
- MCP `credential_adapter_audit_plan`.
- `wom-kit/docs/credential-adapter-audit-plan.md`.

## Behavior

The planner previews a non-secret audit receipt for a future approved local
credential adapter operation.

It can describe:

- adapter id,
- adapter kind,
- store kind,
- operation,
- action kind,
- credential id,
- provider label,
- result status,
- approval dependency,
- manifest dependency,
- non-secret result metadata.

## Boundaries

This release does not execute live adapters, write audit receipts, write
approval receipts, write adapter manifests, open password managers, open
browser password stores, open OS keyrings, read environment variables, read
plaintext secret files, write secrets, migrate secrets, or call providers.

The preview includes no secret values, exact credential refs, usernames, email
addresses, tokens, local paths, provider URLs, passwords, or API keys.

## Relationship To Previous Layers

The intended future sequence is:

```text
credential ref
-> adapter manifest
-> readiness check
-> human approval receipt
-> local adapter operation
-> audit receipt
```

v0.3.27 previews only the final non-secret audit receipt shape. It does not
perform the live operation in the middle.
