# v0.3.80 - Connection Evidence Parser Contract

Date: 2026-06-16

Status: v0.3.80 read-only parser contract checkpoint

v0.3.80 continues the #10 connection-import feedback path by defining the
contract a future Notion connection evidence parser must satisfy before it reads
real exports or produces candidate edge records.

## Added

- CLI `archive connection-evidence-parser-contract --source notion --dry-run`.
- CLI aliases `connection-parser-contract` and
  `notion-connection-parser-contract`.
- MCP `connection_evidence_parser_contract`.
- Public doc `docs/connection-evidence-parser-contract.md`.

## Contract

The contract defines safe input lanes for:

- relation properties,
- synced block references,
- database view/filter snapshots,
- internal page hyperlinks,
- page mentions,
- comment context,
- objet embeds.

It also fixes the future candidate edge record fields:

```text
candidate_id
connection_kind
edge_type
source_ref
target_ref
confidence
snapshot_ref
review_status
evidence_ref
```

Dynamic view/filter and comment-context evidence must include reviewed static
snapshots. The future parser may emit candidate records only; a later
approval-gated writer is still required before durable WOM edges exist.

## Safety Boundary

This release still does not implement a Notion evidence parser or edge writer.

The workflow remains read-only: no Notion call, OAuth, export read, comment
read, media download, parser execution, candidate record write, zet write, edge
write, receipt write, object manifest update, provider URL, local path, raw
export path, page title, comment body, account id, email, token, or secret value
is produced.
