# v0.3.185 - adopt nonmatching resume diagnostics and doctor mint-link progress

v0.3.185 responds to the basoon v0.3.183 revalidation.

The revalidation confirmed that the v0.3.183 `adopt-plan: resume summary` works.
It also showed a new operator question: some object-storage locations can be for
the same provider and same object digest, but still not match the current
`store_ref` or resolved `remote_key`. Those rows are evidence, but they are not
resume-skip candidates for the current run.

This release is additive. Defaults remain conservative.

## Same-provider nonmatching adopt diagnostics

`object-storage-adopt-existing --progress` still emits the matching summary:

```text
[object-storage-adopt] adopt-plan: resume summary matching_locations=<n> wom_uploaded=<n> declared_uploaded=<n> other=<n> skip_existing_wom_uploaded=on
```

When same-provider digest-bound locations exist but do not match the current
store/key run, progress now also emits:

```text
[object-storage-adopt] adopt-plan: resume nonmatching-provider summary same_provider_nonmatching_locations=<n> wom_uploaded=<n> declared_uploaded=<n> other=<n> store_ref_mismatch=<n> remote_key_mismatch=<n>
```

The JSON `adopt_summary` now includes:

- `same_provider_nonmatching_location_count`
- `same_provider_nonmatching_wom_uploaded_count`
- `same_provider_nonmatching_declared_uploaded_count`
- `same_provider_nonmatching_other_count`
- `same_provider_store_ref_mismatch_count`
- `same_provider_remote_key_mismatch_count`

If nonmatching declared rows exist, WOM emits `provider_location_mismatch_gap`.
That warning says the rows are legacy or nonmatching location evidence, but not
`--skip-existing-wom-uploaded` candidates for this run.

## Doctor mint-link sub-steps

`archive doctor --strict --progress` now breaks `checking target mint receipt
link` into smaller steps:

- reading target mint block;
- validating target mint block;
- reading target mint receipt_path;
- formatting mint receipt relative path;
- comparing target mint receipt_path;
- recording target mint receipt link error;
- target mint receipt link ok.

This does not change the diagnostic result. It only gives a better last-known
point when a large archive appears to pause inside the first mint receipt.

## Version import-origin diagnostics

`archive version <root>` text output now includes:

```text
Import module: redacted (pass --no-redact-local-paths to inspect)
```

If the project-local `.zettel-kasten/source` mirror differs from the running CLI
version, the warning now explains that the active `archive` command may be
importing a different WOM-kit module. Operators can run:

```powershell
archive version <root> --no-redact-local-paths
```

to inspect the active module path locally. The default remains path-redacted for
public-safe output.

## Tests

Added focused regression coverage for:

- same-provider nonmatching declared location counts;
- `provider_location_mismatch_gap`;
- the new mint-link doctor sub-steps;
- import-origin path redaction and `--no-redact-local-paths` text output.
