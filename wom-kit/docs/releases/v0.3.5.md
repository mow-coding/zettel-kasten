# v0.3.5 - Derived Text Batch Capture And Hygiene Repair

Release date: 2026-06-13

v0.3.5 is a compatible fast-follow to v0.3.4. It improves the derived-text
registration workflow for large local extraction runs and adds an approved
repair path for missing WOM-kit safe `.gitignore` patterns.

## Added

- CLI `archive derive-text capture <archive-root> --from-manifest <jsonl> --dry-run|--approve --reviewed-by <actor>`.
- JSONL batch derived-text input with required fields:
  `source_object_id`, `text_file`, `derivation_kind`, `tool_name`,
  `tool_version`, and `review_status`.
- Optional batch fields:
  `item_id`, `model_name`, `model_version`, `confidence`, `language`, and
  `born_digital`.
- CLI `archive repair-gitignore <archive-root> --dry-run|--approve --reviewed-by <actor>`.
- Tests for batch dry-run, approved batch capture, duplicate simulation, CLI
  gates, and approved `.gitignore` repair.

## Changed

- `derive-text capture` supports both single-file input and JSONL batch input.
- Relative `text_file` values in a derived-text batch manifest resolve from the
  manifest file location.
- Batch dry-runs simulate duplicate identical rows so repeated items are shown
  as skips rather than as repeated captures.
- Public guardrail code and docs no longer include private dogfood archive
  identifiers; generic live-archive and local `*-objets` protections remain.
- Project intake docs now reflect the current local objet capture, derived text
  capture, and staged cleanup verifier primitives.

## Boundaries

- `derive-text capture --from-manifest` registers already extracted UTF-8 text.
  It does not run OCR, ASR, parsers, LLM vision, provider APIs, drafting, or
  minting.
- `repair-gitignore` appends missing safe defaults only. It does not delete or
  rewrite existing entries, clean files, inspect source bodies, upload, sync, or
  call provider APIs.
- No frontmatter or manifest migration is required for v0.3.4 users.

## Upgrade

After approving derived-text batch capture, rebuild the generated search index
if you want the new derived text to appear in search:

```text
archive index <archive-root>
```
