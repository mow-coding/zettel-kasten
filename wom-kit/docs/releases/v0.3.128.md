# v0.3.128 - Notion Client Fixture Request Package

Release date: 2026-06-22
Status: prerelease

v0.3.128 adds the read-only follow-up package for the v0.3.127 client issue
verifier. Instead of guessing what to ask a client for, WOM-kit can now produce
the minimal sanitized fixture request contract.

## Added

```powershell
archive notion-client-fixture-request-plan <archive-root> `
  --source notion `
  --dry-run `
  --format json
```

MCP:

```text
notion_client_fixture_request_plan
```

The request package lists:

- accepted fixture kinds,
- required safe fields,
- a redaction checklist,
- recommended verification commands after receipt.

When a local sanitized tree, mirror, or ancestor fixture is supplied, the command
can reuse the v0.3.127 verifier and decide whether the next requested fixture
should be a nested tree, block mirror, ancestor result, or none.

## Still Closed

This release still does not send client messages or add live Notion transport.
It does not call providers, start OAuth, open Notion connections, read real
export directories, read page titles or bodies, read comments, download media,
write request files, write or merge fixture files, write zets, mint pages,
write edges, write receipts, update object manifests, or echo provider URLs,
local paths, account ids, emails, tokens, or secret values.
