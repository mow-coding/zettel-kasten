# WOM-kit v0.3.24

Date: 2026-06-15

v0.3.24 adds a read-only credential access approval receipt preview and fixes a
credential-ref validation false positive.

## Added

- CLI `archive credential-access-approval-plan --dry-run`.
- CLI aliases `archive credential-access-approval --dry-run` and
  `archive secret-access-approval-plan --dry-run`.
- MCP `credential_access_approval_plan`.
- `wom-kit/docs/credential-access-approval-plan.md`.

## Fixed

- The secret scanner no longer treats declared credential refs such as
  `secret:keepassxc-personal-mail` or `keyring:openai-api-key` as secret values
  when they appear in the local credential-ref catalog.
- Raw secret-like values are still detected by `doctor` and `validate`.

## Decisions

The planner supports:

- `needs_review`,
- `approve_once`,
- `deny`.

Even `approve_once` is only a preview in this release. It writes no receipt and
grants no live access.

## Behavior

The planner emits a proposed receipt path and receipt preview for future
credential broker approval.

The preview includes safe metadata only:

- credential id,
- credential kind,
- provider,
- purpose,
- ref store and prefix,
- action kind,
- store kind,
- consumer/tool label,
- reviewer label,
- decision.

It does not include the exact credential ref value or any secret value.

## Boundaries

This release does not write approval receipts, grant live approval, open
password managers, open browser password stores, open OS keyrings, read
environment variables, read plaintext secret notes, write vault entries, call
providers, draft zets, or mint zets.
