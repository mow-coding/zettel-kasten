# v0.3.78 - Object Storage Upload Execution Contract

Date: 2026-06-16

v0.3.78 continues the #11 object-storage feedback path without jumping straight
to live uploads.

- Public doc:
  `wom-kit/docs/object-storage-adapter-execution-contract.md`

## What It Defines

- sha256 content-addressed remote key shape,
- verified human approval receipt before future execution,
- local SHA-256 verification before upload,
- provider HEAD/idempotency checks before and after upload,
- bounded retry and resume ledger for large media,
- non-secret execution receipt,
- manifest location update only after provider confirmation,
- no ETag-as-WOM-SHA-256 shortcut unless provider policy verifies it for the
  exact upload mode.

## Safety Boundary

This release still does not implement a live object-storage adapter.

The new command writes nothing, calls no providers, retrieves no secrets, reads
no object bytes, computes no local file hash, uploads nothing, checks no remote
availability, writes no resume ledger or execution receipt, updates no manifest,
and echoes no bucket names, prefixes, provider URLs, local absolute paths, exact
credential refs, tokens, or secret values.
