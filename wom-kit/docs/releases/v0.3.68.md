# v0.3.68 - IMAP Material Selection Record

Status: v0.3.68 approval-gated IMAP material selection record
Date: 2026-06-16

v0.3.68 adds the first non-secret write step after the IMAP material selection
plan.

## Added

- `archive imap-mailbox-material-selection-record <archive-root>
  --execution-receipt <archive-relative-json> --selection-mode
  human_review_queue|body_candidates|attachment_candidates|derived_text_candidates
  --selected-index <n> --dry-run|--approve --format json`.
- Aliases:
  - `archive imap-material-selection-record`
  - `archive mailbox-material-selection-record`
- A public documentation page:
  - [IMAP Mailbox Material Selection Record](https://github.com/mow-coding/zettel-kasten/blob/main/wom-kit/docs/imap-mailbox-material-selection-record.md)

## Behavior

The command validates an existing non-secret IMAP header metadata scan execution
receipt, checks that requested one-based candidate indexes are in range, and
can write a material selection receipt under
`receipts/imap/material-selections/`.

The selection receipt records selected indexes, selected count, candidate pool
count, the execution receipt SHA-256, and the future material lane.

## Safety Boundary

This release still does not implement message body capture, attachment capture,
or mail-derived text extraction.

The command writes no candidate refs, echoes no execution receipt path, opens no
IMAP connection, reads no environment variables, opens no keyring/password
manager, reads no message headers again, reads no message bodies, reads no
attachments, creates no derived text, starts no OAuth, calls no providers, and
echoes no subjects, senders, recipients, local paths, tokens, or secret values.

## Verification

- Capability matrix and public documentation tests cover the new command,
