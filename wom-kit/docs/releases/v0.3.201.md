# v0.3.201 - AI start-here surface

v0.3.201 adds the first compact "start here" surface for AI operators entering
a WOM archive.

No archive migration is required.

## What Changed

New read-only CLI:

```text
archive ai-start-here <archive-root> --dry-run --format markdown
archive ai-start-here <archive-root> --dry-run --format json
```

Aliases:

```text
archive start-here
archive operator-start-here
```

The command turns the existing runtime-context materials into a single first
map:

- archive id, type, scope, and start-here file,
- present and missing entrypoints,
- first safe commands,
- operational-context status and session-start handoff,
- next safe steps,
- conversation status-board sections,
- read-only safety boundaries.

## Why This Exists

WOM already exposed the underlying pieces through `runtime-context`,
`canonical_entrypoints`, `AGENTS.md`, `operational_context`, capabilities, and
response contracts. The missing layer was a beginner- and AI-facing signpost:
one obvious command an entering operator can run before broad archive
exploration.

## Safety

The new surface is a projection over existing metadata. It:

- writes no files,
- calls no providers,
- reads no secrets,
- reads no zettel bodies,
- reads no objet bytes,
- echoes no local absolute paths by default,
- does not summarize private archive knowledge into a new body document.
