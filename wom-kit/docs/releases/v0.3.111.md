# v0.3.111 - LaTeX Path Hygiene And Import Fidelity

Date: 2026-06-18

## What Changed

This release handles the ninth real-use feedback sweep after v0.3.110.

Two areas changed:

- escaped LaTeX commands in draft bodies no longer look like Windows network
  paths to the forbidden-location guard;
- structured `import-external` manifests can preserve more of the migration
  shape without weakening the privacy gates.

## LaTeX Path Hygiene

The forbidden-location detector still blocks real Windows drive paths, UNC
paths, and Unix-style local absolute paths.

It now skips common escaped LaTeX commands such as:

- `\\frac`
- `\\sigma_Y`
- `\\sum`

This prevents the mint checklist from treating math notation as a private local
path.

## Import Fidelity

`archive import-external` now accepts and preserves safe manifest metadata for:

- zettel id overrides such as `zet_*`,
- arbitrary safe `facets`,
- safe object source refs,
- safe non-object `source_refs`.

Dry-run previews include counts and routing metadata, but still do not echo
object id values or provider locator values.

New option:

```powershell
archive import-external <archive-root> `
  --source notion `
  --export <manifest.json> `
  --provider-locator-policy object-ref `
  --dry-run
```

When exactly one safe object source ref is present, the `object-ref` policy can
convert supported Notion body locators to one reviewed `objet:<object_id>`
reference before the draft is written.

The default `preserve` policy remains conservative: imported bodies that would
leave provider URLs or local absolute paths now block before approval, matching
the later mint gate instead of importing a draft that would fail later.

## Safety Boundary

This release does not:

- call Notion, Google Drive, or other providers,
- start OAuth,
- store provider secrets,
- read object bytes,
- upload files,
- write edges,
- create a live importer adapter,
- echo object id values or provider locator values in dry-run batch output,
- rewrite existing zettel bodies.

Approved import still writes only the governed inbox draft and one
external-import receipt.
