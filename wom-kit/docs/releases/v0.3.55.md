# v0.3.55 - IMAP Adapter Preflight Plan

Date: 2026-06-16

v0.3.55 adds a read-only final preflight before any future live IMAP adapter.

The release still does not connect to mail providers. It only composes existing
local planning and approval gates.

## Added

- CLI `archive imap-mailbox-adapter-preflight-plan --dry-run`.
- Aliases:
  - `archive imap-mailbox-adapter-execution-preflight --dry-run`
  - `archive mailbox-adapter-preflight --dry-run`
- MCP tool `imap_mailbox_adapter_preflight_plan`.
- `preflight_state` with:
  - `ready_for_future_adapter_after_approval`
  - `blocked`

## Verify

```bash
archive imap-mailbox-adapter-preflight-plan <archive-root> \
  --adapter-id local-imap \
  --source-id imap:gmail-personal \
  --provider gmail \
  --account-ref imap:account:gmail-personal \
  --username-ref env:WOM_GMAIL_USERNAME \
  --auth-mode oauth_token_ref \
  --oauth-token-ref keyring:gmail-oauth \
  --mailbox-ref imap:mailbox:inbox \
  --credential-id cred:gmail-mail-access \
  --operation header_metadata_scan \
  --selection-rule newest_first \
  --selector-id mail-selection:recent-inbox \
  --approval-decision approve_once \
  --approval-receipt receipts/credentials/access-approvals/<receipt>.json \
  --dry-run \
  --format json
```

The JSON is ready only when:

```text
preflight_state: ready_for_future_adapter_after_approval
gate_summary.adapter_manifest_status: present_and_schema_valid
gate_summary.request_state: ready_for_future_adapter_after_approval
gate_summary.selection_state: ready_for_future_adapter_after_approval
audit_preview_summary.result_status: not_run
closed_actions.imap_connection_opened: false
closed_actions.credential_value_read: false
```

## Safety

The preflight remains read-only. It:

- writes no files,
- opens no IMAP connection,
- attempts no login,
- selects or searches no mailbox,
- lists no candidate messages,
- reads no IMAP UIDs or Message-ID values,
- reads no headers, bodies, or attachments,
- creates no derived text,
- retrieves no secrets,
- starts no OAuth,
- calls no providers,
- echoes no email addresses, username values, exact account refs, exact
  credential refs, exact mailbox refs, IMAP host values, provider URLs, message
  ids, subjects, sender or recipient values, attachment names, approval receipt
  paths, selection receipt paths, schema validation issue values, local absolute
  paths, tokens, or secret values.

No archive migration is required.
