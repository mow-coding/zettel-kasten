# v0.3.66 - External Export Plan

Date: 2026-06-16

## Summary

v0.3.66 adds a read-only pre-export planner for Notion, Google Drive, and
generic workspace exports.

The goal is to stop a human or AI helper before a broad export accidentally
pulls large media, uploaded files, attachments, images, audio, or video into the
first local download when the immediate WOM task only needs text.

## What Changed

- Added `archive external-export-plan`.
- Added source choices for `notion`, `google_drive`, and `generic_workspace`.
- Added export goals for `text_only`, `targeted_pages`, and
  `full_workspace_review`.
- Added media policies for `avoid_bulk_media`, `selected_media_only`, and
  `full_media_requested`.
- Added high-risk `stop_and_split_media_before_export` guidance for broad media
  downloads.
- Linked the later `scan-source`, `import-external`, and object-storage
  recommendation command shapes.

## Safety

The planner remains closed:

- starts no provider export,
- calls no providers,
- starts no OAuth,
- reads no files or media bytes,
- downloads no attachments,
- writes no archive files,
- echoes no provider URLs, local paths, filenames, account ids, emails, tokens,
  or secret values.
