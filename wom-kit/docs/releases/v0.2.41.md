# v0.2.41 Release Notes

Date: 2026-05-26

## Summary

v0.2.41 adds a read-only non-binding attestation statement draft preview for one recorded foreign block attestation review candidate.

The new command answers what a human might review later if they choose to create a real attestation in a future release. It does not create the attestation.

## Added

- CLI `archive attestation-statement-draft <archive-root> --case-id <safe-case-id> --dry-run --format json`.
- Optional `--expected-review-scope`, `--prospective-attestor`, `--statement-style`, and `--review-note`.
- Read-only MCP `foreign_block_attestation_statement_draft_preview`.
- Statement styles: `minimal`, `review_checklist`, and `human_readable`.
- Replay validation against current candidate, candidate receipt, quarantine case/receipt, and decision record/receipt state.

## Safety Boundary

The preview:

- writes nothing,
- keeps `dry_run: true`,
- keeps `trust_state: untrusted_foreign`,
- keeps `draft_status: preview_not_recorded`,
- keeps `attestation_status: not_created`,
- keeps `signature_status: not_created`,
- returns `would_change: []`,
- never reads original foreign artifacts, source payloads, objet bodies, provider URLs, or foreign body text,
- never creates trust, import, attestation, signatures, receipts, minting, sharing, provider calls, ZET transport, acceptance, or apply behavior.

Hash commitments are retained only as `not_verified`, `not_trusted`, and not proof of authenticity.

## Compatibility

No private archive migration is required.

MCP remains read-only and exposes no statement draft write/apply, foreign block attest/sign/trust/import/accept, receipt-write, full-auto, provider, or ZET transport tool.
