# v0.3.227 - Aggregate Full-Doctor Edge Progress

v0.3.227 keeps the v0.3.225 targeted edge-receipt performance improvement
while removing the resulting compact progress flood. It is additive and
requires no archive migration.

## What Changed

- Compact `edge-receipt-index` prints start/done and only adds a count heartbeat
  when the stage remains active for 10 seconds.
- Targeted source loads maintain one complete-Doctor aggregate:
  `sources`, candidate receipt documents opened, and source-result `cache_hits`.
- The shared runtime reporter keeps that aggregate alive across surrounding
  mint-receipt events, emits it on 10-second heartbeat, and prints one final
  `done` summary.
- Per-source candidate-batch `start`, `0/N`, `1/N`, `N/N`, `done`, and cache-hit
  events move to `edge-receipt-source-load-detail`.
- Compact stderr suppresses the detail stage. Direct Doctor
  `--progress-detail verbose` and private `--progress-log` JSONL preserve it.
- Unknown or path-bearing summary text fails the fixed count-only allowlist and
  cannot enter shared progress.
- The lower README versioning example now matches the current public baseline
  instead of labeling v0.3.162 as current.

## Operator Flow

The complete check remains:

```text
archive runtime-context <archive-root> --full-doctor --progress --format json
```

Expect a short index lifecycle, aggregate source-load heartbeat when the work
continues for 10 seconds, and one final count-only source-load summary. The
final source total is dynamic; WOM-kit does not add a second broad pass merely
to predict it.

Use a direct Doctor command only when detailed local diagnosis is needed:

```text
archive doctor <archive-root> --strict --progress --progress-detail verbose
archive doctor <archive-root> --strict --progress --progress-log logs/doctor-progress.jsonl
```

## Benchmark

The content-free synthetic benchmark can be rerun without a real archive:

```text
python tools/benchmark_doctor_progress_volume.py --source-count 8583 --index-receipt-count 21539 --format json
```

Observed on the release workstation:

- shared runtime compact: 4 lines, 358 UTF-8 bytes, 0.045399 seconds;
- direct verbose: 51,457 lines, 6,324,409 UTF-8 bytes, 0.180271 seconds;
- line reduction: 99.9922%;
- byte reduction: 99.9943%;
- verbose/shared in-memory elapsed ratio: 3.971x.

Timing uses in-memory stderr capture and measures Python formatting/write
overhead only. Interactive terminal rendering can add more cost; the timing is
a conservative lower bound, not a real-archive completion-time claim.

## Privacy And Work Boundary

The patch changes progress routing only. It changes no diagnostic, receipt,
manifest, zet, objet, provider, credential, or archive file. Shared output
contains fixed labels and counts only. Private progress logs retain detailed
events under the existing local-artifact policy.
