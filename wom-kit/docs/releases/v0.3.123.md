# v0.3.123 - Containment Link Type

Date: 2026-06-21

## What Changed

Real Notion backup work exposed a relation WOM could not name cleanly:

```text
parent page or zet contains child page, child database, collection view, or nested archive object
```

Earlier edge types were close but wrong:

- `view_query` is for reviewed static view/filter result snapshots.
- `references` is a loose pointer.
- `material` is source input.
- `inherited_by` is lineage or succession.

v0.3.123 adds a dedicated `contains` link type for structural containment.

## Connection Planning

`archive connection-import-plan --source notion --connection-kind all --dry-run`
now includes `notion_containment` evidence and maps it to `contains`.

The read-only parser contract and sanitized fixture parser also recognize:

```text
child_page
child_database
collection_view
nested export refs
```

The fake fixture stays safe: it uses only fake refs and reads no client export.

## Model-Gap Guardrail

This release also makes the AI behavior rule explicit:

```text
If the captured relation does not fit the active WOM edge vocabulary,
do not silently map it to the nearest existing type.
Report a model gap and ask for a developer decision before durable writes.
```

That rule prevents structural containment from being forced into `view_query`,
`references`, `material`, or `inherited_by`.

## Compatibility

Existing archives do not need an automatic migration, but stale archive-local
`zettel-kasten/types.yml` files can use:

```powershell
archive migrate <archive-root> --target link-types-v0.3 --dry-run --format json
archive migrate <archive-root> --target link-types-v0.3 --approve --format json
```

The migration appends missing recommended connection edge types from the base
template and writes a receipt. Revert remains receipt-based and blocks if the
appended type is already used or locally modified.

## Safety Boundary

This release does not:

- call Notion or other providers,
- read real exports,
- read zettel bodies,
- read derived-text bodies,
- write durable edges by itself,
- write candidate records,
- upload objects,
- add MCP write tools,
- infer containment from private page titles or body text.

It only adds the vocabulary, read-only planning contract, sanitized fixture
coverage, beginner-facing explanation, and tests.
