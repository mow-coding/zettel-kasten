# WOM-kit v0.3.16

Date: 2026-06-14

v0.3.16 clarifies the Notion page snapshot and external `store-ref` model
without adding provider sync or new write behavior.

## Added

- `wom-kit/docs/notion-page-snapshot-model.md`.
- Public documentation coverage for Notion `recordMap` / `blocks` JSON as a
  provider page snapshot source objet.
- Documentation tests that keep page snapshots, derived text, and human-authored
  zets separate.

## Clarified

- A page snapshot JSON is source/original evidence when preserved.
- Extracted readable block text is a derived text record.
- A human-reviewed note, conclusion, summary, or decision is a later draft or
  minted zet.
- In prehashed ledgers, `object_id` answers "what bytes?", `store_kind` answers
  "what storage family?", and `store_ref` answers "which reviewed external
  store label?".
- `store_ref` is not a raw local path, private URL, account id, token, email
  address, secret, or byte-availability proof.

## Boundaries

This release does not add Notion API calls, OAuth, provider sync, page-snapshot
schemas, extraction helpers, object upload, byte materialization, drafting,
minting, ZET transport, or cleanup automation.
