# v0.3.102 - Operational Terminology And Batch Ergonomics

Date: 2026-06-17

## Summary

v0.3.102 turns the latest Phase 3 field feedback into three local-first safety
improvements:

- `archive ai-response-concept-guide --topic operational_terms --locale ko-KR`
  now returns a read-only translation layer for edge types, lifecycle terms, and
  connection kinds.
- `connection-edge-intelligence-plan` can recommend the existing `supersedes`
  edge type when sanitized fixture metadata carries a version-chain hint.
- `zettel-edge-batch` resolves `--plan workbench/...` archive-relative first
  and supports explicit `--skip-existing` reruns for already-written edge rows.

## User-Facing Meaning

AI runtimes no longer need to show raw terms such as `derived_from`,
`references`, or `supersedes` without explanation. The guide returns plain
phrases such as:

```text
This note was made from that source.
This note points to or refers to that one.
This newer note replaces the older one.
```

For Korean locale, the same card includes beginner-facing Korean phrases for
the same internal terms.

## Safety Boundary

This release adds no live provider adapter and no broader parser. The new and
changed surfaces:

- call no providers,
- start no OAuth,
- read no real exports,
- read no source bodies or derived-text bodies,
- call no LLM classifier,
- write no candidate records,
- expose no MCP write tool,
- echo no local absolute paths, provider URLs, account ids, emails, tokens, or
  secret values.

Durable edge writes still require the existing approval-gated `zettel-edge` or
`zettel-edge-batch` paths.
