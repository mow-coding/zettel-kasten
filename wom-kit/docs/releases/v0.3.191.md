# v0.3.191 - Doctor target edge-receipt evolution progress

v0.3.191 drills inside `checking target edge-receipt evolution` for
`doctor --strict --progress`.

The previous release localized a slow mint receipt to the target file-reference
check. This release localizes the next inner phase: deciding whether a target
sha mismatch is explained by later approved zettel-edge receipts.

## Edge Evolution Drilldown

When a mint receipt target sha differs from the current target file, progress
can now emit:

- `target edge evolution target <archive-relative-path>`
- `loading target edge receipt candidates`
- `loading edge receipt index`
- `listing edge receipts`
- `scanning edge receipts <current>/<total>`
- `still scanning edge receipts <current>/<total>`
- `edge receipt index ready sources=<count> receipts=<indexed>/<total>`
- `loaded edge receipt index`
- `edge receipt index cache hit`
- `checking target edge receipt candidates <count>`
- `reading target zettel for edge evolution`
- `checking target evolution history strict`
- `checking target evolution history inclusive`
- `target edge-receipt evolution ok`
- `target edge-receipt evolution no match`

The messages are content-free. They expose counts and archive-relative paths,
but not zettel bodies, provider locations, local absolute paths, tokens, or
secret values.

## Long Scan Heartbeat

Edge receipt index scans now have a 30-second quiet-interval heartbeat:

```text
still scanning edge receipts <current>/<total>
```

The regular count-based scan messages still appear at the first item, every
250th item, and the final item.

## Compatibility

Additive; **no migration required**.

Only optional `--progress` stderr output changes. Result JSON, diagnostics,
receipts, manifests, and archive files are unchanged.
