# v0.2.50 Release Notes

Date: 2026-05-27

## Summary

v0.2.50 adds the Korean product-language baseline for WOM, zettel-kasten, zet, ZET, objet, block/header wording, lifecycle verbs, sharing forms, and surface/recommendation phrases.

This is a documentation and version bookkeeping release. It does not add product CLI/MCP behavior.

## Added

- [Korean Product Language Baseline](https://github.com/mow-coding/zettel-kasten/blob/main/wom-kit/docs/concepts/korean-product-language-baseline.ko.md).
- [Public-safe work log](https://github.com/mow-coding/zettel-kasten/blob/main/wom-kit/plans/work-log-2026-05-27-korean-product-language-baseline.md) for the v0.2.50 Korean product-language baseline batch.
- README and public documentation map pointers to the Korean product-language baseline.

## Clarified

- `WOM` is a proper noun pronounced `옴`, not `웜`.
- `zet` can be explained as `쪽글` or `토막글`, but the product term remains `zet`.
- `ZET` can be explained as `공유 계층`, but the product term remains `ZET`.
- `block = zet + header`, with Korean explanation `상자 = zet + 초록`.
- Korean lifecycle terms: `mint -> 발행`, `delegate -> 공유`, `attest -> 수용`, `anchor -> 반영`, and receiver-side `attest + anchor -> 갱신`.
- Korean intake/safety terms: `foreign block -> 소포`, `quarantine -> 검문소`, `trust -> 인증`, `import -> 반입`, and `acceptance -> 채택`.
- Korean surface/action terms such as `projection -> 구현`, `surface -> 수제 앱`, `prompt-as-algorithm -> 수제 알고리즘`, `neighbor -> 이웃`, `feed -> 담벼락`, `broadcast -> 송출`, `receipt -> 영수증`, `provenance -> 족보`, `canonical -> 정본`, and `node -> 노드`.
- SNS-type ZET phrases: `젯 갱신하기` and `쿠키 굽기`.
- Messenger-type ZET thread spelling: `스레드`.

## Compatibility

No private archive migration is required.

No archive product CLI or MCP behavior changed.

The new Korean terms are product-facing explanation guidance only. CLI commands, JSON fields, schema fields, filenames, and implementation identifiers stay English.

## Safety Boundary

v0.2.50 does not add real ZET transport, real trust/import/acceptance/anchor, attestation or signature writes, RF access, key-sharing registry, mirroring delivery, provider calls, provider sync, WordPress publishing, projection writes, projection receipts, recommendation fetching, recommendation ranking, feed updates, payments, staking, consensus, blockchain, model training, backpropagation, or full-auto behavior.
