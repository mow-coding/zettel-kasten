# WOM-kit v0.3.29

Date: 2026-06-15

v0.3.29 adds a read-only plaintext credential migration planner.

## Added

- CLI `archive credential-plaintext-migration-plan --dry-run`.
- CLI aliases `archive secret-migration-plan --dry-run` and
  `archive credential-import-plan --dry-run`.
- MCP `credential_plaintext_migration_plan`.
- `wom-kit/docs/credential-plaintext-migration-plan.md`.

## Behavior

The planner describes a safe future workflow for moving secrets from a
human-selected plaintext note into an external vault/keyring/store.

It can describe:

- safe source label,
- target store id,
- target store kind,
- future adapter kind,
- WOM ref prefix,
- human file-selection requirement,
- per-entry human confirmation requirement,
- old plaintext cleanup as a separate human review,
- relationship to vault onboarding, broker, approval, readiness, manifest, and
  audit layers.

## Boundaries

This release does not read plaintext files, print plaintext file paths, hash
plaintext bytes, detect secret values, return candidate secret values to AI,
open password managers, open browser password stores, open OS keyrings, read
environment variables, write vault entries, write keyring entries, write `.env`
files, call providers, start OAuth, delete plaintext notes, quarantine
plaintext notes, write receipts, draft zets, or mint zets.

It writes no files and includes no secret values, exact credential refs,
usernames, email addresses, tokens, local paths, provider URLs, passwords, or
API keys.

## Relationship To Previous Layers

The intended safe sequence is:

```text
credential-store-recommendation
-> credential-vault-onboarding-plan
-> credential-plaintext-migration-plan
-> credential-ref-plan
-> credential-ref-inventory
-> credential-access-broker-plan
-> credential-access-approval-plan
-> credential-adapter-readiness-plan
-> credential-adapter-manifest-plan
-> credential-adapter-audit-plan
```

v0.3.29 adds only the pathless migration planning step. Live secret scanning,
vault writes, keyring writes, cleanup, and deletion remain future local UI and
adapter work.
