# v0.3.87 - Connection Edge Intelligence Plan

Date: 2026-06-17

Status: v0.3.87 read-only connection edge intelligence checkpoint

v0.3.87 adds the first read-only review-intelligence layer above sanitized
connection candidate parsing.

## Added

- `archive connection-edge-intelligence-plan --evidence <json> --source notion --dry-run`.
- Alias:

```text
connection-edge-classification-plan
```

- Documentation: `wom-kit/docs/connection-edge-intelligence-plan.md`.

## Behavior

The command reuses sanitized fixture candidate previews and reports:

- separate `source_mechanism` and `relationship_meaning` axes,
- current active edge-type meanings,
- provisional meaning labels such as `format_variant`, `responds_to`,
  `fulfills`, `enabling`, and `sequence`,
- ambiguity flags,
- parsimony review signals,
- human review queue entries before any `zettel-edge` write.

The provisional meanings are not active durable edge types in this release.
They are labels for repeated review evidence, so the model can grow without
creating too many edge types too early.

## Safety Boundary

This release remains read-only. It reads no real Notion exports, source body
text, derived-text bodies, or comment bodies, calls no providers, starts no
OAuth, calls no LLM, runs no multi-lens AI classifier, downloads no media,
writes no candidate records, writes no zets, writes no edges, writes no
receipts, and updates no object manifests.

The output also avoids provider URLs, local absolute paths, raw export paths,
page titles, comment bodies, source body text, derived-text body text, account
ids, emails, tokens, or secret values.
