# v0.3.99 - Policy Batch Zettel Edge Write

Date: 2026-06-17

## Added

- CLI `archive zettel-edge-batch <archive-root> --plan <json>
  --dry-run|--approve`.
- Aliases `archive bulk-zettel-edge` and `archive batch-zettel-edge`.
- Public docs for policy-level batch edge approval:
  `wom-kit/docs/zettel-edge-batch.md`.

## Why

The single-edge writer from `v0.3.82` is safe, but it does not scale when a
reviewed import or AI pass produces many obvious candidates.

This release adds the next safe step: a human can approve one explicit policy,
and WOM-kit writes only the candidates that match that policy. Ambiguous,
low-confidence, blocked, or policy-mismatched candidates are returned in
`human_review_queue` instead of being written.

## Behavior

- Reads a JSON plan with `policy` and `edges`.
- Requires either `--dry-run` or `--approve`.
- Requires `--reviewed-by` for approved writes.
- Treats `auto_write_edge_types` and `minimum_confidence` as the batch policy.
- Sends rows outside the policy to `human_review_queue`.
- Dry-runs every policy-writable item through the existing single-edge writer.
- Approved mode writes the individual edge receipts plus one batch receipt.
- Blocks duplicate batch receipts and duplicate edge rows.
- Restores touched zettel and receipt files if an approved batch write fails.

## Safety

The new command is still CLI-only. MCP exposes no matching write tool.

It does not classify candidates by itself, parse real exports, call providers,
start OAuth, open Notion, read comments, download media, call an LLM, write
candidate records, update object manifests, upload objects, or create provider
URLs.

Output avoids zettel body text, zettel titles, provider URLs, local absolute
paths, page titles, comment bodies, account ids, emails, tokens, and secret
values.
