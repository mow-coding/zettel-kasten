# v0.3.254 - Independent Audit Follow-Up And Hash-Bound Body Reading

v0.3.254 selectively implements findings from an independent read-only
v0.3.253 audit while preserving WOM's artifact-first, local-first, and
human-approved boundaries.

## What Changed

- `read-zettel` can return an opt-in bounded body page through CLI and MCP.
  Every nonzero continuation cursor requires the complete decoded-body hash
  returned by the first page, so a changed zet cannot be mixed into one read.
- Derived-text coverage and Doctor now identify changed pre-normalization
  transcript or export bytes whose matching objet identity is absent from the
  object manifest.
- AI artifact inventory and session-handoff results state that their scan is
  allowlisted and cannot prove archive-wide absence. New receipts preserve that
  scope statement without invalidating existing valid receipts.
- Live-file catalog, matching-title separation, and MOW Harness namespace
  isolation now have behavioral regressions instead of documentation evidence
  alone.
- Current operator language uses `zet` for document traversal while preserving
  `node` for a subject/archive participant and leaving compatibility fields and
  historical records unchanged.
- Object-storage upload help now describes the live approval-gated
  S3-compatible transport that already shipped.

## Why It Matters

The release closes practical gaps behind WOM's Memento Problem answer. A host
can read one large zet without spending the whole context at once, but it cannot
silently join pages from different body versions. A normalized transcript no
longer implies that the exact input bytes remain recoverable merely because a
hash was recorded. A clean session handoff no longer reads like proof that the
entire archive was searched for every possible AI-generated file.

## Compatibility And Safety Boundary

- No archive migration is required.
- Existing default full-body reads are unchanged.
- CLI body paging is JSON-only; MCP remains structured JSON.
- No original transcript or export is reconstructed, copied, or captured
  automatically.
- Manifest identity coverage does not prove current local or remote byte
  availability.
- Existing valid session-handoff receipts remain compatible.
- No schema field, command, output key, receipt, historical release note, or
  archive identifier is renamed for the terminology clarification.
- No provider, model, database, credential store, or network is called by the
  new read-only diagnostics.

## Provenance

The corrections were prompted by an independent read-only v0.3.253 audit from
Claude Opus 4.8. The candidate patch was then handed back from an accidentally
used client-support session. The WOM development session independently reviewed
the complete diff, corrected an unbound-continuation flaw, preserved historical
decision wording, reverified source behavior, and ran the release gates. The
audit is credited as input; it is not treated as implementation or test proof.

## Verification

Focused regressions cover the live upload help, stale generated index,
matching-title separation, bounded handoff scope, source-byte objet identity,
hash-bound body pages, and MOW local namespace isolation. The complete WOM-kit
test suite, package-resource synchronization, clean-wheel installation, public
privacy scan, and release-readiness gate remain mandatory for publication.
