# v0.3.73 - Prehashed MIME And Derived Text Coverage Feedback

Status: v0.3.73 prehashed external ledger and derived-text coverage feedback checkpoint
Date: 2026-06-16

v0.3.73 responds to real archive-intake feedback where externally verified
prehashed objets were registered without useful MIME metadata, while derived
text had already been captured for many of those objects.

## Added

- `archive prehashed-objet-ledger --mime-field <json-field>`.
- MCP `prehashed_objet_ledger_preview` input `mime_field`.
- `wom-kit/schemas/derived-text-capture-manifest-item.schema.json`.

## Behavior

`prehashed-objet-ledger` still reads only UTF-8 JSONL ledger rows, but it can
now preserve safe MIME values from a reviewed field such as `mime`. The output
counts MIME-present, invalid-MIME, and duplicate-MIME-conflict rows without
echoing row values.

`derive-text coverage` still classifies textual candidates by extension and
MIME first. If an older external manifest row lacks useful MIME/extension data
but already has a matching derived-text record, coverage now treats that
derived-text record as a conservative textual signal. This prevents a real
extraction pass from looking like a misleading `0/0` coverage result.

The derived-text batch manifest required fields are now documented with a JSON
Schema, including `tool_version`.

## Safety Boundary

This release does not implement object-storage upload, object download,
provider sync, Notion sync, OCR, parser execution, ASR, LLM vision, drafting,
minting, or live material capture.

The changed paths read no blob bytes, source text bodies, provider URLs, local
absolute paths, tokens, or secret values. They write no files in dry-run mode,
call no providers, upload nothing, download nothing, and still avoid echoing
ledger row values, source filenames, local paths, provider URLs, tokens, or
secret values.

## Verification

- CLI tests cover safe MIME preservation from prehashed ledgers and fallback
  derived-text coverage when external manifests have unknown MIME.
- Documentation and capability matrix text cover the new MIME field, coverage
  fallback, and derived-text capture manifest schema.
