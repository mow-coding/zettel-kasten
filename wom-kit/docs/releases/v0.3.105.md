# v0.3.105 - External Import Source Ref Preservation

Date: 2026-06-17

## Summary

v0.3.105 tightens the import-time material-clue contract. When an external
import manifest already contains an explicit safe object ref, approved
`archive import-external` writes now preserve that clue in the imported draft
zettel's `source_refs`.

Supported manifest clues are:

```text
object_id
source_object_id
approved_object_id
target_object_id
objet_ref
source_refs[] with object_id or objet_ref values
```

## User-Facing Meaning

This release handles the safest import-time preservation case:

```text
If the manifest already knows the object id, keep that object id in source_refs
when writing the imported draft.
```

Dry-run output reports `source_ref_count` and `source_refs_preserved` without
echoing the object id value in the batch preview.

## Safety Boundary

This release does not make import-external a live provider adapter. It:

- does not treat the imported text body hash as a source object id,
- does not preserve provider URLs as object refs,
- reads no object bytes,
- calls no providers,
- uploads nothing,
- writes no edges,
- creates no extra receipts beyond the existing approved external-import
  receipt,
- adds no MCP apply tool.

## Migration

No archive migration is required.

Existing imports are unchanged. New approved imports can preserve safe material
clues only when the manifest explicitly supplies them.
