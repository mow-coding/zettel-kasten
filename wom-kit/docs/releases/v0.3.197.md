# v0.3.197 - Reconcile dry-run next-action guidance

v0.3.197 follows the v0.3.196 doctor path-policy release by improving the next
operator step after a reconcile dry-run classifies drift.

No archive migration is required.

## What Changed

`archive remint-reconcile --dry-run --format json` and
`archive retire-draft-reconcile --dry-run --format json` now include:

```json
{
  "status": "needs_content_change_review",
  "overall_status": "needs_content_change_review",
  "suggested_next_action": "review_content_change_before_approval",
  "would_write": false,
  "approval_would_write": true,
  "approval_requires_content_changed_ack": true,
  "next_safe_actions": [
    "Review the current canonical and receipt/snapshot evidence outside the redacted JSON summary.",
    "If the content change is intentional, rerun `archive retire-draft-reconcile <archive-root> --zettel-id <id> --approve --reviewed-by <actor> --content-changed-ack`.",
    "If the content change is not intentional, stop and restore or repair the content before reconciling."
  ]
}
```

The exact command name follows the command being run.

`format_drift` plans now report:

- `status: format_drift_ready_for_review`
- `suggested_next_action: approve_reconcile_after_format_review`
- `approval_requires_content_changed_ack: false`

Clean/no-op and blocked plans also receive explicit status fields.

## Text Mode

`archive retire-draft-reconcile --format text` now prints `Next safe actions`
when the plan includes them. This brings it in line with the JSON guidance and
with nearby operator-facing commands.

## Safety

This is an additive operator-guidance release.

- Reconcile classification is unchanged.
- Dry-runs still write nothing.
- `content_change` still requires `--content-changed-ack` on approve.
- `--approve` still requires `--reviewed-by`.
- No zettel content is edited by reconcile.
