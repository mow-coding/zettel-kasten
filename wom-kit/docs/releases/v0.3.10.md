# v0.3.10 - Project Intake Cookbook

Release date: 2026-06-14

v0.3.10 is a compatible documentation and regression-test follow-up to v0.3.9.
It makes the human-guided project-intake spine easier to rehearse on fake data
before using a real personal archive.

## Added

- `wom-kit/docs/project-intake-cookbook.md`, a fake-archive walkthrough for the
  manual command spine from project-intake session planning through one-answer
  receipt recording, status review, source-intake, capture selection, capture,
  draft/mint gates, and cleanup verification.
- README and public documentation map links for the cookbook.
- README release-baseline bookkeeping for v0.3.10, including the Korean README
  release tag list.

## Changed

- Strengthened the project-intake-to-objet-capture roundtrip regression so it
  creates its project-intake receipt through the single-answer
  `project-intake-record-answer` path before passing that receipt into
  source-intake and capture context checks.
- Clarified that the project-intake primitives now form a manual command spine,
  not a joined executor.

## Boundaries

- The cookbook uses public-safe fake data only.
- The workflow remains manual and review-gated. It does not classify a whole
  folder automatically, run provider sync, upload files, create drafts, mint
  zets, or delete staged folders without separate command gates.
- Receipts remain context evidence, not automatic execution permission.

## Upgrade

No archive migration is required for v0.3.9 users.
