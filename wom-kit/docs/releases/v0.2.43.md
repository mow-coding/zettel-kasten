# v0.2.43 Release Notes

Date: 2026-05-26

## Summary

v0.2.43 adds a read-only review index for recorded foreign block attestation statement drafts.

This is not an attestation, signature, trust, import, mint, anchor, delegation, sharing, provider sync, or ZET transport release.

## Added

- `archive attestation-statement-draft-review <archive-root> --format json`
- Optional `--case-id`, `--statement-style`, `--review-scope`, and `--include-receipts`
- Read-only MCP `foreign_block_attestation_statement_draft_review_index`
- Consistency validation for statement draft records and matching receipts
- Revalidation of current candidate, quarantine case, and decision chain metadata

## Safety

The review index keeps:

- `dry_run: true`
- `trust_state: untrusted_foreign`
- `attestation_status: not_created`
- `signature_status: not_created`
- `index_status: indexed_not_modified`
- `would_change: []`
- all import/trust/attestation/signature/mint/provider/ZET/share flags false

It does not read original foreign payloads, source payloads, objet bodies, provider URL targets, or external provider state.

## Filter Semantics

`--statement-style` and `--review-scope` filter displayed records only. WOM-kit still validates every discovered statement draft record before setting top-level `ok`.

`--case-id` scopes the consistency verdict to the requested case.

## Compatibility

No private archive migration is required.

Existing v0.2.42 statement draft records remain local untrusted review records. v0.2.43 only adds a read-only index and validation surface for those records.
