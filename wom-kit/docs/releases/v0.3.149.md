# v0.3.149 - Operator Feedback Lifecycle

v0.3.149 gives operator-generated tool feedback a first-class metadata and
lifecycle surface.

## What Changed

- Added read-only `archive operator-feedback-plan`.
- Added aliases `archive feedback-plan` and `archive ops-feedback-plan`.
- Added approval-gated `archive operator-feedback-record`.
- Added aliases `archive feedback-record` and `archive feedback-register`.
- Approved records write:
  - `ops/feedback/<feedback-id>.yml`,
  - a receipt under `receipts/operator-feedback/`.
- Added explicit lifecycle statuses:
  - `draft`,
  - `delivered`,
  - `acknowledged`,
  - `resolved`,
  - `archived`.

## Safety Boundary

The commands:

- read no feedback body,
- copy no body files,
- submit nothing externally,
- call no providers,
- check no network,
- echo no feedback ref values,
- echo no title values,
- echo no local absolute paths, tokens, or secret values.
