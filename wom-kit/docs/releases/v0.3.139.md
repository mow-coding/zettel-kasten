## Tiro Recovery

The Tiro recovery plan records the official-data surfaces a future live adapter
must cover: workspaces, notes, note metadata, transcript paragraphs, summaries,
generated documents, document templates, folders, word memories, wiki data,
share links, translations, audio availability or gaps, pagination, rate-limit,
and error observations.

The capture command preserves a reviewed raw Tiro recovery JSON bundle as a WOM
objet:

```text
workbench/*.json
-> objects/sha256/<prefix>/<sha256>
-> objects/manifests/files.jsonl
-> receipts/tiro/lossless-recovery/*.json
```

The stored object keeps the raw bundle bytes exactly. Output and receipts report
only hashes, counts, archive-relative paths, and gap categories. They do not
echo meeting titles, transcript text, participant names, emails, provider URLs,
tokens, or secret values.

## zet Markdown Tilde Guard

WOM zets remain Markdown-compatible. For range notation, AI writers should now
use:

```markdown
A ~ B
```

Do not use `A~~B` or `A ~~ B` for ranges. Double tilde remains reserved for
intentional Markdown strikethrough.
