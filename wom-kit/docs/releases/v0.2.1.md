# v0.2.1 Public Documentation And Hygiene Patch

Date: 2026-05-23
Status: pre-release

## Summary

`v0.2.1` improves the public repository as a versioned, reader-friendly project.

It does not introduce a breaking archive protocol change.

## What Changed

- Added `UPGRADE.md`.
- Added per-version release notes under `wom-kit/docs/releases/`.
- Clarified that public releases are compatibility checkpoints.
- Clarified that ordinary document files such as `.hwp`, `.hwpx`, `.docx`, `.xlsx`, `.pdf`, `.txt`, `.md`, and `.csv` can be source/original objects.
- Generalized public examples that previously looked too close to local or personal context.
- Replaced private-looking example family identifiers with neutral example identifiers.

## Compatibility

Compatible with `v0.2.0`.

No schema migration is required for existing fake examples.

## Upgrade From `v0.2.0`

For users inspecting the public reference implementation:

```bash
git fetch --tags
git checkout v0.2.1
```

For contributors using `main`:

```bash
git pull
cd wom-kit
python -m unittest discover -s tests
python cli/archive.py doctor examples/fake-life-archive --strict
```

## Archive Migration

No private archive migration is required.

## Verification

Expected checks:

```bash
cd wom-kit
python -m unittest discover -s tests
python cli/archive.py doctor examples/fake-life-archive --strict
```

Expected result:

```text
tests pass
doctor reports 0 errors and 0 warnings
```

## Privacy

This patch continues the rule:

```text
public repo -> code, specs, schemas, fake examples, public-safe docs
private repo -> real archives, real zets, real source maps, provider credentials
```
