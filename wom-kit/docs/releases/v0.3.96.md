# v0.3.96 - Bulk Notion Objet Link Index

Date: 2026-06-17

## Added

- CLI `archive notion-objet-link-index <archive-root> --dry-run`.
- MCP `notion_objet_link_index`.
- Public docs for archive-wide Notion provider locator to manifested objet link
  candidate indexing.

## Why

Phase 3 real-use feedback showed that one-zettel locator planning is helpful,
but a large Notion import also needs an archive-wide map before any reviewed
body rewrite or durable `embed` edge write.

This release adds that map without turning it into an automatic converter.

## Behavior

- Scans non-redacted zets under `zettels/` and `inbox/` for Notion provider
  locator fingerprints.
- Groups provider locators by opaque `sha256:` fingerprints.
- Matches locator fingerprints against reviewed metadata in
  `objects/manifests/files.jsonl`.
- Reports archive-wide counts for zets with locators, locator rows, distinct
  locators, and locator rows with or without manifested objet candidates.
- Returns safe zettel summaries plus safe object ids and suggested
  `objet:sha256:<hex>` refs for later human review.

## Safety

The new path writes nothing, rewrites no zettel body, writes no edges, calls no
providers, starts no OAuth, creates no provider or presigned URLs, reads no
object bytes, and echoes no zettel body text, frontmatter values, provider URLs,
page titles, absolute local paths, account ids, emails, tokens, or secret
values.

Redacted zettels are counted but their body/frontmatter locator content is not
scanned or exposed.

Approval-gated Notion locator rewrite remains future work.
