# v0.2.46 Release Notes

Date: 2026-05-26

## Summary

v0.2.46 adds a read-only ZET projection plan dry-run preview.

This lets a human review which local zet and which operator-declared surface kind are being considered before any render, write, provider call, publication, receipt, minting, trust, attestation, signature, or ZET transport exists.

## Added

- `archive projection-plan <archive-root> --zet <zet-id-or-path> --surface <surface-kind> --dry-run --format json`
- Read-only MCP `zet_projection_plan_check`
- Safe metadata-only projection plan output with closed mutation/provider flags
- Documentation for ZET projection plan previews

## Compatibility

No private archive migration is required.

The new command is dry-run only. It writes nothing and does not change existing mint, draft, foreign block, source intake, block header, or provider setup behavior.

## Safety

The preview:

- reads one local archive zet only enough to confirm existence and extract safe metadata,
- does not output the full zet body,
- uses archive-relative paths only,
- returns `would_change: []`,
- keeps provider, WordPress, projection-write, receipt-write, trust, import, acceptance, attestation, signature, mint, ZET transport, and full-auto flags false.

v0.2.46 does not introduce Redis, queues, background workers, payments, staking, consensus, blockchain, WOM coin, token mechanics, model training, or backpropagation.
