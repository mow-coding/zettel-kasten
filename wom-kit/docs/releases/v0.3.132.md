# v0.3.132 - Notion Media Byte Fetch Contract

Release date: 2026-06-22
Status: prerelease

v0.3.132 closes the next provider-fetch planning gap reported from live WOM
use: nested Notion leaf pages may contain image/file media bytes that cannot be
proven preserved from local URL markers or object ledgers alone.

This release does not implement live Notion media transport. It adds the
read-only adapter contract and local result verifier needed before a future
credential-bounded adapter can safely fetch, hash, preserve, and report those
bytes.

## Added

```powershell
archive notion-media-fetch-adapter-execution-contract <archive-root> `
  --tree workbench/notion-nested-tree.sample.json `
  --source notion `
  --scope-leaf-ref page:fake:db2-nested-live-log `
  --credential-ref env:wom_notion_readonly `
  --dry-run `
  --format json
```

MCP:

```text
notion_media_fetch_adapter_execution_contract
```

The contract:

- reuses `notion-nested-tree-plan` to scope candidate live content leaf pages,
- states that media block counts and already-preserved counts are unknown until
  future live page/block fetch plus byte hashing,
- requires fresh provider file refs before download,
- requires sha256 objet preservation before preservation claims,
- defines sanitized `notion_media_result_fixture` output,
- keeps the v0.3.131 actor boundary: WOM local credential-bounded adapter,
  human credential approval, AI plan/review/verify only.

Also added:

```powershell
archive notion-media-result-verification-plan <archive-root> `
  --media-result workbench/notion-media-result.sample.json `
  --source notion `
  --dry-run `
  --format json
```

MCP:

```text
notion_media_result_verification_plan
```

The verifier checks sanitized media result fixtures against
`objects/manifests/files.jsonl` for:

- `object_id` / `sha256` consistency,
- manifest presence,
- preservation states: `already_preserved`, `newly_preserved`, `fetch_failed`.

## Still Closed

This release still does not call providers, start OAuth, open Notion
connections, retrieve credential values, refresh signed URLs, download media
bytes, hash media bytes, read object bytes, update object manifests, write
receipts, write zets, mint pages, write edges, or echo provider URLs, local
paths, account ids, emails, tokens, secret values, or media bytes.
