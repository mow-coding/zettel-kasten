# v0.3.60 - Credential Semantic Extraction Recipe

Released: 2026-06-16

This release adds a read-only semantic recipe for splitting complex credential
notes before plaintext migration planning.

## Added

- `archive credential-semantic-extraction-recipe <archive-root>
  --source-label <safe-label> --dry-run --format json`.
- Aliases `archive credential-extraction-recipe` and
  `archive secret-semantic-extraction-recipe`.
- Entry classes for multi-account, multi-secret, mail, API/CLI token,
  SSO/passkey, recovery-code, wallet/private-key, and status/toggle notes.
- Human review questions and classification rules that keep WOM refs safe and
  prevent AI from treating a messy note as one password.

## Safety Boundary

The command reads no plaintext files, detects no secret values, opens no
password managers, keyrings, browser password stores, or environment variables,
calls no providers, writes no files, and returns no secret values to AI.

## Still Future Work

This release does not implement automatic candidate extraction, secret
scanning, vault writes, browser password imports, wallet seed migration, or live
adapter execution.
