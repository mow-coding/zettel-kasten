# WOM-kit v0.3.21

Date: 2026-06-14

v0.3.21 adds a read-only credential ref inventory and onboarding guide.

## Added

- CLI `archive credential-ref-inventory --dry-run`.
- CLI aliases `archive credentials --dry-run` and
  `archive credential-status --dry-run`.
- MCP `credential_ref_inventory`.
- `wom-kit/docs/credential-ref-inventory-and-onboarding.md`.

## Behavior

The inventory scans:

- `provider-bindings.yml`,
- `source-bindings.yml`,
- ignored local `profiles/local/credential-refs.local.yml`.

It reports safe metadata such as credential id, credential kind, provider,
purpose, and ref prefix/store type.

It does not print exact credential ref values, passwords, OAuth tokens, API
keys, email addresses, local absolute paths, provider URLs, or environment
variable names.

## IMAP Account Ref Clarification

`account_ref` is a non-secret account label such as:

```text
imap:account:personal-mail
```

Credential-store refs such as `keyring:`, `env:`, `secret:`, and `wallet:`
belong in username, app-password, OAuth-token, or generic credential fields.
If one is passed as `account_ref`, the IMAP mailbox planner now blocks with a
more specific explanation and does not echo the mistaken value.

## Boundaries

This release does not store passwords, read passwords, write to Windows
Credential Manager, write to macOS Keychain, write to Linux keyrings, open
KeePassXC, read environment variables, start OAuth, call mail providers, call
OpenAI, call OCR providers, call object-storage providers, upload files,
download files, draft zets, or mint zets.

The actual secret value should live in a password manager, OS credential store,
or external secret manager. WOM stores and displays only safe references and
catalog metadata.
