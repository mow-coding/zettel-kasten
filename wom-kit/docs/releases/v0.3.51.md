# v0.3.51 - IMAP Mailbox Adapter Manifest Plan

Date: 2026-06-16

v0.3.51 adds a read-only preview for the non-secret manifest a future IMAP
mailbox adapter should declare before any live adapter exists.

The release still does not connect to mail providers. It only defines the safe
adapter declaration shape.

## Added

- `archive imap-mailbox-adapter-manifest-plan --dry-run`.
- Alias `archive imap-mailbox-adapter-manifest --dry-run`.
- Alias `archive mailbox-adapter-manifest-plan --dry-run`.
- MCP tool `imap_mailbox_adapter_manifest_plan`.
- New documentation at `wom-kit/docs/imap-mailbox-adapter-manifest-plan.md`.
- Capability matrix and public documentation map entries for the IMAP adapter
  manifest preview boundary.

## Verify

```bash
archive imap-mailbox-adapter-manifest-plan <archive-root> \
  --adapter-id local-imap \
  --provider gmail \
  --provider naver \
  --operation header_metadata_scan \
  --selection-rule newest_first \
  --selection-rule unread_first \
  --dry-run \
  --format json
```

The JSON should include `lifecycle_action: imap_mailbox_adapter_manifest_plan`
and a proposed path under `config/imap-adapters/`.

## Safety

The planner remains read-only. It:

- writes no adapter manifests,
- executes no live adapters,
- opens no IMAP connection,
- attempts no login,
- selects no mailbox,
- searches no mailbox,
- lists no candidate messages,
- reads no IMAP UIDs or Message-ID values,
- reads no headers, bodies, or attachments,
- creates no derived text,
- retrieves no secrets,
- starts no OAuth,
- calls no providers,
- echoes no email addresses, username values, exact account refs, exact
  credential refs, exact mailbox refs, IMAP host values, provider URLs, message
  ids, subjects, sender or recipient values, attachment names, approval receipt
  paths, selection receipt paths, local absolute paths, tokens, or secret values.

No archive migration is required.
