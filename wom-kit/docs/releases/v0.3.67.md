# v0.3.67 - IMAP Material Selection Plan

Status: v0.3.67 read-only IMAP material selection planning checkpoint
Date: 2026-06-16

v0.3.67 adds a read-only planning checkpoint between the first IMAP header
metadata scan receipt and any future message body, attachment, or mail-derived
text capture.

## Added

- `archive imap-mailbox-material-selection-plan <archive-root>
  --execution-receipt <archive-relative-json> --selection-mode
  human_review_queue|body_candidates|attachment_candidates|derived_text_candidates
  --dry-run --format json`.
- Aliases:
  - `archive imap-material-selection-plan`
  - `archive mailbox-material-selection-plan`
- A public documentation page:
  - [IMAP Mailbox Material Selection Plan](https://github.com/mow-coding/zettel-kasten/blob/main/wom-kit/docs/imap-mailbox-material-selection-plan.md)

## Behavior

The command reads one existing non-secret IMAP header metadata scan execution
receipt from `receipts/imap/adapter-executions/`, validates that it is a
redacted `imap_mailbox_header_metadata_scan` receipt, and reports the candidate
pool count for a future human review lane.

The selected lane may be:

- `human_review_queue`
- `body_candidates`
- `attachment_candidates`
- `derived_text_candidates`

## Safety Boundary

This release does not implement broad mail ingestion.

The command opens no IMAP connection, reads no environment variables, opens no
keyring/password manager, reads no message headers again, reads no message
bodies, reads no attachments, creates no derived text, starts no OAuth, calls no
providers, writes no queue files, and echoes no execution receipt path,
candidate refs, subjects, senders, recipients, local paths, tokens, or secret
values.

## Verification

- Capability matrix and public documentation tests cover the new command,
