# v0.3.77 - Connection Import Plan

Date: 2026-06-16

v0.3.77 starts the #10 connection-tool feedback path without jumping straight to
live Notion sync or canonical zet mutation.

## Added

- CLI `archive connection-import-plan --source notion --dry-run`.
- MCP `connection_import_plan`.
- A Notion connection evidence map for:
  - relation/pre/post properties,
  - synced block references,
  - database view/filter snapshots,
  - internal Notion hyperlinks,
  - page mentions,
  - comment context,
  - objet embeds.
- Recommended edge vocabulary:
  - `material`
  - `derived`
  - `semantic`
  - `embed`
  - `mention`
  - `view_query`
  - `comment_context`
- Archive link-type status reporting so the planner can say which recommended
  edge types are missing from `zettel-kasten/types.yml` before any write command
  exists.

## Safety

- The command is dry-run/read-only.
- It calls no providers, starts no OAuth, opens no Notion connection, reads no
  export files, reads no comments, downloads no media, writes no zets, writes no
  edges, writes no receipts, updates no object manifests, and echoes no provider
  URLs, local paths, page titles, comment bodies, account ids, emails, tokens, or
  secret values.
- It does not implement a connection importer, typed-edge write command, Notion
  API sync, database view API automation, comment import, or canonical zet
  mutation.

## Verification

- Added CLI tests for the Notion evidence map, edge vocabulary, dynamic snapshot
  standard, missing allowed-link-type reporting, and read-only/privacy guards.
- Added MCP tests for `connection_import_plan`, allowed-root enforcement, and
  dry-run enforcement.
- Updated README, WOM-kit README, capability matrix, public documentation map,
  changelog, release notes, and version metadata.
