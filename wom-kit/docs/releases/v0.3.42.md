# v0.3.42 - Connected Accounts Catalog Status

Date: 2026-06-15

v0.3.42 clarifies `connected-accounts` status handling.

## Changed

- `archive connected-accounts <archive-root> --dry-run` now separates the
  account map status from the ignored local credential catalog status.
- A malformed `profiles/local/credential-refs.local.yml` row now appears under:

```text
credential_catalog.ok
credential_catalog.status
credential_catalog.blockers
```

- The top-level `ok` still reports whether safe provider/source account
  metadata can be listed.
- Unsafe account labels, archive-id mismatches, and unreadable provider/source
  binding files still remain top-level blockers.

## Safety

The command remains read-only. It:

- writes no files,
- opens no password manager, keyring, or browser password store,
- reads no environment variables,
- calls no providers,
- opens no IMAP connection,
- reads no mail or source bytes,
- echoes no exact credential refs,
- echoes no emails, usernames, tokens, provider URLs, local paths, or secret
  values.

No archive migration is required.
