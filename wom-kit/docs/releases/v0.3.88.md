# v0.3.88 - Overview-First And Phase 3 Unblock

Status: v0.3.88 overview-first and Phase 3 unblock checkpoint
Date: 2026-06-17

v0.3.88 responds to the second Phase 3 real-use sweep. It opens a cheap
first-read surface for AI navigation and removes several blockers found while
trying to write reviewed connection edges.

## Added

- `archive read-zettel --section overview`
  - returns a first-read card with `gist`, `facets`, `tie_summary`, and
    `edges_preview`,
  - omits the full body and full frontmatter details,
  - skips unsafe first-read values such as provider URLs instead of echoing
    them.
- `archive block-header --dry-run` now returns top-level `first_read`.
  - This includes gist, facets, tie counts, and safe edge preview.
  - It stays outside `header_preview`, so `header_sha256` and
    `block_hash_preview` keep the existing sanitized header boundary.
- `archive migrate --target link-types-v0.3 --dry-run|--approve`
  - appends missing recommended connection edge types from the base
    `zettel-kasten/types.yml`,
  - helps stale archives gain `material`, `derived`, `semantic`, `embed`,
    `mention`, `view_query`, and `comment_context` without hand-editing.
- `archive zettel-edge` can resolve safe local external refs such as
  `zet:notion:ZET637` to an existing archive-local id such as
  `zet_notion_db3_ZET0637` when exactly one match exists.

## Hardened

- `mint-zet` and draft safety checks now treat plain `https://` provider URLs
  as provider locators.
- The machine-enforced `object_id_only` and `allowed_edges` checks are rerun at
  mint time even when legacy frontmatter checklist values claimed those items
  had already passed.

## Still Closed

This release does not implement:

- live Notion, IMAP, object-storage, or provider API calls,
- OAuth,
- real export parsing,
- bulk edge candidate writes,
- bulk `zettel-edge` application,
- Notion embed or mention URL to `sha256` objet conversion,
- object manifest updates,
- source/object byte reads,
- LLM relationship classification.

The next natural Phase 3 batch is a reviewed Notion objet-link conversion plan:
detect imported Notion embed/mention references, match them to already
manifested objets, and propose safe `sha256`/`objet:sha256` edges without
echoing provider URLs.
