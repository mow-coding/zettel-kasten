# v0.3.13 - Human Artifact Store Contract

Release date: 2026-06-14

v0.3.13 is a compatible documentation and regression-test checkpoint. It turns
the existing human artifact store note into the shared contract for future
WordPress, Joplin, Notion, Obsidian, and Markdown/workspace adapters.

## Changed

- `wom-kit/docs/human-artifact-store-contract.md` now defines a role matrix for
  raw data stores, human artifact stores, projection surfaces, and system/AI
  artifact records.
- The contract now lists adapter questions future app-specific integrations must
  answer before writing to a user-facing app.
- The contract now documents a capture-action shape for note/report/handoff
  workflows: explicit human capture, human-facing artifact write, and separate
  WOM system records.
- `wom-kit/docs/zet-surface-prototypes.md` now links back to the shared
  contract and names the common questions every future surface adapter must
  answer.
- README and the public documentation maps now link the human artifact store
  contract directly.

## Boundaries

- This release does not implement a Joplin connector, WordPress publisher,
  Notion adapter, Obsidian vault writer, provider sync, OAuth, note creation,
  post publishing, attachment upload, projection receipt write, minting,
  cleanup, or ZET transport.
- The existing `archive human-artifact-store --dry-run` and
  `archive zet-surface-prototype --dry-run` commands remain read-only planning
  surfaces.
- The local pilot planning files used as input were not published verbatim.

## Upgrade

No archive migration is required for v0.3.12 users.
