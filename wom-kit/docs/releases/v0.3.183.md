# v0.3.183 - adopt resume-gap diagnostics and doctor frontmatter progress

v0.3.183 responds to the basoon v0.3.182 revalidation. That run showed that
`--skip-existing-wom-uploaded` works, but the archive's partial state had many matching provider
locations and only a small number of verified `wom_uploaded` rows. The result was correct but
surprising: most rows still needed remote HEADs.

This release is additive. Defaults remain conservative.

## Adopt resume-gap diagnostics

`object-storage-adopt-existing --progress` now emits a content-free plan-time resume summary before
the verified remote-HEAD loop:

```text
[object-storage-adopt] adopt-plan: resume summary matching_locations=<n> wom_uploaded=<n> declared_uploaded=<n> other=<n> skip_existing_wom_uploaded=on
```

The adopt JSON summary now includes:

- `existing_matching_location_count`
- `existing_wom_uploaded_count`
- `existing_declared_uploaded_count`
- `existing_other_location_count`
- `expected_resume_skip_count`
- `skipped_existing_wom_uploaded_count`

This lets an operator see, before waiting through the HEAD loop, whether the current manifest can
actually benefit from `--skip-existing-wom-uploaded`.

## Declared-upload gap warning

Matching `declared_uploaded` rows now emit `declared_upload_resume_gap`. Those rows prove that WOM
has a declared provider/store/key location, but not a WOM-verified provider confirmation. They
remain non-gating and cannot be skipped by `--skip-existing-wom-uploaded`.

This is deliberate: using `declared_uploaded` as a remote-HEAD skip would weaken the verified-adopt
contract. The safe next step is to keep running verified adopt so those rows are promoted to
`wom_uploaded` after a successful presence+size HEAD.

## Deeper doctor target-frontmatter progress

`archive doctor --strict --progress` now breaks `mint-receipts` target frontmatter loading into
finer sub-steps:

- target frontmatter cache hit;
- target frontmatter reading text;
- target frontmatter checking BOM;
- target frontmatter parsing fence;
- target frontmatter loading YAML;
- target frontmatter loaded;
- checking target BOM cache;
- checking target mint receipt link.

This keeps diagnostics unchanged but gives a better last-known internal step when a large archive
spends time inside the first `mint-receipts` row.

## Tests

Added focused regression coverage that:

- declared-upload-only partial state is counted separately from `wom_uploaded`;
- the plan-time resume summary is emitted before the HEAD loop;
- `declared_upload_resume_gap` warns without weakening the skip contract;
- doctor frontmatter progress includes internal read/YAML sub-steps.

## Compatibility

Additive; **no migration required**. Existing verified-adopt behavior still re-HEADs by default.
Only verified `wom_uploaded` rows can be skipped with `--skip-existing-wom-uploaded`.
