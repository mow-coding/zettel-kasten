# v0.3.234 - Canonical zet Revision Plan

v0.3.234 adds the read-only first half of WOM's ordinary canonical correction
workflow. One complete private proposal can now be checked against the exact
current canonical zet before either file is changed.

## What Changed

- Added CLI `zet-revision-plan` with aliases `revise-zet-plan` and
  `canonical-revision-plan`.
- Added MCP `zet_revision_plan` with the same no-write contract.
- Confined proposals to `.wom-scratch/revisions/*.md`, rejected symbolic-link
  components, and bounded both canonical and proposal reads to 16 MiB.
- Bound canonical bytes, proposal bytes, normalized proposal semantics,
  change categories, and pending abstract/body review evidence into one plan
  digest.
- Froze system-managed identity, creation, lifecycle, and original creator
  metadata while allowing reviewed knowledge fields to change.
- Reused explicit-abstract, private-locator, allowed-edge, provenance,
  visibility, note-kind, quality, and self-containment checks.
- Added content-free change categories and warnings for body changes that
  reuse an abstract or proposals that also change edges.

## Upgrade And Compatibility

No archive migration is required. Existing canonical zets and receipts remain
unchanged. Prepare a private full-zet proposal, then run:

```powershell
archive zet-revision-plan <archive-root> --zettel-id <safe-id> --proposal .wom-scratch/revisions/<private>.md --dry-run --format json
```

## Safety And Claim Boundary

- The command writes no zet, receipt, objet, map, index, or database row.
- It returns hashes and fixed change categories but no actual zet id, path,
  filename, title, abstract, body, custom frontmatter value, reviewer id,
  provider URL, absolute path, or secret value.
- It calls no model, provider, credential store, object store, or network.
- `ready_for_human_review` is not truth, approval, application, model
  understanding, or permission to edit the canonical file manually.
- The approval-gated atomic writer and immutable revision receipt remain the
  next release rung.
