# v0.3.241 - Selective Freshness Body Reads

v0.3.241 follows a successful real-archive v0.3.240 validation with one more
bounded performance improvement. Abstract freshness no longer opens complete
body bytes for zets that do not have a valid explicit abstract.

## What Changed

- Every canonical zet receives a bounded frontmatter read.
- Archives with at least 64 canonical paths use at most eight frontmatter
  workers; smaller archives remain sequential.
- Complete zet bytes are opened only for valid explicit-abstract targets whose
  current body hash is required.
- Selected targets are reparsed from those complete bytes before their id,
  abstract, or body hash is trusted.
- `scan` now reports frontmatter files scanned, body files read and not read,
  body-selection policy, scan mode, and worker count.

## Upgrade And Compatibility

No archive migration is required. Restart the AI operator process.

The abstract-freshness schema and state names remain unchanged. The new scan
fields are additive. Evidence lookup, progress stages, exit behavior, and
attention rows remain compatible with v0.3.240.

## Safety And Claim Boundary

- The command remains read-only and creates no receipt, cache, provider call,
  model call, canonical change, or external-database change.
- A selected explicit-abstract target still receives a complete bounded read
  and exact abstract/body hash calculation.
- No title, abstract/body text, hash, reviewer id, provider value, secret, or
  absolute local path is added to output.
- A body skipped because no valid explicit abstract exists is not claimed
  readable, valid, true, complete, or reviewed.

## Verification

- Focused tests prove missing-abstract rows trigger zero complete body reads,
  explicit-abstract rows still trigger exact body reads, unrelated receipts stay
  unopened, and the bounded parallel path reports its worker contract.
- Final repository suite: `1381 passed, 11 skipped`.
- Public privacy and release-readiness publication gates passed before the tag
  and GitHub release were created.
