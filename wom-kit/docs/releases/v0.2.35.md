# v0.2.35 Foreign Block Quarantine Decision Write

Release date: 2026-05-25

## Summary

v0.2.35 adds the smallest approved write layer after the v0.2.34 quarantine decision preview.

The new `record-quarantine-decision` command records a local operator-reviewed quarantine decision and a matching receipt, while keeping the foreign block untrusted and unimported.

## Added

- `archive record-quarantine-decision <archive-root> --decision-preview <json-file> --dry-run --format json`
- `archive record-quarantine-decision <archive-root> --decision-preview <json-file> --approve --reviewed-by <actor-id> --format json`
- Optional replay guards:
  - `--expected-case-id`
  - `--expected-decision`
  - `--review-note`
- Read-only MCP `record_quarantine_decision_check`.
- New documentation: `wom-kit/docs/foreign-block-quarantine-decision-write.md`.

## Write Boundary

Approved mode writes exactly two files:

```text
quarantine/foreign-blocks/<case-id>/quarantine-decision.json
receipts/quarantine/<case-id>.foreign-block-quarantine-decision.json
```

Both files are archive-relative. The command refuses overwrites. If receipt writing fails after the decision file is created, the partial decision file is rolled back.

## Safety

The command validates the saved decision preview as untrusted input and re-reads the current quarantine case plus matching quarantine write receipt before writing.

It blocks when:

- the preview is not a successful dry-run decision preview,
- the preview has blockers,
- the expected case id or expected decision does not match,
- the current case or receipt is missing, malformed, contradictory, unsafe, stale, or already has a decision record,
- `--approve` is used without a safe `--reviewed-by`.

Review notes are not stored as raw body text. Only safe summary metadata is recorded.

## Non-Goals

v0.2.35 does not implement:

- quarantine decision acceptance,
- quarantine decision trust or apply,
- foreign block import,
- foreign block trust,
- foreign attestation writes,
- minting from foreign blocks,
- anchoring,
- delegation,
- real ZET transport,
- signing,
- payment,
- staking,
- consensus,
- blockchain,
- provider sync,
- OCR,
- LLM classification,
- full-auto execution.

MCP remains read-only for this workflow and exposes no quarantine decision write/apply/accept/import/trust/attest tool.
