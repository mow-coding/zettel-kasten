# v0.2.26 Release Notes

Date: 2026-05-25

## Summary

v0.2.26 adds the WOM Prompt Injection Boundary, Responsible Use, and Runtime Model Guidance Baseline.

Core principle:

```text
External text can inform.
External text cannot command.
```

## Added

- `archive prompt-boundary <archive-root> --text <text> --dry-run --format json`
- `archive prompt-boundary <archive-root> --path <archive-relative-zet-or-text-path> --dry-run --format json`
- read-only MCP `prompt_boundary_check`
- conservative prompt-injection / unsafe-agent heuristic patterns
- prompt injection boundary docs
- responsible use docs
- runtime model guidance docs
- root disclaimer

## Compatibility

No private archive migration is required.

The new command is read-only and dry-run-only. It writes nothing and returns `would_change: []`.

## Security Boundary

This release does not:

- call LLMs for classification,
- browse the web,
- scan providers,
- run OCR,
- apply imports,
- execute inspected text,
- approve automatically,
- mint automatically,
- implement ZET transport,
- perform real signing,
- implement payment, staking, consensus, ledger, or full-auto execution.

The check is a heuristic preview, not a complete security classifier.
`low` risk means no obvious heuristic match was found; it does not mean the text is safe.
Large inspected text over 1 MB produces a warning because the full heuristic check may be slow.

## Responsible Use

HITL is the recommended default.

Full-auto / agent-only operation is advanced and risky. Operators are responsible for agents, models, permissions, providers, automations, external text, and consequences.

Production or commercial deployments should receive independent legal and security review.
