# v0.3.198 - Reconcile approval-result status cleanup

v0.3.198 follows the v0.3.197 dry-run guidance release by fixing the matching
post-approval result surface.

No archive migration is required.

## What Changed

Before this release, a reconcile approval could succeed with:

```json
{
  "ok": true,
  "writes": "applied",
  "status": "needs_content_change_review"
}
```

That was technically explainable because the approval result started from the
dry-run plan, but it was confusing for an operator: the write had already been
applied, yet the status still sounded like the pre-approval review step.

Now both approval surfaces report:

```json
{
  "ok": true,
  "writes": "applied",
  "status": "reconcile_applied",
  "overall_status": "reconcile_applied",
  "suggested_next_action": "run_doctor_to_verify_reconcile",
  "would_write": false,
  "approval_would_write": false,
  "approval_requires_content_changed_ack": false
}
```

Affected commands:

```text
archive remint-reconcile <archive-root> --approve --reviewed-by <actor> --format json
archive retire-draft-reconcile <archive-root> --approve --reviewed-by <actor> --format json
```

The result also carries `next_safe_actions` that route the operator back to a
doctor verification run and say to restart from dry-run if fresh drift appears.

## Safety

- Reconcile classification is unchanged.
- Dry-run guidance from v0.3.197 is unchanged.
- `content_change` still requires `--content-changed-ack` before approval.
- The approval result still preserves `content_change_ack: true` when the
  reviewer explicitly acknowledged an intentional content change.
- No zettel content is edited by this status cleanup.

## Field Notes

Two field-observed workflow notes are worth carrying forward:

- On Windows PowerShell, prefer explicit UTF-8 capture when saving JSON examples
  for later parsing. Bare `>` redirection can produce UTF-16 output in some
  shells.
- If an operator directly repairs a schema enum in zettel frontmatter and that
  zettel also has retired-draft receipts, run the paired reconcile flow: first
  `remint-reconcile`, then `retire-draft-reconcile`, each dry-run first and only
  then approved after review. The schema repair can clear one diagnostic while
  legitimately changing the receipt hashes that the retired-draft receipt binds.
