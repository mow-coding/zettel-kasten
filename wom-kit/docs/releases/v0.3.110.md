# v0.3.110 - Mint Draft Retirement

Date: 2026-06-18

## What Changed

This release closes the post-mint inbox draft lifecycle gap.

New command:

- `archive retire-draft --dry-run`
- `archive retire-draft --approve --reviewed-by <actor>`

The command verifies that an inbox draft is already backed by:

- the canonical minted zet,
- the original mint receipt,
- the draft snapshot,
- matching archive-relative paths,
- matching SHA-256 evidence.

Approved retirement removes only the verified inbox draft and writes a receipt
under `receipts/mint/retired-drafts/`.

## Validation

`archive doctor` and `archive validate` now understand two post-mint states:

- an inbox draft that is still present but fully backed by mint artifacts is an
  informational cleanup candidate, not a mint metadata failure;
- a mint receipt whose source draft was retired remains valid when the
  retired-draft receipt proves the closure.

## Title And Duplicate Friction

The mint checklist title gate is now CJK-width aware. Short meaningful Korean,
Japanese, or Chinese titles can pass when they are not generic placeholders.

Near-duplicate checks no longer warn on title alone when the body is materially
different. `same_title` remains a warning when the body start is also very
similar.

## Safety Boundary

This release does not:

- call providers,
- read source exports,
- read object bytes,
- delete canonical zets,
- delete mint receipts,
- delete draft snapshots,
- clean unverified inbox drafts,
- infer duplicate meaning from title alone.
