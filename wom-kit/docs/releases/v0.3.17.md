# WOM-kit v0.3.17

Date: 2026-06-14

v0.3.17 adds the first read-only objet reference resolver for the reading-side
UX.

## Added

- CLI `archive resolve-objet-ref --object-id sha256:<hex> --dry-run`.
- MCP `resolve_objet_ref`.
- `wom-kit/docs/objet-ref-resolution.md`.
- Tests for local candidates, external candidates, dry-run-only behavior,
  allowed-root behavior, and privacy boundaries.

## Behavior

The resolver reads `objects/manifests/files.jsonl` for one `object_id` and
returns:

- local archive-relative candidates when local content-addressed bytes exist,
- external provider/store labels when the manifest declares an external store,
- record summaries without echoing full manifest provenance bodies.

## Boundaries

This release does not open files, echo absolute local paths, echo provider URLs,
read object bytes, re-hash object bytes, call providers, create presigned URLs,
download objects, upload objects, prove remote availability, decide deletion
safety, draft, mint, clean, or run ZET transport.
