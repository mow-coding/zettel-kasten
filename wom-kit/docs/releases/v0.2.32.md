# v0.2.32 Foreign Block Quarantine Write Approval

Release date: 2026-05-25

v0.2.32 adds the first approved local quarantine write for foreign blocks.

This is still not import, not trust, not attestation, and not ZET transport. Quarantine is an isolation record only.

## Added

- `archive quarantine-foreign-block <archive-root> --plan <json-file> --dry-run --format json`
- `archive quarantine-foreign-block <archive-root> --plan <json-file> --approve --reviewed-by <actor-id> --format json`
- optional `--expected-case-id` and `--review-note` safety inputs
- approved writes under `quarantine/foreign-blocks/<case-id>/quarantine-case.json`
- quarantine write receipts under `receipts/quarantine/<case-id>.foreign-block-quarantine.json`
- read-only MCP `quarantine_foreign_block_check`

## Safety Boundary

The approved CLI write:

- consumes a successful v0.2.31 `foreign_block_quarantine_plan`,
- requires explicit `--approve --reviewed-by`,
- writes only a sanitized quarantine case JSON and quarantine write receipt JSON,
- keeps `trust_state: untrusted_foreign`,
- refuses blocked or hold/manual-review plans,
- refuses unsafe local paths, provider URLs, token-like values, private keys, seed phrases, and secret-like strings,
- refuses to overwrite an existing quarantine case.

The approved CLI write does not:

- copy the original foreign artifact,
- store raw foreign body text,
- import the foreign block,
- mark the foreign block trusted,
- create an attestation,
- mint, anchor, delegate, sign, or execute anything,
- call provider APIs.

## MCP

MCP exposes only `quarantine_foreign_block_check` for this workflow. It is read-only and dry-run only.

MCP does not expose quarantine write/apply, import, trust, attestation, mint, anchor, delegate, signing, provider sync, ZET transport, or full-auto tools.

## Compatibility

No private archive migration is required.

Existing foreign-block intake, trust preview, attestation packet preview, and quarantine plan outputs remain compatible. The new write command is an explicit CLI-only approval step.
