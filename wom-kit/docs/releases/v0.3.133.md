# v0.3.133 - Notion Recursive Live Fetch Contract

Release date: 2026-06-22
Status: prerelease

v0.3.133 answers the remaining read-only contract question from nested Notion
recovery testing: a future live ancestor fetch adapter must be recursive.

This release still does not implement live Notion transport. It tightens the
adapter contract so the next execution step is unambiguous before credentials
or provider APIs are allowed into the loop.

## Changed

`archive notion-ancestor-fetch-adapter-execution-contract --dry-run` now
declares a `recursive_fetch_contract`:

```text
required_for_live_adapter: true
direction: parent_chain_upward
starts_from: crawl_request_queue.ancestor_ref
must_fetch_newly_discovered_missing_parent_refs: true
must_not_claim_complete_when_stopped_before_generation_root: true
```

The future adapter must continue fetching parent refs until one of these stop
conditions applies:

```text
known_generation_root_ref_reached
space_or_workspace_root_reached
max_depth_reached
parent_ref_missing_or_ambiguous
unsafe_ref_or_provider_secret_detected
```

After a sanitized result fixture is merged, `notion-ancestor-crawl-plan` should
run again. If reviewed scope still contains crawl requests, the next adapter
pass repeats from those remaining seeds.

## Scope Guidance

`archive notion-ancestor-crawl-plan --dry-run` now warns when a generation-id
scope filter matches no crawl requests while unfiltered requests exist:

```text
scope_generation_id_may_not_match_generation_unknown_untraceable_leaf_requests
```

For generation-unknown untraceable leaves, use:

```text
--scope-leaf-ref
--scope-root-ref
--scope-ancestor-ref
```

`--scope-generation-id` only matches requests that already carry affected
generation ids.

## Still Closed

This release still does not call providers, start OAuth, open Notion
connections, retrieve credential values, read page titles, read page bodies,
read comments, download media, write fixtures, write receipts, write zets,
mint pages, write edges, update object manifests, or echo provider URLs, local
paths, account ids, emails, tokens, or secret values.
