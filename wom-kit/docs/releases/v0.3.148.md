# v0.3.148 - Agent Operator Capabilities

v0.3.148 adds a machine-readable capabilities manifest for AI operators that
need to know what the current local WOM-kit installation can actually run.

## What Changed

- Added read-only `archive capabilities --machine`.
- The manifest is generated from the actual local CLI parser.
- It reports:
  - command names,
  - aliases,
  - help text,
  - required positionals,
  - options,
  - nested subcommands,
  - command count,
  - current version,
  - release notes presence,
  - local git tag presence,
  - local release state.
- The command uses a stable agent-facing envelope:
  `ok / state / summary / data / blockers / warnings / privacy_guards`.

## Command Shape

```powershell
archive capabilities --machine
```

```powershell
archive capabilities --machine --no-commands
```

## Safety Boundary

The command:

- writes nothing,
- calls no providers,
- checks no network,
- opens no archive content,
- echoes no local absolute paths,
- echoes no tokens or secret values.

## Still Future

- Applying the same response envelope across every CLI command.
- Public remote release verification inside a supervisor workflow.
- Per-command write/read/body/provider risk metadata.
- Agent-facing handoff manifests for human approval steps.
