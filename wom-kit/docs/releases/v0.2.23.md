# v0.2.23 Source Intake Draft Composer

Date: 2026-05-25

## Summary

v0.2.23 connects the v0.2.22 source intake planner to draft creation.

`create-draft` can now consume a saved `source-intake --dry-run --format json` result and merge safe source refs into the draft preview or approved inbox draft write.

## Added

- CLI `archive create-draft --source-intake-plan <json-file>`.
- MCP `create_draft_zettel` support for a structured `source_intake_plan` object.
- Source intake plan validation before draft composition.
- Optional draft frontmatter `source_intake` metadata:
  - `plan_sha256`,
  - `input_kind`,
  - `source_kind`,
  - `objet_status`,
  - `object_storage_configured`,
  - metadata-only `content_access` proof.
- Mint receipt previews and real mint receipts preserve `source_intake` metadata when present.
- Candidate refs from local-file intake plans are anonymized during draft composition so a no-redact upstream plan cannot persist a private filename stem.

## Validation

A source intake plan is accepted only when:

- `ok` is true,
- `dry_run` is true,
- `lifecycle_action` is `source_intake_plan`,
- `blockers` is an empty list,
- `source_refs_for_draft` is a list of safe refs,
- `content_access` proves metadata-only behavior.

Refs that contain local absolute paths, provider URLs, token-like strings, or secret-like values are blocked.

## Boundaries

This release does not:

- apply source intake,
- capture or manifest objets,
- copy, upload, import, OCR, transcribe, or parse source content,
- calculate full source hashes,
- call provider APIs,
- automatically mint,
- add real mint/apply tools to MCP.

The source intake plan file path is not stored in draft frontmatter.

`source_intake.plan_sha256` commits to the supplied dry-run JSON plan object. It is not an independent re-run of source intake and does not verify live source existence.

## Upgrade Notes

No private archive migration is required.

Existing `create-draft` behavior remains compatible when `--source-intake-plan` is omitted. Existing zets remain valid because `source_intake` is optional frontmatter.
