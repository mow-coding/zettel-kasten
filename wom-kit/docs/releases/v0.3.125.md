# v0.3.125 - Notion Ancestor Crawl Request Plan

Release date: 2026-06-22
Status: prerelease

v0.3.125 closes the next safe step after the v0.3.124 nested tree planner.
The nested tree planner already reports `untraceable` nested Notion leaves; this release
packages those missing ancestors into a read-only crawl request queue and
hardens the fixture contract so large or contradictory inputs do not look
successful by accident.

## Added

```powershell
archive notion-ancestor-crawl-plan <archive-root> `
  --tree workbench/notion-nested-tree.sample.json `
  --source notion `
  --dry-run `
  --format json
```

MCP:

```text
notion_ancestor_crawl_plan
```

The command reuses `notion_nested_tree_plan`, reads the hold queue, groups
`missing_parent_record` / `no_known_generation_root` leaves, and returns a
`crawl_request_queue` with:

- `ancestor_ref`,
- `affected_leaf_refs`,
- `known_child_refs`,
- `suggested_direction`,
- `stop_conditions`,
- `required_return_fields`,
- `merge_target: sanitized_nested_tree_fixture`,
- `write_status: not_written`.

## Hardened

- `content_class` may be omitted and derived from `node_kind` without reading
  page titles or bodies.
- Structural node kinds declared as `content` route to human review instead of
  recovery.
- Multiple generation roots may share the same `generation_id` when `root_ref`
  values are unique.
- `source_status=trashed` normalizes to `trash`.
- Prefix-like `root_ref` / `parent_ref` format mismatches surface as warnings.
- Oversized nested-tree fixtures now block instead of returning a partial
  success; the result includes a truncation receipt.

## Still Closed

This release does not add live Notion deep-crawl transport. It does not call
providers, start OAuth, open Notion connections, read real exports, read page
titles or bodies, read comments, download media, write or merge fixtures, write
zets, mint pages, write edges, write receipts, update object manifests, or echo
provider URLs, local paths, account ids, emails, tokens, or secret values.
