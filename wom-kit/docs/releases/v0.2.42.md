# v0.2.42 Release Notes

Date: 2026-05-26

## Summary

v0.2.42 adds a CLI-only write approval layer for recording a foreign block attestation statement draft as a local untrusted record and receipt.

This is not an attestation, signature, trust, import, mint, anchor, delegation, sharing, provider sync, or ZET transport release.

## Added

- `archive record-attestation-statement-draft <archive-root> --draft-preview <json-file> --dry-run --format json`
- `archive record-attestation-statement-draft <archive-root> --draft-preview <json-file> --approve --reviewed-by <safe-actor-id> --format json`
- Read-only MCP `record_attestation_statement_draft_check`
- Stale/tamper validation for v0.2.41 draft preview JSON
- Rollback-safe exclusive creation of:
  - `quarantine/foreign-blocks/<case-id>/attestation-statement-draft.json`
  - `receipts/quarantine/<case-id>.foreign-block-attestation-statement-draft.json`

## Safety

The write layer revalidates current candidate, candidate receipt, quarantine case/receipt, and decision record/receipt state before recording anything.

The record and receipt keep:

- `trust_state: untrusted_foreign`
- `attestation_status: not_created`
- `signature_status: not_created`
- all import/trust/attestation/signature/mint/provider/ZET/share flags false

The command rejects raw review-note bodies, foreign body text, provider URLs, local absolute paths, tokens, and secret-like values.

## Compatibility

No private archive migration is required.

Existing v0.2.41 statement draft previews remain read-only. v0.2.42 only adds a separate explicit CLI approval path for recording a local draft record and receipt.
