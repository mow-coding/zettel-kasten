# v0.3.9 - Project Intake Answer Recording

Release date: 2026-06-14

v0.3.9 is a compatible fast-follow to v0.3.8. It removes a practical bit of
manual JSON merging from the human-guided project intake loop by adding a
single-answer receipt writer.

## Added

- CLI `archive project-intake-record-answer --dry-run|--approve` for appending
  exactly one human-reviewed project-intake answer to a new session or existing
  decisions receipt.
- Single-answer JSON schema marker
  `wom-kit/project-intake-answer/v0.1`.
- `project-intake-decision-template` now also returns an
  `answer_record_template` for the one-question-at-a-time persistence path.

## Boundaries

- The command does not echo current or previous answer values in CLI output.
- The command writes a normal `project-intake-decisions` receipt on approval so
  existing status, source-intake, and objet-capture receipt checks continue to
  work.
- It rejects duplicate or out-of-order checklist ids.
- It does not inspect staged file bodies, run source-intake, capture objets,
  derive text, create drafts, mint zets, call providers, upload, or clean
  staged folders.

## Upgrade

No archive migration is required for v0.3.8 users.

To record the first answer in a project-intake session:

```text
archive project-intake-record-answer <archive-root> --answer <answer.json> --session-id <safe-session-id> --dry-run
archive project-intake-record-answer <archive-root> --answer <answer.json> --session-id <safe-session-id> --approve --reviewed-by <actor>
```

To continue from an existing project-intake decisions receipt:

```text
archive project-intake-record-answer <archive-root> --answer <answer.json> --receipt <receipt> --dry-run
archive project-intake-record-answer <archive-root> --answer <answer.json> --receipt <receipt> --approve --reviewed-by <actor>
```
