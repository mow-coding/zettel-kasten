# v0.3.208 - seeded but exhaustive node-reading order

v0.3.208 lets verified host-goal nodes guide what WOM reads first without
dropping any other node. It is additive and requires no archive migration.

## What Changed

- Added CLI `--order seeded_connection_walk` with repeated
  `--start-zettel-id`; MCP uses `order` and `start_zettel_ids`.
- Added deterministic breadth-first ordering over incoming/outgoing safe zet
  edges as undirected reading passages.
- Added deterministic traversal of every disconnected component and isolated
  node after the seed-connected prefix.
- Added `order_evidence` for seed resolution, prefix/fallback/passages,
  all-node preservation, no edge rewrite, no ranking, and no persisted map.
- Strict continuation v0.2 binds the order descriptor and fixed-size seed-list
  SHA-256. Seed changes, unsafe/missing ids, and mismatched modes block.
- Updated runtime/agent guidance to use seeds only when the host goal already
  has verified ids and never to stop after the seed component.

## Scale Evidence

A temporary synthetic 10,000-node chain seeded at node 9,000 completed 179
strict reading pages in 52.1 seconds on the development Windows host. It
traversed 9,999 passages, placed all 10,000 nodes in the seed-connected prefix,
needed zero fallback components, returned 10,000 unique ids, stayed within the
8,000-token page estimate, and reached archive-wide claim readiness.

This is a local synthetic observation, not an SLA or real-archive result.

## Compatibility And Safety

Path order remains the default. Existing archives and non-seeded calls need no
migration. Restart v0.3.207 in-flight strict passes at cursor zero because the
continuation schema changed; tokens are transient and not archive records.

The feature reads frontmatter only, calls no model/provider, scores no
relevance, rewrites no edge, hides no component, writes no map/index/state, and
creates no WOM goal/loop.
