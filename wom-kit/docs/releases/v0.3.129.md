# v0.3.129 - Notion Ancestor Crawl Scope Filters

Release date: 2026-06-22
Status: prerelease

v0.3.129 responds to the client-side finding that a full workspace fixture can
produce an overly broad ancestor crawl queue before the live fetch adapter
exists. The crawl request planner can now narrow broad queues before they are
handed to any future credential-bounded adapter.

## Added

```powershell
archive notion-ancestor-crawl-plan <archive-root> `
  --tree workbench/notion-nested-tree.sample.json `
  --source notion `
  --scope-generation-id DB1 `
  --dry-run `
  --format json
```

Additional filters:

```text
--scope-root-ref
--scope-ancestor-ref
--scope-leaf-ref
```

MCP:

```text
notion_ancestor_crawl_plan
```

The planner now reports:

- `scope_filter.active`,
- `scope_filter.unfiltered_crawl_request_count`,
- `scope_filter.filtered_crawl_request_count`,
- `scope_filter.excluded_crawl_request_count`,
- request-level `affected_generation_ids`,
- request-level `affected_root_refs`,
- request-level `lineage_refs_seen`.

This makes the broad-workspace case explicit: a fixture can be planned broadly,
then narrowed to only the requested generation, root/ref, ancestor, or leaf refs
before any adapter sees the queue.

## Still Closed

This release still does not add live Notion transport. It does not call
providers, start OAuth, open Notion connections, read page titles or bodies,
read comments, download media, write queue files, write or merge fixture files,
write zets, mint pages, write edges, write receipts, update object manifests,
or echo provider URLs, local paths, account ids, emails, tokens, or secret
values.
