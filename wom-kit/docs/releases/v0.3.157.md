# v0.3.157 - Honest Capture Guidance

v0.3.157 makes `source-intake` and `objet-capture` guidance tell the truth
about the local objet capture flow that has existed since v0.3.2.

## What Changed

- Replaced the stale `candidate_unmanifested` next safe actions in
  `archive source-intake --dry-run`. The planner no longer tells operators to
  wait for a future explicit objet capture/import flow; it now routes to:
  - staging the file inside the archive root and preparing an approved
    selection with `objet-capture-selection`,
  - `objet-capture --selection <selection-path> --dry-run` first, noting
    that capture requires a sandbox-marked archive in this version,
  - `prehashed-objet-ledger` plus `object-storage-upload-evidence` for bytes
    already stored externally.
- Renamed the vague setup-planner warning: when object storage is missing, the
  planner now says object storage is not configured in `provider-bindings.yml`
  and points at `archive object-storage --dry-run`.
- Scoped the "outside registered source roots" warning to genuinely external
  files. Files that resolve inside the archive root (for example
  `staging/incoming/`) are archive content and no longer warn; when the
  warning fires it now names `archive add-source`.
- Added static `hint` strings to `objet-capture` refusals:
  - `sandbox_marker_required` explains the `.wom-sandbox` marker / top-level
    `environment: sandbox` requirement and that real-archive capture
    enablement is a separate planned flow,
  - `external_live_never_touch` explains the live-store protection pattern.
- Added `.m4a` -> `audio/mp4` (the standard MP4-audio mime) to
  `OBJET_CAPTURE_MIME_BY_EXTENSION`; `source-intake` records the same value
  where the platform mimetypes database maps `.m4a`.
- Corrected the artifact-hygiene "Current Gaps" list: local objet capture
  exists (v0.3.2+) but only for sandbox-marked archives; a real-archive
  capture enablement flow does not exist yet.

## Safety Boundary

Refusal outputs stay blocker-only: the new hints are static, blocker-keyed
strings that never echo item provenance, filenames, or receipt paths. No new
reads, writes, provider calls, or capture paths were added; the sandbox marker
requirement for capture is unchanged.

## Still Future

- A real-archive (non-sandbox) capture enablement flow.
- A provider capture/import flow for `provider_reference` items.
- Registering source roots automatically from intake plans.
