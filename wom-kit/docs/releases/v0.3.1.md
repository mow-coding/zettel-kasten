# v0.3.1 - Shared Update Route Preview

Release date: 2026-06-04

v0.3.1 adds a read-only receiver-side route preview for one local shared update record.

## Added

- CLI `archive shared-update-route-preview <archive-root> --record <path> --dry-run --format json`.
- Local service `shared_update_route_preview`.
- Route pointer fields for `delegate`, `attest`, `anchor`, and `none`.
- Explicit `related_shared_update_review_required_flags` when the route points toward `shared-update-attestation-review`.
- Hardening so free-form or hostile proposed-action metadata is not echoed as a route.
- Documentation: [Shared Update Route Preview](https://github.com/mow-coding/zettel-kasten/blob/main/wom-kit/docs/shared-update-route-preview.md).
- Work log: [Shared Update Route Preview Work Log](https://github.com/mow-coding/zettel-kasten/blob/main/wom-kit/plans/work-log-2026-06-04-shared-update-route-preview.md).

## Behavior

The command first reuses `zet_shared_update_record_review_preview`.

If the preview blocks, the route preview blocks.

If the preview passes, the command writes no files and returns only route pointers:

```text
delegate
attest
anchor
none
```

The route preview does not perform the route. It points to an existing canonical command surface and returns `would_change: []`.

## Safety Boundary

This release does not implement real ZET transport, key creation, key-sharing registry, radio-frequency access creation, mirroring delivery, neighbor feed update, automatic renewal, trust graph mutation, import, acceptance, real attestations, signatures, anchors, apply behavior, public proof anchoring, DID/wallet/key custody, provider sync, WordPress publishing, projection writes, projection receipts, queues, workers, payment, staking, consensus, blockchain, token, system token, model training, backpropagation, or full-auto behavior.

MCP exposes no shared-update route write/apply/approve tool and no new shared-update route MCP tool for this boundary.

## Upgrade

The route-preview command itself requires no provider, transport, or
shared-update record migration.

Archives authored from older `wom-kit/zettel-kasten-rules/v0.2-draft`
frontmatter guidance may still need a frontmatter compatibility preview before
strict v0.3 validation:

```text
archive migrate <archive-root> --target frontmatter-v0.3 --dry-run
```

See the [Upgrade Guide](https://github.com/mow-coding/zettel-kasten/blob/main/UPGRADE.md) and [Changelog](https://github.com/mow-coding/zettel-kasten/blob/main/CHANGELOG.md).
