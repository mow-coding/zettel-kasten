# v0.3.127 - Notion Client Issue Verification Bundle

Release date: 2026-06-22
Status: prerelease

v0.3.127 adds one more read-only safety layer after v0.3.126: a single
client-facing verification command for sanitized Notion nested-tree issue
bundles.

## Added

```powershell
archive notion-client-issue-verification-plan <archive-root> `
  --tree workbench/notion-nested-tree.sample.json `
  --ancestors workbench/notion-ancestor-result.sample.json `
  --source notion `
  --dry-run `
  --format json
```

MCP:

```text
notion_client_issue_verification_plan
```

The verifier orchestrates:

- sanitized nested tree planning,
- optional reviewed block mirror fixture preview,
- missing ancestor crawl request packaging,
- optional sanitized ancestor merge and after-merge replan.

It returns one verdict-oriented `plan_state`, including:

```text
client_issue_reproduced_missing_ancestor_evidence_needed
client_issue_verified_closed_by_sanitized_ancestor_merge
client_issue_still_needs_more_sanitized_ancestor_evidence
no_missing_ancestor_issue_detected_in_sanitized_input
```

## Still Closed

This release still does not add live Notion transport. It does not call
providers, start OAuth, open Notion connections, read real export directories,
read page titles or bodies, read comments, download media, write verification
files, write or merge fixture files, write zets, mint pages, write edges, write
receipts, update object manifests, or echo provider URLs, local paths, account
ids, emails, tokens, or secret values.
