# v0.3.62 - IMAP Header Metadata Scan

Released: 2026-06-16

This release opens the first narrow, approval-gated live IMAP adapter path:
header metadata scan only.

## Added

- `archive imap-mailbox-header-metadata-scan <archive-root> --dry-run|--approve
  --format json`.
- Aliases `archive imap-header-metadata-scan` and
  `archive mailbox-header-metadata-scan`.
- Approved-mode local IMAP TLS connection, login, read-only `INBOX` select,
  UID search, limited header fetch, and non-secret execution receipt writing.
- `env:` username and app-password ref support for this first live slice.

## Safety Boundary

The command requires the existing manifest, approval receipt, and execution
contract gates. It supports only `header_metadata_scan`, app-password auth,
environment refs, and inbox mailbox refs.

It returns no username values, password values, environment variable names,
exact credential refs, exact mailbox refs, IMAP host values, raw UID values,
Message-ID values, subjects, sender/recipient values, raw headers, bodies,
attachments, provider URLs, or local absolute paths.

## Still Future Work

OAuth, keyring/password-manager/secret-manager retrieval, message body capture,
RFC822 capture, attachment capture, derived-text capture, mailbox mutation, and
provider-specific account setup remain future work.
