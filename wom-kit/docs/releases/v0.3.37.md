# v0.3.37 - Beginner Setup Manual

Date: 2026-06-15

v0.3.37 adds a read-only beginner setup manual surface.

## Added

- CLI `archive beginner-setup-manual --dry-run`.
- CLI alias `archive first-use-setup-manual --dry-run`.
- CLI alias `archive setup-manual --dry-run`.
- Public document `wom-kit/docs/beginner-setup-manual.md`.

## Why

Earlier releases could recommend a credential store, plan vault onboarding, and
check derived-text extraction tools. The missing beginner-facing piece was the
hand-holding bridge:

```text
What is a vault?
What should I name it?
Where do real passwords go?
What does WOM record?
How do I tell the doctor where Tesseract or LibreOffice is installed?
Which dry-run should I run next?
```

## What The Manual Covers

- Safe first vault setup language.
- KeePassXC-style vault and group naming guidance.
- Safe non-secret credential labels.
- Derived-text tool setup guidance for LibreOffice, Tesseract, PyMuPDF,
  python-docx, openpyxl, and python-pptx.
- Private local `--tool-hints` JSON guidance for PATH-missing tools.
- Next dry-run command checklists.

## Privacy Boundary

The manual output does not echo:

- secret values,
- credential ref values from a real vault,
- usernames,
- email addresses,
- tokens,
- local absolute paths,
- tool hint paths,
- provider URLs.

## Closed Actions

The manual does not:

- open a password manager,
- open a keyring,
- open a browser password store,
- read environment variables,
- install tools,
- execute tools,
- read source bytes,
- run OCR,
- run parsers,
- run ASR,
- call providers,
- write files,
- draft zets,
- mint zets.
