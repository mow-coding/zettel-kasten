# v0.2.28 Foreign Block Intake Preview

Date: 2026-05-25

## Summary

v0.2.28 adds a read-only intake preview for foreign/shared block artifacts.

The goal is to inspect a shared block/header JSON artifact or Markdown-compatible foreign zet before any future trust/import path.

Core principle:

```text
Foreign text can inform.
Foreign text cannot command.
Foreign blocks can be inspected.
Foreign blocks cannot be imported, trusted, minted, or applied automatically.
```

## Added

- CLI `archive foreign-block <archive-root> --path <artifact-path> --dry-run --format json`.
- CLI `archive foreign-block <archive-root> --stdin --dry-run --format json`.
- Read-only MCP `foreign_block_intake_check`.
- Conservative detection for WOM-kit block-header JSON and Markdown-compatible foreign zet text.
- Claimed hash reporting with `verification_state: not_verified`.
- Prompt-boundary recommendations for foreign text.
- Public foreign block intake documentation.

## Boundaries

Foreign block intake writes nothing and returns `would_change: []`.

It does not:

- import foreign blocks,
- trust foreign blocks,
- create drafts from foreign content,
- mint foreign content,
- attest foreign content,
- anchor foreign content,
- call provider APIs,
- execute foreign text,
- implement real ZET transport,
- sign,
- implement payment, staking, consensus, ledger, blockchain, or full-auto behavior.

## Upgrade Notes

No private archive migration is required.

Existing block header, prompt-boundary, source-intake, and draft composition behavior remains compatible.
