# v0.2.52 Release Notes

Date: 2026-05-27

## Summary

v0.2.52 adds a local public privacy hygiene checker for release and public-documentation safety.

This is a docs/tooling guardrail and version bookkeeping release. It does not add product CLI/MCP behavior.

## Added

- [Public Privacy Hygiene](https://github.com/mow-coding/zettel-kasten/blob/main/wom-kit/docs/public-privacy-hygiene.md) documentation.
- [Public privacy checker](https://github.com/mow-coding/zettel-kasten/blob/main/wom-kit/tools/check_public_privacy.py).
- Unit tests for local path, token-like string, private key header, seed-phrase-like text, private endpoint, credential-bearing URL, placeholder, repository-pass, and network-free checker scope.
- [Public-safe work log](https://github.com/mow-coding/zettel-kasten/blob/main/wom-kit/plans/work-log-2026-05-27-public-privacy-hygiene-checker.md) for the v0.2.52 checker batch.

## Compatibility

No private archive migration is required.

No archive product CLI or MCP behavior changed.

The checker reads local Git-known public text files only. It catches obvious local/private endpoints and credential-bearing URLs, but it is not a general-purpose cloud secret scanner. It does not rewrite files, fetch external URLs, call providers, inspect private archives, edit GitHub Releases, or scan the whole disk.

## Safety Boundary

v0.2.52 does not add product CLI commands, product MCP tools, archive service behavior, real ZET transport, RF access, key-sharing registry, mirroring delivery, real trust/import/acceptance/anchor, attestation/signature writes, provider calls, provider sync, WordPress publishing, projection writes, projection receipts, recommendation fetching, recommendation ranking, feed updates, Redis, queues, background workers, payments, staking, consensus, blockchain, model training, backpropagation, or full-auto behavior.
