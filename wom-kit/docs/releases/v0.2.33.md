# v0.2.33 Foreign Block Quarantine Review Index

Release date: 2026-05-25

v0.2.33 adds a read-only review index for existing foreign block quarantine cases.

This is still not import, not trust, not attestation, not acceptance, and not ZET transport. The index is a safer review table for untrusted quarantine cases only.

## Added

- `archive quarantine-review <archive-root> --format json`
- optional `--case-id <safe-id>`
- optional `--status written_untrusted|all`
- optional `--include-receipts`
- read-only MCP `foreign_block_quarantine_review_index`
- case summary validation for `quarantine/foreign-blocks/<case-id>/quarantine-case.json`
- matching quarantine write receipt consistency checks under `receipts/quarantine/<case-id>.foreign-block-quarantine.json`

## Safety Boundary

The review index:

- reads only existing quarantine case JSON and matching quarantine receipts,
- keeps `trust_state: untrusted_foreign`,
- emits archive-relative paths only,
- blocks unsafe case ids, malformed case JSON, trust-state contradictions, receipt contradictions, and unsafe private values,
- returns warnings for missing receipts or review metadata gaps,
- always returns `would_change: []`.

The review index does not:

- write files,
- read the original foreign artifact,
- import the foreign block,
- mark the foreign block trusted,
- accept or apply a quarantine case,
- create an attestation,
- mint, anchor, delegate, sign, or execute anything,
- call provider APIs.

## MCP

MCP exposes only `foreign_block_quarantine_review_index` for this workflow. It is read-only and requires `dry_run: true`.

MCP does not expose quarantine review apply, accept, import, trust, attestation, receipt write, mint, anchor, delegate, signing, provider sync, ZET transport, or full-auto tools.

## Compatibility

No private archive migration is required.

Existing v0.2.32 quarantine cases and quarantine write receipts remain compatible. Archives without quarantine cases return an empty index with a warning and write nothing.
