# v0.3.33 - KeePassXC Write Adapter

Date: 2026-06-15

v0.3.33 adds the first minimal live credential write adapter.

## Added

- CLI `archive credential-keepassxc-write --approve`.
- CLI alias `archive keepassxc-write --approve`.
- Dry-run mode `archive credential-keepassxc-write --dry-run`.
- Non-secret execution receipts under:

```text
receipts/credentials/keepassxc-writes/
```

- `wom-kit/docs/credential-keepassxc-write.md`.

## What It Does

The command verifies a written approval receipt through the same policy gate as
the v0.3.32 command preflight, then approved CLI mode invokes:

```text
keepassxc-cli add --password-prompt <local database path> <entry>
```

The local `.kdbx` path is used only for the subprocess call. It is not written
to JSON output or receipts.

The command writes one non-secret execution receipt after the attempt and blocks
replay with the same approval receipt and entry target.

## Safe Chain

```text
credential-access-approval --approve
-> credential-policy-check --approval-receipt <path> --dry-run
-> credential-keepassxc-command-plan --approval-receipt <path> --dry-run
-> credential-keepassxc-write --approval-receipt <path> --approve
-> non-secret KeePassXC write execution receipt
```

## Still Not Included

v0.3.33 does not add:

- MCP live KeePassXC execution,
- secret retrieval,
- plaintext secret file scanning,
- OS keyring writes,
- browser password-store access,
- OAuth,
- provider API calls,
- OpenAI API calls,
- paid OCR calls.

## Privacy Boundary

The command and receipt do not echo or store:

- secret values,
- database passwords,
- `.kdbx` paths,
- exact credential refs,
- usernames,
- email addresses,
- tokens,
- provider URLs,
- raw adapter stdout/stderr.

The new entry password is typed into the local `keepassxc-cli
--password-prompt` flow. WOM-kit does not accept it as an argv value and does
not pipe it through stdin.
