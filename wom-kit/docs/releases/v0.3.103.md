# v0.3.103 - Notion Source Map Material Link Planner

Date: 2026-06-17

## Summary

v0.3.103 adds a read-only fallback planner for the Phase 3 Notion material-link
case where imported zettel bodies no longer contain provider locators.

The new command is:

```powershell
archive notion-objet-source-map-link-plan <archive-root> --dry-run --format json
```

It can also use explicit archive-relative source maps and download/retrieval
ledgers:

```powershell
archive notion-objet-source-map-link-plan <archive-root> --source-map source-maps/notion-export.jsonl --ledger receipts/import/notion-download-ledger.jsonl --dry-run --format json
```

MCP exposes the same read-only planner as:

```text
notion_objet_source_map_link_plan
```

## User-Facing Meaning

Earlier Notion objet-link tools worked when the zettel body still contained a
provider locator that could be fingerprinted. Real public-safe imports may
remove that locator from the body first. This release gives WOM-kit a safer next
step: join source maps, optional ledgers, zettel frontmatter metadata, and the
object manifest by private-value fingerprints, then propose human-review
`embed` candidates.

This means an AI runtime can say:

```text
The body is clean, so we should not search the body for the old locator.
Use the source map and ledger trail to recover candidate material links.
```

## Safety Boundary

This release adds no live provider adapter and no material-link writer. The new
planner:

- writes nothing,
- rewrites no zettel body text,
- writes no edges or receipts,
- reads no zettel body text,
- reads no object bytes,
- calls no providers,
- creates no presigned URLs,
- echoes no provider URLs, provider locator text, page titles, frontmatter
  values, local absolute paths, account ids, emails, tokens, or secret values.

Durable edge writes still require later human review through the existing
approval-gated `zettel-edge` or `zettel-edge-batch` paths.

## Migration

No archive migration is required.

Future import adapters should preserve the material clue outside the zettel body
as a safe `source_refs` object id, a planned/reviewed `embed` edge candidate, or
a source-map/ledger row that can join page -> file -> `sha256`.
