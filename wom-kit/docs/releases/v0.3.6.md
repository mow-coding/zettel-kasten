# v0.3.6 - Project Intake Decision Recording

Release date: 2026-06-13

v0.3.6 is a compatible fast-follow to v0.3.5. It turns the first
human-guided project intake checklist into a durable local receipt flow while
keeping capture, drafting, minting, provider calls, and cleanup behind separate
gates.

## Added

- CLI `archive project-intake-decisions <archive-root> --decisions <json-file> --dry-run|--approve --reviewed-by <actor>`.
- CLI `archive project-intake-status <archive-root> --receipt <receipt> --dry-run`.
- Optional CLI context link `archive source-intake --project-intake-receipt <receipt>`.
- Tests for decision-record dry-run, approved receipt writing, receipt status
  integrity, tamper detection, unsafe value blocking, and source-intake context
  validation.

## Changed

- `archive project-intake-plan --dry-run` now has a stronger human review
  checklist, classification labels, and a draft decision-record template.
- `source-intake` can carry a reviewed project-intake receipt as session
  evidence for one-locator metadata dry-runs.
- Project intake docs and the capability matrix now distinguish the implemented
  decision-recording surfaces from the still-planned full intake executor.

## Boundaries

- `project-intake-decisions` reads the user-provided decisions JSON, validates
  safe checklist ids and unsafe-string boundaries, and writes one local receipt
  only after `--approve --reviewed-by`.
- CLI output reports counts, checklist ids, receipt paths, hashes, blockers,
  and warnings; it does not echo recorded answer text.
- `project-intake-status` is read-only and never authorizes automatic execution.
- `source-intake --project-intake-receipt` treats the receipt as context only.
  It does not run capture, derive text, create drafts, mint zets, call
  providers, or approve staged-folder cleanup.

## Upgrade

No archive migration is required for v0.3.5 users.

To start a human-guided intake session, first run:

```text
archive project-intake-plan <archive-root> --staged-folder <folder> --dry-run
```

Then record reviewed answers with:

```text
archive project-intake-decisions <archive-root> --decisions <json-file> --approve --reviewed-by <actor>
```
