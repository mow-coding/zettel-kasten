# v0.3.101 - Notion Objet Link Convert

Date: 2026-06-17

## Added

- CLI `archive notion-objet-link-convert <archive-root> --path <zet.md>|--zettel-id <id>
  --locator-fingerprint sha256:<hex> --object-id sha256:<hex>
  --target-mode embed_edge --expected-occurrence-count <n>
  --dry-run|--approve`.
- Public docs for the approved embed edge conversion:
  `wom-kit/docs/notion-objet-link-convert.md`.

## Why

The v0.3.98 rewrite plan could prove that one reviewed Notion locator
fingerprint still matched one manifested object. The v0.3.100 manifest label
write let an object manifest gain that non-secret locator fingerprint when it
was missing.

This release adds the next narrow write: after review, WOM-kit can convert the
selected locator/object match into one durable `embed` edge without editing the
zettel body.

## Behavior

- Requires either `--dry-run` or `--approve`.
- Requires `--reviewed-by` for approved writes.
- Requires `--expected-occurrence-count` for approved writes.
- Re-runs `notion-objet-link-rewrite-plan` before writing.
- Supports only `--target-mode embed_edge`.
- Routes the edge write through the existing `zettel-edge` validation and
  receipt path.
- Writes one conversion receipt under
  `receipts/objects/notion-link-conversions/`.
- Restores touched zettel and receipt paths if the conversion write fails
  partway.

## Safety

The new command is CLI-only. MCP exposes no matching write tool.

It does not rewrite zettel body text, replace provider locator text, call
Notion, start OAuth, read real source exports, read object bytes, write
candidate records, update object manifests, upload, download, or create
provider URLs.

Output avoids provider URLs, provider locator text, zettel body text, zettel
titles, frontmatter values, page titles, absolute local paths, account ids,
emails, tokens, and secret values.
