# v0.3.182 - adopt resume and doctor mint-receipts progress

v0.3.182 responds to the basoon v0.3.180 revalidation. That run confirmed the
`object-storage-adopt-existing` adopt-plan manifest index: the plan stage dropped from minutes of
repeated manifest scans to a few seconds on the large key-map batch. The remaining operator
problems were the long `adopt-verify` remote HEAD loop and insufficient `doctor mint-receipts`
progress detail.

This release is additive. Defaults remain conservative.

## Adopt resume helper

`archive object-storage-adopt-existing ... --approve --skip-existing-wom-uploaded` is a new
explicit resume helper for interrupted verified adopts.

When an object already has a matching `wom_uploaded` manifest location for the same
provider/store/resolved key, the command reports that row as `already_wom_uploaded_manifest` and
does not remote-HEAD it again. This is intended for the common interrupted-run shape where most
rows were already verified and written before the operator stopped the long batch.

The default remains unchanged: without `--skip-existing-wom-uploaded`, verified adopt still
re-HEADs each resolved key. The resume option is refused together with `--content-hash-verify`,
because a fresh content-hash verification request must not be silently converted into a manifest
resume skip.

The adopt summary now includes:

- `existing_wom_uploaded_count`
- `skipped_existing_wom_uploaded_count`
- `skip_existing_wom_uploaded`

When existing matching locations are found but the resume option is not enabled, the command emits
a `resume_hint` warning.

## Doctor mint-receipts progress

`archive doctor --strict --progress` now reports sampled `mint-receipts` sub-steps, including
receipt JSON loading, schema checks, source/target/snapshot file-ref checks, and target
frontmatter loading.

It also prints a cache summary for:

- file SHA-256 cache hits/misses;
- zettel frontmatter cache hits/misses;
- zettel BOM evidence cache hits/misses.

This does not change diagnostics. It gives an operator a clearer last-known sub-step when a large
archive spends a long time inside `mint-receipts`.

## Tests

Added focused regression coverage that:

- `--skip-existing-wom-uploaded` avoids a remote HEAD for an already matching manifest location;
- the skipped row is counted and reported as `already_wom_uploaded_manifest`;
- `doctor` mint-receipts progress emits sub-step messages and a cache summary.

## Compatibility

Additive; **no migration required**. Existing commands keep their default behavior unless the new
resume flag is supplied.
