# v0.3.236 - Canonical Revision Receipt Audit

v0.3.236 adds the archive-wide read-only verifier for the canonical correction
path introduced in v0.3.234 and v0.3.235.

## What Changed

- Added CLI `zet-revision-receipt-audit` with aliases
  `revision-receipt-audit` and `canonical-revision-audit`.
- Validated receipt schema, archive identity, filename/write digest, safe
  reviewer shape, timestamps, human edge review, and abstract/body evidence.
- Reconstructed each private revision history as one before/after hash chain
  from current canonical bytes through valid superseded receipts.
- Detected branches, duplicate states, identity conflicts, unreachable
  receipts, current drift, evidence gaps, and non-monotonic revision time.
- Classified leftover per-canonical locks as completed, recoverable missing
  receipt, prewrite, ambiguous, invalid, or unsupported.
- Kept work bounded to one open per receipt/lock and at most one current
  canonical read per identity; no per-receipt full zet-tree resolution.
- Kept output content-free and the operation local, provider-free, and
  write-free. No MCP tool was added.

## Upgrade And Compatibility

No archive migration is required. Restart the AI operator process so the new
CLI command is visible. Run the audit after canonical revision work and before
assuming that retained receipts describe one current history.

## Safety And Claim Boundary

- The command writes and deletes nothing.
- It never removes a completed, prewrite, recoverable, ambiguous, invalid, or
  unsupported lock.
- `recoverable_missing_receipt` directs the operator back to the exact
  approved v0.3.235 command; it does not rewrite the canonical zet.
- Output omits zet id/path, proposal filename, title, abstract/body text,
  reviewer id, provider URL, absolute path, and secret values.
- A green result proves bounded local evidence consistency, not factual truth,
  backup, external synchronization, or model understanding.
- Canonical revert remains separate because hashes alone cannot recreate the
  deliberately unstored old body.
