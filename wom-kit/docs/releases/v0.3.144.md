# v0.3.144 - AI Scratch Cleanup And zet Self-Contained Checks

v0.3.144 fixes a real AI collaboration failure mode: AI scratch files should
not become permanent archive dependencies, and public external citation URLs
should not be confused with private provider locators or original-file storage
locations.

## What Changed

- Added read-only `archive zet-self-contained-check`, with alias
  `archive zettel-self-contained-check`.
- Added approval-gated `archive ai-scratch-gc`, with aliases
  `archive ai-residue-cleanup` and `archive scratch-gc`.
- `.wom-scratch/` and `workbench/ai-scratch/` are now generated archive
  `.gitignore` defaults and artifact hygiene `DISPOSABLE_AFTER_REVIEW` paths.
- `create-draft` and `mint-zet` now allow public external citation URLs in zet
  bodies and `source_refs`.
- Private provider locators, local absolute paths, and object-store locations
  remain blocked by the `object_id_only` machine gate.
- Approved `mint-zet` removes explicit AI scratch refs from canonical
  `source_refs` and runs the same cleanup gate after canonical, mint receipt,
  and draft snapshot writes succeed.

## Command Shape

```powershell
archive zet-self-contained-check <archive-root> `
  --path inbox/example.md `
  --dry-run `
  --format json
```

```powershell
archive ai-scratch-gc <archive-root> `
  --path inbox/example.md `
  --dry-run `
  --format json
```

```powershell
archive ai-scratch-gc <archive-root> `
  --path inbox/example.md `
  --approve `
  --reviewed-by human:<id> `
  --format json
```

## Safety Boundary

- Cleanup touches only explicit plain files under `.wom-scratch/` or
  `workbench/ai-scratch/`.
- Cleanup never scans broad archives and never touches `objets/`.
- `objets/` remains the place for human-selected original material.
- AI intermediate reports, research notes, and draft composition files belong in
  scratch unless a human explicitly preserves them as objets.
- Command output echoes no zet body text, provider URLs, local absolute paths,
  tokens, or secret values.

## Upgrade Notes

No archive migration is required.

Existing draft and mint commands keep working. Archives can run
`archive repair-gitignore --dry-run` and approve the repair if they want the new
scratch ignore patterns appended to older `.gitignore` files.
