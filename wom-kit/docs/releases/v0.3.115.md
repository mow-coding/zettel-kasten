# v0.3.115 - Public Roadmap Baseline

Date: 2026-06-19

## What Changed

This release adds a public roadmap for how to read the pre-1.0 WOM version
lines.

The roadmap records the intended phase meaning:

- `v0.1.x`: idea and protocol language,
- `v0.2.x`: local implementation,
- `v0.3.x`: WOM real-use feedback and safety hardening,
- `v0.4.x`: custom UI control layer,
- `v0.5.x`: ZET real-use feedback.

The new roadmap lives at:

```text
wom-kit/docs/product-roadmap.md
```

It is linked from the README, public documentation map, capability matrix, and
this release note so readers do not need to infer the project direction from
individual changelog entries.

## Why This Matters

The `v0.3.x` line has become a real-use feedback line for WOM as a private,
local archive system. The new roadmap makes that explicit and also marks the
future phases:

- `v0.4.x` should connect a custom UI/control layer to the existing CLI,
  receipts, validation, and safety gates;
- `v0.5.x` should begin ZET real-use feedback only after ordinary WOM operation
  is clear enough that feedback can focus on sharing and communication
  behavior.

## Safety Boundary

This release is documentation-only. It does not add:

- new product CLI behavior,
- UI behavior,
- live provider adapters,
- ZET transport,
- wallet creation,
- token mechanics,
- provider sync,
- queues or background workers.

The capability matrix remains the plain-language source of truth for what WOM
can do today.
