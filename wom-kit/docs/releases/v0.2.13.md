# v0.2.13 Release Note

Date: 2026-05-23

## Summary

`v0.2.13` is a compatible naming baseline patch.

It records `WOM` as the umbrella name for the project and starts moving the public product surface away from generic SaaS words and earlier implementation names.

```text
WOM = Widesider of Modernity
zet = active text primitive
node = subject/archive participant
mint -> delegate -> attest -> anchor = preferred lifecycle
```

The release also adds compatibility-safe CLI aliases:

```text
mint-zet  -> preferred name for minting a zet
parcel    -> preferred name for creating a bounded portable unit
admit     -> preferred name for previewing governed admission of a parcel/workpack
```

## Why This Exists

The project is no longer just an "AI archive kit."

That name was useful while the first implementation was being assembled, but it does not carry the full design philosophy:

- local-first archive memory,
- AI-native conversation-to-zet workflows,
- Web3-like subject-owned verification,
- social/urban ideas of nodes, ties, anchoring, and governed admission,
- future messenger, feed, and collaboration behavior built from the same zet lifecycle.

`WOM` is the broader name for that worldview.

- Tests that confirm the new aliases work without breaking old commands.

## Compatibility

Existing v0.2 names remain valid:

- `archive mint-zettel`
- `archive pack`
- `archive import --dry-run`
- `archive promote`
- `archive share --dry-run`

Existing file paths and schema names remain unchanged:

- `wom-kit/`
- `zettels/`
- `receipts/`
- `workpacks/`
- receipt schema filenames

This is intentional. `v0.2.13` sets the language direction without forcing archive migrations or breaking scripts.

## Migration

No private archive migration is required.

Recommended new command style:

```powershell
python wom-kit\cli\archive.py mint-zet <archive> --path inbox\some_draft.md --dry-run
python wom-kit\cli\archive.py parcel <archive> --view <view-id> --purpose "Portable context." --mode reference
python wom-kit\cli\archive.py admit <archive> <parcel-or-workpack> --dry-run
```

Old command names keep working as compatibility aliases.

## Not Included

This release does not implement:

- real P2P transport,
- real attestation writes,
- real anchor writes,
- real parcel/workpack admission writes,
- blockchain settlement,
- package/folder/schema renames,
- UI.
