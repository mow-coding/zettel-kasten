# v0.3.202 - WOM operator vocabulary translation

v0.3.202 adds a broader Korean-first vocabulary layer for AI operators helping a
human use WOM.

No archive migration is required.

## What Changed

`archive ai-response-concept-guide` now supports:

```text
archive ai-response-concept-guide <archive-root> --topic operator_vocabulary --locale ko-KR --dry-run --format json
```

The `all` topic also includes the new `operator_vocabulary` section.

The vocabulary covers:

- archive entry terms: `archive`, `archive_root`, `AGENTS.md`,
  `runtime_context`, `ai_start_here`, `operational_context`, `capabilities`;
- knowledge records: `zet`, `draft`, `inbox`, `canonical`, `frontmatter`,
  `source_refs`;
- evidence layers: `objet`, `object_id`, `derived_text`, `manifest`,
  `source_map`, `receipt`;
- actions and checks: `dry_run`, `approve`, `mint`, `retire`, `reconcile`,
  `doctor`, `validate`, `privacy_scan`;
- connections: `node`, `edge`, `tie`, `link_type`, `source_mechanism`,
  `relationship_meaning`;
- providers and secrets: `provider`, `adapter`, `credential_ref`, `secret`,
  `vault`, `provider_locator`, `object_storage`.

## Korean-First Examples

- `ai_start_here` -> "첫 안내판 / 시작 지도"
- `zet` -> "정식 메모 / 기억 조각"
- `objet` -> "원본 자료 / 오브제"
- `object_id` -> "자료 지문"
- `receipt` -> "작업 영수증"
- `dry_run` -> "미리보기"
- `mint` -> "정식 발행"
- `doctor` -> "건강검진"
- `tie` -> "관계 / 이어짐"
- `credential_ref` -> "비밀값 이름표"

## Safety

This is a read-only language guide. It writes nothing, reads no archive bodies,
calls no providers, reads no secrets, and does not change machine, JSON, or
receipt terminology. The AI still preserves exact WOM terms in backticks or
logs when precision matters.
