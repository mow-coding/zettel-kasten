# v0.3.141 - Notion One-Click Connection Contract And Actionable Failures

v0.3.141 responds to the beta-tester recovery retrospective: the recovery
capability worked, but the credential and connection experience still felt like
a developer procedure.

## What Changed

- Added `archive notion-connection-plan`.
- Added aliases `archive notion-connect-plan` and
  `archive notion-one-click-connection-plan`.
- Added safe Notion provider failure classification to the live ancestor fetch
  adapter.
- Surfaced those categories through `archive notion-recover`.
- Added `wom-kit/docs/notion-connection-plan.md`.
- Updated the capability matrix, README summaries, changelog, and tests.

## Connection Contract

The new read-only plan records the target product experience:

```text
Connect Notion -> human approves once in browser -> WOM runs recovery locally -> AI tidies without seeing secrets
```

The command separates the official Notion connection models:

- internal connection: static installation token, page/database sharing required,
- personal access token: user-scoped local trusted-tool fallback,
- public OAuth connection: target "Connect Notion" product direction.

The existing env/file token path remains a power-user fallback. This release
does not claim browser OAuth or a managed callback server is implemented.

## Actionable Failures

`notion-recover` now reports safe categories when live provider checks fail:

```text
token_invalid_or_expired
notion_connection_not_shared_or_permission_denied
notion_object_missing_or_not_shared
provider_rate_limited
network_or_timeout
provider_temporarily_unavailable
provider_request_failed
```

This keeps raw provider errors redacted while giving the human a next action.
For the beta-tester failure shape, the important signal is usually
`notion_connection_not_shared_or_permission_denied`: the token may be valid, but
the relevant page or database has not been shared with the connection.
