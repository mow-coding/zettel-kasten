# v0.3.98 - Notion Objet Link Rewrite Plan

Date: 2026-06-17

## Added

- CLI `archive notion-objet-link-rewrite-plan <archive-root>
  --path <zet.md>|--zettel-id <id> --locator-fingerprint sha256:<hex>
  --object-id sha256:<hex> --dry-run`.
- MCP `notion_objet_link_rewrite_plan`.
- Public docs for the reviewed Notion locator/object conversion checkpoint.

## Why

`v0.3.96` added an archive-wide index and `v0.3.89` added a one-zettel locator
plan, but actual Notion locator conversion is still too risky to automate.

This release adds the missing approval-prep step: after a reviewer selects one
locator fingerprint and one manifested object id, WOM-kit can re-check that the
pair still matches the current zettel and manifest before any future write
command exists.

## Behavior

- Re-runs the one-zettel Notion locator plan.
- Verifies the selected `locator_fingerprint` is still present.
- Verifies the selected `object_id` is still a reviewed manifest candidate for
  that locator.
- Supports `target_mode` values `objet_ref_rewrite` and `embed_edge`.
- Supports optional `expected_occurrence_count` as a drift guard.
- Returns an approval checklist and a `would_change` preview for the future
  approved conversion shape.

## Safety

The new path writes nothing, rewrites no provider locators, writes no `embed`
edges, reads no object bytes, calls no providers, creates no provider or
presigned URLs, and echoes no provider URLs, zettel body text, frontmatter
values, page titles, absolute local paths, account ids, emails, tokens, or
secret values.

The result is a review checkpoint, not a conversion.
