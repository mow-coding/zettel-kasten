# v0.3.41 - Object Storage Adapter Readiness

Date: 2026-06-15

v0.3.41 adds a read-only object storage adapter readiness planner.

## Added

- CLI `archive object-storage-adapter-readiness-plan <archive-root> --dry-run`.
- Aliases:
  - `archive object-storage-adapter-plan`
  - `archive objet-storage-adapter-readiness`
- MCP tool:
  - `object_storage_adapter_readiness_plan`
- New document:
  - `wom-kit/docs/object-storage-adapter-readiness-plan.md`

## Behavior

The planner checks whether local metadata is ready for a future object-storage
adapter. It bridges:

- `provider-status`,
- setup-managed object-storage bindings,
- local provider setup receipts,
- future credential broker requirements,
- future credential policy checks,
- human approval receipts,
- future adapter manifests,
- future non-secret audit receipts.

Supported planned operations:

- `upload_object`
- `download_object`
- `head_object`
- `presigned_download`
- `presigned_head`
- `list_metadata_only`

## Safety

The command and MCP tool are read-only. They:

- call no provider APIs,
- retrieve no credential values,
- create no presigned URLs,
- upload nothing,
- download nothing,
- read no object bytes,
- check no remote availability,
- echo no bucket names,
- echo no prefixes,
- echo no provider URLs,
- echo no local absolute paths,
- echo no exact credential refs,
- echo no provider setup receipt paths,
- write no files or receipts.

This release still has no live object-storage adapter.
