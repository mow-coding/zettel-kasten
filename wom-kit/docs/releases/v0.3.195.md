# v0.3.195 - Doctor output path-policy clarification

v0.3.195 follows the v0.3.194 doctor result-capture release with smaller
operator-facing clarifications.

## Output Path Semantics

`archive doctor --output <path>` still writes the full diagnostics JSON array to
an archive-relative file.

The summary now makes that explicit:

```json
"output": {
  "written": true,
  "path": "logs/doctor-result.json",
  "path_kind": "archive_relative",
  "relative_to": "archive_root",
  "absolute_path_echoed": false,
  "contains": "full_diagnostics"
}
```

This avoids confusing `--output logs/...` with a CWD-relative shell path.

## Local Artifact Policy

The summary also labels the result file as:

```text
local_diagnostic_artifact_not_archive_receipt
```

The intended policy is: do not commit the full doctor result by default. Keep it
locally, summarize the relevant diagnostics into feedback, or delete it after
review.

## Retired-Draft Drift Hint

`mint_retired_draft_sha_mismatch` now carries a hint explaining that
`retire-draft-reconcile --dry-run` is the read-only classifier for retired-draft
receipt drift. Approval should happen only after the drift class and referenced
file are reviewed.

## Compatibility

Additive; **no migration required**.

Default doctor behavior and exit-code semantics are unchanged.
