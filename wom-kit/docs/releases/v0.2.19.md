# v0.2.19 WOM-kit Naming And Path Cleanup

Status: public pre-release patch

Date: 2026-05-25

## Summary

`v0.2.19` makes the implementation/tooling name match the current WOM product language.

The public system remains:

```text
WOM = full system / worldview / open infrastructure
zet = unit document minted inside a zettel-kasten
ZET = communication layer / service / protocol
```

The local implementation/tooling layer is now:

```text
WOM-kit  = display/product name for the local toolkit
wom-kit/ = repository implementation folder
wom_kit  = Python import package
```

The repository root remains `zettel-kasten`.

## Added

- Package metadata now uses `wom-kit`.
- Preferred console script aliases:
  - `wom`
  - `wom-mcp`
- Compatibility console scripts remain:
  - `archive`
  - `archive-mcp`

## Changed

- Current-facing docs, examples, schema titles, tests, wrapper scripts, and command examples now use `WOM-kit`, `wom-kit`, and `wom_kit` by context.
- The implementation folder now lives at `wom-kit/`.
- The Python package now lives at `wom-kit/src/wom_kit/`.

## Compatibility

Command behavior is unchanged.

The existing `archive` and `archive-mcp` console scripts remain available. MCP wire identifiers such as `zettel-kasten-archive-mcp` and `mcp:zettel-kasten-archive-mcp` remain stable for compatibility even though the public implementation name is now WOM-kit.

No private archive migration is required.

Existing lifecycle commands remain available.

## Non-Goals

This release does not add:

- source intake,
- GitHub repository creation,
- provider API sync,
- UI,
- any change to WOM/zet/ZET philosophy.
