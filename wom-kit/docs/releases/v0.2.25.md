# v0.2.25 Release Notes

Date: 2026-05-25

## Summary

v0.2.25 adds the WOM Profile Wallet Concept Baseline.

The key idea is that a WOM profile should feel closer to a wallet-ready identity than a SaaS account profile, while still avoiding real wallet security primitives in this release.

```text
WOM profile -> human-facing selectable profile
WOM node -> subject/principal in the WOM network
WOM wallet layer -> future signing/capability layer
```

## Added

- `archive profile-wallet <archive-root> --profile <profile-id-or-label> --dry-run --format json`
- read-only MCP `wom_profile_wallet_check`
- optional profile registry `node` metadata
- optional profile registry `wallet` metadata
- profile wallet model documentation
- safe example registry entry with placeholder wallet-ready metadata

## Compatibility

Existing profile registry entries remain valid.

No archive migration is required.

The new `node` and `wallet` fields are optional and public-safe only.

## Security Boundary

This release does not:

- generate private keys,
- store seed phrases,
- store wallet secrets,
- perform real cryptographic signing,
- create wallets,
- register wallets,
- call blockchain APIs,
- call provider APIs,
- implement WOM coin,
- implement NFT-like access,
- implement payments,
- implement staking,
- implement consensus, ledger, or P2P transport.

WOM profile is not a crypto wallet in v0.2.25. It is a wallet-ready identity model.

## Recommended Flow

When identity/capability authority matters, an AI runtime may use:

```text
profile-resolve
-> profile-wallet --dry-run
-> runtime-context
-> source-intake --dry-run when needed
-> create-draft --dry-run
-> human draft approval
-> mint dry-run
-> human mint approval
```

The preview is not approval to write, mint, sign, or anchor.
