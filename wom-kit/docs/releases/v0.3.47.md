# v0.3.47 - IMAP Mailbox Adapter Readiness Plan

Date: 2026-06-15

v0.3.47 adds the read-only readiness planner that sits between the IMAP
operation request package and any future live IMAP adapter.

## Added

- `archive imap-mailbox-adapter-readiness-plan --dry-run`
- Alias: `archive imap-mailbox-adapter-plan --dry-run`
- Alias: `archive mailbox-adapter-readiness --dry-run`
- MCP tool: `imap_mailbox_adapter_readiness_plan`
- Public doc:
  [IMAP Mailbox Adapter Readiness Plan](https://github.com/mow-coding/zettel-kasten/blob/main/wom-kit/docs/imap-mailbox-adapter-readiness-plan.md)

## What It Does

The planner combines:

- IMAP mailbox source planning,
- IMAP operation request packaging,
- credential policy state,
- local runtime module checks for future adapter implementation.

It returns a `readiness_state` such as:

- `ready_for_request_package`
- `ready_for_future_adapter_after_approval`
- `denied_by_human_decision`
- `denied_by_policy`
- `blocked`

## Important Safety Boundary

This release still does not implement live IMAP mailbox reads.

The new command does not:

- open an IMAP connection,
- attempt login,
- select a mailbox,
- read message headers,
- read message bodies,
- read attachments,
- create derived text,
- retrieve secrets,
- start OAuth,
- send email,
- delete email,
- change message flags,
- write files or receipts.

It also does not echo email addresses, username values, exact account refs,
exact credential refs, exact mailbox refs, IMAP host values, provider URLs,
message headers, message bodies, attachment names, approval receipt paths, local
absolute paths, tokens, or secret values.

## Migration

No archive migration is required.
