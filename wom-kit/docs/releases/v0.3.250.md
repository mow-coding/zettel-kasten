# v0.3.250 - Receipt-Backed Session Handoff Checkpoint

v0.3.250 closes the bounded gap between WOM's durable operating memory and a
host AI chat that can disappear at context reset.

## What Changed

- New CLI `session-handoff-checkpoint`, with aliases
  `session-memory-checkpoint` and `context-reset-checkpoint`.
- Read-only inspection checks the current approved operational context, its
  matching receipt, and the allowlisted AI artifact inventory together.
- The resulting `state_digest` binds exact current operational-context bytes,
  receipt identity, and a content-free inventory snapshot.
- Approval requires `--confirm-chat-reviewed`, a safe reviewer, and the exact
  digest returned by a fresh preview.
- Approval writes one content-addressed receipt under
  `receipts/session-handoffs/` and no semantic document.
- Existing receipts are reported as current only while all bound evidence
  still matches.
- New operational-context writes now serialize exact UTF-8 bytes so their
  receipt hash matches the file on disk. Older Windows newline-normalized
  receipts remain recognized and are labeled as legacy evidence.
- The installed `wom-archive` Agent Skill now routes AI operators through this
  close procedure before a context reset or session handoff.

## Why It Matters

`ops/operational-context.yml` already tells the next AI what the current
mission and work state are. `ai-artifact-inventory` already finds generated
files that may otherwise be forgotten. The missing step was proof that an
operator reviewed these surfaces together before leaving the chat.

The checkpoint does not introduce another competing free-form handoff document.
It records a bounded review of the durable artifacts WOM already owns.

## Safety Boundary

- Missing or unreceipted operational context blocks approval.
- Unreviewed AI artifacts or a truncated 1,000-item inventory block approval.
- A stale state digest or receipt collision fails closed.
- Any later operational-context or AI artifact inventory change makes the
  checkpoint stale.
- WOM does not read the host chat transcript or AI artifact bodies.
- The reviewer confirmation is not automatic semantic proof and does not
  require preservation of every raw chat sentence.
- The command calls no model, provider, network, object store, database, or
  credential store.
- It writes no zet and does not prove GitHub, object-storage, or external-DB
  backup completion.

## Verification

Regression coverage proves missing-receipt detection, explicit conversation
review, no-write preview, stale-plan refusal, approved receipt creation,
idempotent current-state audit, exact-byte staleness, unreviewed artifact
refusal, body/path redaction, CLI capability discovery, and operational-context
write compatibility. Release gates also cover the full test suite, runtime
skill validation, package resources, clean-wheel installation, public privacy,
and release readiness.
