# v0.3.64 - R2 Setup Guide Bridge

Date: 2026-06-16

v0.3.64 closes the gap between object-storage recommendation and the provider
dashboard fields a beginner sees during Cloudflare R2 setup.

## Added

- Added `archive beginner-setup-manual <archive-root> --topic
  object_storage_setup_manual --dry-run --format json`.
- Extended `archive object-storage-recommendation` with:
  - `recommended_setup_values.bucket_name`,
  - the `zettel-kasten-<profile-slug>-objets` naming rule,
  - `next_exact_commands.object_storage_setup_manual`,
  - `next_exact_commands.object_storage_dry_run`,
  - R2 setup field guidance when Cloudflare R2 is recommended.
- Added R2 field guidance for:
  - bucket name,
  - Location,
  - Jurisdiction,
  - Default storage class,
  - Public access,
  - API token permission,
  - bucket scope,
  - TTL / expiration,
  - client IP filtering,
  - secret handling and `credential-ref-plan` bridging.

## Safety

- The new setup guide is read-only.
- It does not open Cloudflare, create buckets, create API tokens, call provider
  APIs, check live pricing, check bucket availability, upload/download objects,
  read object bytes, read secrets, write files, create presigned URLs, or start
  OAuth.
- CLI JSON continues to avoid provider URLs, local absolute paths, usernames,
  emails, tokens, and secret values.

## Verification

- Added CLI coverage for surfaced bucket names, exact next commands, R2
  setup-field guidance, and the new beginner manual topic.
- Added public documentation and capability matrix coverage.
