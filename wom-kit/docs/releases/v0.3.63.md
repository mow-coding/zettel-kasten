# v0.3.63 - IMAP Header Scan Receipt Audit

Date: 2026-06-16

v0.3.63 adds a non-secret audit checkpoint for the IMAP header metadata scan
execution receipt introduced in v0.3.62.

## Added

- Added `archive imap-mailbox-header-scan-receipt-audit <archive-root>
  --execution-receipt <archive-relative-json> --dry-run|--approve --format json`.
- Added aliases `archive imap-header-scan-receipt-audit` and
  `archive mailbox-header-scan-audit`.
- The new command reads an existing receipt from
  `receipts/imap/adapter-executions/`, verifies it is an
  `imap_mailbox_header_metadata_scan` receipt, checks candidate refs are opaque
  `imap-candidate:<sha256>` refs, and checks sensitive redaction flags.
- Approved mode writes one non-secret audit receipt under
  `receipts/imap/adapter-execution-audits/`.

## Safety

- The audit command opens no IMAP connection, logs into nothing, selects no
  mailbox, searches no mailbox, reads no headers/bodies/attachments, reads no
  environment variables, opens no OS keyring/password manager, starts no OAuth,
  and calls no providers.
- The command does not echo the original execution receipt path, candidate
  refs, username/password values, environment variable names, email addresses,
  raw UIDs, Message-ID values, subjects, senders, recipients, headers, bodies,
  attachment names, provider URLs, local absolute paths, tokens, or secret
  values.

## Verification

- Added CLI coverage for dry-run, approved audit receipt writing, and blocked
  redaction failures.
- Added documentation matrix coverage for the new audit checkpoint.
