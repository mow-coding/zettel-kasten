# v0.3.233 - Abstract Freshness Evidence

v0.3.233 closes the next gap after explicit abstract publication: WOM can now
tell whether a canonical zet's current abstract and current body still match a
retained human-review event. The change is additive and requires no archive
migration.

## What Changed

- Approved mint and legacy promotion receipts add a strict, text-free
  `abstract_review_basis` with abstract and canonical-body hashes.
- CLI `archive abstract-freshness <archive-root> --dry-run` and MCP
  `abstract_freshness` classify canonical zets as `fresh`, `stale`,
  `unverified`, `missing`, `unreadable`, or policy `excluded`.
- The scanner recognizes new receipt evidence, retained v0.3.232 mint snapshots
  or promotion sources, and existing reviewed abstract-backfill receipts.
- Edge-only or other frontmatter-only edits remain fresh when the abstract and
  body are unchanged. Body or abstract edits create a bounded human review row.
- AI start-here and shipped archive-agent instructions now place the check
  after first-read readiness and before the complete private catalog pass.

## Upgrade And Compatibility

No old zet or receipt is rewritten. Existing archives can run:

```text
archive abstract-freshness <archive-root> --dry-run --progress --format json
```

Old material without reconstructable evidence is `unverified`; it is not
called stale or invalid. Review attention rows manually, then use an ordinary
reviewed revision or the existing abstract-backfill flow. The command never
grants write authority.

## Scale, Safety, And Privacy

- Receipts are indexed once and canonical zets are scanned once, with an
  `O(canonical_zets + receipt_files + receipt_items)` contract.
- Receipt, receipt-count, and zet-size reads are bounded.
- Output contains no title, abstract text, body text, hash value, receipt path,
  reviewer id, provider URL, absolute local path, or secret.
- No provider, model, credential store, objet byte, database, or generated map
  is accessed or written.
- `fresh` proves only an exact local hash-pair match to retained review
  evidence, not truth, completeness, usefulness, or model consumption.
