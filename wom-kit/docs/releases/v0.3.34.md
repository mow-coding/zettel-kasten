# v0.3.34 - Derived Text Coverage Gate

Date: 2026-06-15

v0.3.34 adds the first agent-facing derived-text coverage layer.

## Added

- CLI `archive derive-text coverage --dry-run`.
- CLI alias `archive derive-text-coverage --dry-run`.
- CLI `archive derive-text toolchain --extension <ext> --dry-run`.
- CLI alias `archive derive-text-toolchain --extension <ext> --dry-run`.
- CLI `archive derive-text agent-contract --dry-run`.
- CLI alias `archive derive-text-agent-contract --dry-run`.
- `wom-kit/docs/derived-text-coverage-and-toolchain.md`.

## Why

Derived text is not just a capture command. It is a coverage obligation:

```text
textual objet -> derived text, unless a blocker is recorded
```

The new gate helps agents avoid claiming "done" just because one file family,
such as PDFs, was processed.

## What Coverage Checks

The coverage gate compares:

```text
objects/manifests/files.jsonl
objects/manifests/derived-text.jsonl
```

It classifies textual candidates by extension/MIME and reports:

- `covered`,
- `missing_derived_text`,
- `needs_password_or_encrypted`,
- `not_textual_or_unknown`.

The command returns `ok: false` while textual candidates are missing derived
text.

## Toolchain Recommendations

The toolchain command recommends routes for:

- plain text and markup,
- `.docx`, `.xlsx`, `.pptx`,
- legacy `.doc`, `.xls`, `.ppt`,
- `.hwp` and `.hwpx`,
- PDFs,
- image scans,
- audio.

It does not run the tools.

## Privacy Boundary

The new commands do not:

- read source bytes,
- echo source filenames,
- echo local absolute paths,
- echo provider URLs,
- echo source bodies,
- run OCR,
- run parsers,
- run ASR,
- call LLM vision,
- call providers,
- write derived text,
- write receipts,
- draft zets,
- mint zets.

## References
