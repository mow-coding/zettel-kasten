# v0.3.222 - Fast AI Start-Here And Honest Progress

v0.3.222 makes archive entry fast by default while keeping complete Doctor
inspection explicit. It is additive and requires no archive migration.

## What Changed

- `ai-start-here` now defaults to bounded identity, authority, entrypoint, and
  operational-context inspection.
- Added `--full-doctor` for the complete zet/receipt/archive health scan.
- Added explicit inspection mode, Doctor checked state, and claim boundary.
- Corrected safety reporting so full Doctor records observed zet-body, local
  objet-byte, and archive-text secret-pattern reads.
- Suppressed unchanged compact stage/count output for 30 seconds while keeping
  count-bearing heartbeat.
- Added work unit, stage elapsed time, processed rate, and ETA to counted
  progress. `mint-receipts` means mint receipt files.

## Upgrade Guidance

Use the normal quick entry map:

```text
archive ai-start-here <archive-root> --dry-run --progress --format json
```

Add `--full-doctor` only when complete archive diagnostics are required. Any
automation that previously used `ai-start-here` as an implicit Doctor must add
that flag.

## Privacy And Write Boundary

Quick mode reads no zet body or objet byte. Full mode can read them for local
validation and says so. Neither mode accesses a credential store/provider or
writes archive state. Explicit `--output` remains private local scratch only.

Progress emits no private archive values. Detailed Doctor progress logs remain
private local diagnostics and must not be committed by default.
