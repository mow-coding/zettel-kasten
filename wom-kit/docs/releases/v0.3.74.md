# v0.3.74 - R2 Locale And S3 Credential Walkthrough

Date: 2026-06-16

v0.3.74 responds to first-use Cloudflare R2 setup friction: provider docs and
WOM v0.3.64 had the right general field recommendations, but a translated
dashboard and the post-creation credential screen still left a beginner asking
where the token lives and which value an S3 client actually needs.

## Added

- Extended `archive object-storage --provider cloudflare-r2 --dry-run` and
  `archive object-storage-recommendation --scenario ... --dry-run` provider
  setup guidance with:
  - English/Korean label hints for R2 bucket fields,
  - location-based navigation hints for R2 bucket creation,
  - location-based navigation hints for R2 API token creation from the R2
    account/overview area,
  - Korean labels for permission, bucket scope, TTL, and client IP filtering,
  - explicit S3 credential-use notes after token creation.
- Extended `archive beginner-setup-manual --topic object_storage_setup_manual
  --dry-run` with the same Cloudflare R2 walkthrough data.
- Configured the MCP stdio entrypoint to use UTF-8 so localized guidance labels
  are safe on Windows pipes.
- Updated the beginner setup manual, README status, capability matrix, and
  changelog.

## Clarified

- The R2 token screen is reached from the R2 account/overview area, not from one
  individual bucket's settings page.
- The beginner default permission for a reviewed private objet bucket remains
  `Object Read & Write` / `개체 읽기 및 쓰기`.
- The bucket scope should be the specific WOM objet bucket, not every bucket in
  the account.
- After token creation, S3-compatible object access uses the Access Key ID plus
  Secret Access Key pair. The separate Token value should not be pasted into
  chat or stored in WOM unless a future non-S3 API-token flow explicitly asks
  for it.

## Safety

- The commands remain read-only dry-run guidance.
- They do not open Cloudflare, create buckets, create API tokens, call provider
  APIs, check live pricing, check bucket availability, upload/download objects,
  read object bytes, read secrets, write files, create presigned URLs, or start
  OAuth.
- CLI JSON continues to avoid provider URLs, local absolute paths, usernames,
  emails, tokens, exact credential refs, bucket account IDs, and secret values.

## Verification

- Added CLI tests for R2 locale hints, token creation location notes, Korean
  field labels, and S3 credential-pair guidance.
- Re-ran focused MCP object-storage tests to verify localized guidance can flow
  through stdio without Windows encoding failures.
- Re-ran public release hygiene gates before release.
