# v0.3.230 - Digest-Bound Content-Change Review

v0.3.230 gives a human a complete, content-free review route when reconcile
finds a real or uncertain content change, then binds any approval to the exact
bytes and receipt refs that were reviewed. It requires no archive migration and
runs no archive write automatically.

## What Changed

- `remint-reconcile` and `retire-draft-reconcile` now return an ordered
  `human_review_plan` and `review_plan_sha256` on `content_change`.
- The plan identifies local evidence by archive-relative role, path, presence,
  and recorded/current SHA-256. It reports only changed field or ref names, not
  document bodies, titles, frontmatter values, local absolute paths, or secrets.
- It provides separate next actions for `intentional_change`,
  `unintentional_change`, and `uncertain`, plus independent stop conditions.
- A content-change approval now requires `--reviewed-plan-sha256` together with
  `--reviewed-by` and `--content-changed-ack`.
- Approval recomputes the plan before any write. A changed canonical, snapshot,
  source, mint receipt, or retire receipt invalidates the old digest and stops
  the run.
- Successful approval records the reviewed digest in reconcile provenance and
  the immutable audit receipt. Generated BOM commands retain `--strip-bom`.

## Upgrade And Compatibility

No migration or full-Doctor rerun is required. Restart the operator process,
then rerun only the affected reconcile dry-run. If it reports
`content_change`, review the local evidence and use the exact command under
`human_review_plan.commands.approve_if_intentional` only after a named human
accepts every change.

The new flag is required only for `content_change` approval. Existing
`format_drift` approval remains compatible. Old content-change approval
commands without the reviewed digest now fail closed before writing.

## Safety And Privacy

- Dry-runs remain zero-write; no provider or credential surface is involved.
- The digest proves review/apply freshness, not correctness. Human judgment is
  still the authority for intentional content changes.
- One reconcile target never approves another. `uncertain` and unexplained
  changes always stop.
- Diagnostic-only JSON keeps the new plan while omitting document content and
  frontmatter values.
