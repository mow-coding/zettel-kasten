# v0.2.22 Source Intake Planner

Status: public pre-release patch

Date: 2026-05-25

## Summary

`v0.2.22` adds a dry-run-only source intake planner for classifying source/objet references before draft creation.

It helps AI runtimes decide whether a presentation, document, image, provider item, manifested objet, local file, source map item, or AI artifact has a safe reference that an inbox draft zet can cite.

## Added

- `archive source-intake <archive-root> --dry-run --format json`
- Locator modes for:
  - local file metadata,
  - source map item id,
  - source-relative path,
  - `objet:sha256:...`,
  - technical `object_id`,
  - provider object reference,
  - AI artifact reference.
- Stable JSON output with:
  - `source_refs_for_draft`,
  - `objet_status`,
  - `objet_ref`,
  - object storage context,
  - content access flags,
  - blockers, warnings, and next safe actions.
- Read-only MCP `source_intake_plan`.
- Regression coverage for source-relative path intake, traversal rejection, provider ref rejection, and MCP redaction enforcement.

## Compatibility

No private archive migration is required.

Existing source maps and object manifests remain valid.

`create-draft --dry-run` can already accept the returned refs through `--source-ref`, and mint dry-runs preserve those refs in receipt previews.

## Non-Goals

This release does not:

- read file bodies,
- calculate full hashes,
- copy files,
- upload files,
- import source content,
- OCR,
- transcribe,
- parse/extract document bodies,
- call provider APIs,
- create drafts automatically,
- mint,
- sync providers,
- add MCP apply/capture/upload/sync/provider API tools.

## Review Note

Pre-release review found and fixed a `--relative-path` crash caused by a missing internal path-normalization import. `v0.2.22` includes regression tests for that locator mode before release.
