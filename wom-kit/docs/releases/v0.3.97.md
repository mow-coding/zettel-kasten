# v0.3.97 - Saved View Recommendation Plan

Date: 2026-06-17

## Added

- CLI `archive view-recommendation-plan <archive-root> --dry-run`.
- MCP `view_recommendation_plan`.
- Public docs for saved view recommendations from indexed navigation facets.

## Why

Phase 3 real-use feedback showed that default AI navigation views can go empty
when the archive's real facet axes differ from the starter template.

`view-health` can diagnose the drift. This release adds the next read-only
step: suggest candidate saved-view filters from actual indexed navigation
facets before any human edits `views/*.yml`.

## Behavior

- Reuses `view-health` saved view counts, facet role classification, and indexed
  facet distributions.
- Proposes candidate single-facet saved views from likely navigation axes such
  as `subject`, `institution`, `record_type`, `source_category`, and `domain`.
- Includes suggested `view.ai.<axis>.<value>` ids, suggested
  `facets.<key>: <value>` filters, match counts, and whether the key/value pair
  is already used by an existing saved view filter.

## Safety

The new path writes nothing, rebuilds no index, rewrites no zettel facets, reads
no zettel bodies, reads no object bytes, calls no providers, creates no provider
or presigned URLs, and echoes no zettel titles, absolute local paths, provider
URLs, account ids, emails, tokens, or secret values.

The recommendation remains a proposal for human review, not an automatic
taxonomy decision.
