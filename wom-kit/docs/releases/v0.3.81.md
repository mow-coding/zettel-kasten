# v0.3.81 - Connection Evidence Fixture Parser

Date: 2026-06-16

Status: v0.3.81 read-only sanitized fixture parser checkpoint

v0.3.81 continues the #10 connection-import feedback path by adding the first
parser-shaped checkpoint for sanitized Notion connection evidence fixtures. It
does not parse real client exports.

## Added

- CLI `archive connection-evidence-parse-fixture --evidence <archive-relative-json> --source notion --dry-run`.
- CLI aliases `connection-evidence-parser-fixture` and
  `notion-connection-evidence-parser-fixture`.
- MCP `connection_evidence_parse_fixture`.
- Fake archive fixture `workbench/connection-evidence.sample.json`.
- Public doc `docs/connection-evidence-fixture-parser.md`.

## Behavior

The fixture parser reads a sanitized archive-internal JSON fixture and emits
candidate edge previews for:

- relation properties,
- synced block references,
- database view/filter snapshots,
- internal page hyperlinks,
- page mentions,
- comment context,
- objet embeds.

The fake archive sample currently has 8 fixture records and produces 9 candidate
edge previews because one database-view fixture record has two result refs.

## Safety Boundary

This release still does not implement a real Notion export parser or edge
writer.

The workflow remains read-only: no Notion call, OAuth, real export read, comment
read, media download, candidate record write, zet write, edge write, receipt
write, object manifest update, provider URL, local path, raw export path, page
title, comment body, account id, email, token, or secret value is produced.
