# v0.3.76 - Large Media Export Trap

Date: 2026-06-16

v0.3.76 tightens the external export safety gate for a real first-use failure
mode: the human only wants text or database review, but a broad workspace export
also pulls uploaded files, attachments, images, audio, or video into a huge local
download.

## Added

- `archive external-export-plan` now returns `large_media_export_trap`.
- The trap output names the broad workspace/database export risk, whether the
  first export should stop, and whether a text-only or targeted pass is required
  first.
- The planner now returns exact safe first-pass command shapes:
  - `rerun_text_only_first`
  - `rerun_targeted_pages_first`
- Notion guidance now includes a large-workspace fallback: split an oversized or
  failing workspace export into smaller top-level page or database batches before
  retrying media-heavy work.

## Safety

- The command remains dry-run/read-only.
- It starts no provider export, calls no provider APIs, starts no OAuth, reads no
  files or media bytes, downloads no attachments, writes no archive receipts, and
  echoes no provider URLs, local paths, filenames, account ids, emails, tokens,
  or secret values.
- It does not implement Notion API sync, Google Drive API sync, Takeout
  automation, attachment download, object upload, media deduplication, or
  automatic import.

## Verification

- Re-ran release readiness, including public link, Korean product-language, and
