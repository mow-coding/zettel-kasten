# v0.3.36 - Derived Text Doctor Tool Hints

Date: 2026-06-15

v0.3.36 improves the read-only derived-text toolchain doctor for Windows and
other local environments where an extractor is installed but not on `PATH`.

## Added

- CLI `archive derive-text doctor --tool-hints <json> --dry-run`.
- CLI alias `archive derive-text-doctor --tool-hints <json> --dry-run`.
- Executable hint support for:
  - `soffice`,
  - `libreoffice`,
  - `tesseract`,
  - `hwp5txt`.

Example hint file shape:

```json
{
  "schema": "wom-kit/derived-text-tool-hints/v0.1",
  "executables": {
    "soffice": "<local-soffice-path>",
    "tesseract": "<local-tesseract-path>"
  }
}
```

## Fixed

- The doctor can now mark a PATH-missing local executable as available when a
  user supplies an existing local path hint.
- A grouped tool such as LibreOffice is no longer listed as missing when one of
  its accepted executable probes is available.

## Privacy Boundary

The hint file may contain private local paths. The doctor output does not echo:

- the hint file path,
- executable hint paths,
- Python import paths,
- local absolute paths,
- source filenames,
- source bodies,
- provider URLs,
- usernames,
- secret values.

It reports only tool names, probe labels, booleans, route families, and missing
readiness categories.

## Closed Actions

The doctor still does not:

- install tools,
- execute hinted tools,
- read source bytes,
- run OCR,
- run parsers,
- run ASR,
- call LLM vision,
- call providers,
- write derived text,
- write receipts,
- draft zets,
- mint zets.
