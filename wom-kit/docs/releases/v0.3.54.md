# v0.3.54 - IMAP Adapter Manifest Status in Readiness

Date: 2026-06-16

v0.3.54 lets the IMAP adapter readiness planner check whether a reviewed
non-secret adapter manifest exists and still matches the safe manifest
contract.

The release still does not connect to mail providers. It only reads local
archive-relative manifest metadata.

## Added

- Optional CLI `--adapter-id <id>` for
  `archive imap-mailbox-adapter-readiness-plan --dry-run`.
- Optional MCP `adapter_id` for `imap_mailbox_adapter_readiness_plan`.
- `adapter_manifest_summary.status` with:
  - `not_checked`
  - `missing`
  - `present_and_schema_valid`
  - `invalid`
  - `blocked`
- Schema validation against `imap-mailbox-adapter-manifest.schema.json` for a
  manifest under `config/imap-adapters/`.

## Verify

```bash
archive imap-mailbox-adapter-readiness-plan <archive-root> \
  --source-id imap:gmail-personal \
  --adapter-id local-imap \
  --provider gmail \
  --account-ref imap:account:gmail-personal \
  --username-ref env:WOM_GMAIL_USERNAME \
  --auth-mode oauth_token_ref \
  --oauth-token-ref keyring:gmail-oauth \
  --mailbox-ref imap:mailbox:inbox \
  --credential-id cred:gmail-mail-access \
  --operation header_metadata_scan \
  --dry-run \
  --format json
```

The JSON should include:

```text
lifecycle_action: imap_mailbox_adapter_readiness_plan
adapter_manifest_summary.status: present_and_schema_valid
closed_actions.imap_connection_opened: false
closed_actions.credential_value_read: false
```

If the manifest has not been written yet, the status is `missing`; that is a
local readiness signal, not permission to read a mailbox.

## Safety

The check remains read-only. It:

- writes no files,
- opens no IMAP connection,
- attempts no login,
- selects or searches no mailbox,
- lists no candidate messages,
- reads no IMAP UIDs or Message-ID values,
- reads no headers, bodies, or attachments,
- creates no derived text,
- retrieves no secrets,
- starts no OAuth,
- calls no providers,
- echoes no email addresses, username values, exact account refs, exact
  credential refs, exact mailbox refs, IMAP host values, provider URLs, message
  ids, subjects, sender or recipient values, attachment names, approval receipt
  paths, selection receipt paths, schema validation issue values, local absolute
  paths, tokens, or secret values.

No archive migration is required.
