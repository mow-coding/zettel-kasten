# v0.2.36 Foreign Block Quarantine Decision Review Index

Release date: 2026-05-25

## Summary

v0.2.36 adds a read-only review index for recorded foreign block quarantine decisions.

The new `quarantine-decision-review` command lets an operator or AI runtime list recorded quarantine decisions, validate their receipts, and compare them against the current quarantine case and original quarantine receipt before any later human review path.

## Added

- `archive quarantine-decision-review <archive-root> --format json`
- Optional filters:
  - `--case-id <safe-case-id>`
  - `--decision keep_quarantined|reject_and_keep_record|eligible_for_attestation_review|needs_more_review|all`
  - `--include-receipts`
- Read-only MCP `foreign_block_quarantine_decision_review_index`.
- New documentation: `wom-kit/docs/foreign-block-quarantine-decision-review-index.md`.

## Output Boundary

The command returns a stable JSON index with:

- `lifecycle_action: foreign_block_quarantine_decision_review_index`,
- `dry_run: true`,
- `trust_state: untrusted_foreign`,
- `review_status: indexed_not_modified`,
- decision summaries,
- displayed and total decision counts,
- a separate case-level projection,
- optional receipt summaries,
- blockers and warnings,
- `would_change: []`.

It writes nothing.

`--decision` filters only the displayed decision summaries. It does not relax consistency validation. Top-level `ok` reflects every discovered decision record, decision receipt, current quarantine case, and original quarantine receipt.

When receipt summaries are included, boolean field names use direct semantics. For example, `trust_granted: false` means trust was not granted.

## Safety

The index validates:

- decision record shape,
- safe case id, decision, reviewer, timestamp, and review-note summary fields,
- matching decision receipt presence and content,
- current quarantine case presence,
- current original quarantine receipt presence,
- expected source case and receipt hashes when recorded.

Malformed, contradictory, unsafe, missing, or stale records produce blockers or warnings without exposing raw unsafe values.

## Non-Goals

v0.2.36 does not implement:

- quarantine decision acceptance,
- quarantine decision trust or apply,
- foreign block import,
- foreign block trust,
- foreign attestation writes,
- minting from foreign blocks,
- anchoring,
- delegation,
- real ZET transport,
- signing,
- payment,
- staking,
- consensus,
- blockchain,
- provider sync,
- OCR,
- LLM classification,
- full-auto execution.

MCP remains read-only for this workflow and exposes no quarantine decision review write/apply/accept/import/trust/attest tool.
