# v0.3.204 - zet abstract and live catalog

v0.3.204 begins WOM's archive-wide AI reading layer. It is additive and requires
no archive migration.

## What Changed

- Added optional `frontmatter.abstract`, bounded to 360 characters.
- Added `archive create-draft --abstract <text>` with compact-text and privacy
  validation.
- Added read-only `archive zet-catalog <archive-root> --dry-run --format
  json`, with `zettel-catalog` and `abstract-catalog` aliases.
- Catalog pages report total, returned, remaining, cursor, next cursor,
  `complete`, and `truncated` explicitly.
- Catalog continuation can bind to `--expected-snapshot-id`; local changes
  block continuation with `catalog_snapshot_changed` and no items.
- Every returned non-redacted zet includes its safe compact abstract state,
  facets, complete frontmatter edge projection, and tie summary.

## Compatibility

Existing zets remain valid. The catalog distinguishes explicit `abstract`,
legacy compatibility fields, missing abstracts, redacted zets, and unreadable
frontmatter. It does not write generated abstracts into old zets.

## Safety And Scope

The catalog reads local frontmatter directly, requires no generated index,
reads and echoes no zet body text, reads no objet bytes or secrets, calls no
providers, and writes no files.

The snapshot hashes path/size/mtime/frontmatter projection evidence but not body
content. It detects ordinary catalog changes between pages; it is not a body
integrity receipt.

This release does not add MCP catalog access, change the MCP `read_zettel`
contract, create a WOM goal/loop engine, add a vector database, rank nodes, or
make a generated map authoritative.
