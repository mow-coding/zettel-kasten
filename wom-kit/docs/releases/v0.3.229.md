# v0.3.229 - Executable BOM Reconcile Guidance

v0.3.229 replaces the unresolved `<id>` in a `zettel_has_bom` Doctor suggestion
with the canonical zet's validated frontmatter id. It is additive and requires
no archive migration.

## What Changed

- Doctor reads the id from the canonical frontmatter it has already parsed.
- A safe id produces a complete command:

  ```text
  archive remint-reconcile <archive-root> --zettel-id <actual-id> --dry-run --strip-bom --diagnostic-only --format json
  ```

- The same command is retained by `doctor_findings.items` and the deduplicated
  `doctor_findings.suggested_commands` result.
- A missing or unsafe id leaves the finding and hint intact but omits the
  command. WOM-kit does not infer identity from the filename and does not emit
  an unresolved or shell-unsafe selector.

## Review Sequence

The Doctor suggestion is a content-redacted, zero-write classifier. For a BOM
finding, run it first. Stop if it reports a blocker, `content_change`, a body
change, or a required content-change acknowledgment.

When it reports `format_drift`, `bom_stripped: true`, `body_changed: false`, and
no acknowledgment requirement, repeat the dry-run without `--diagnostic-only`
and review the current content/frontmatter. Only that review-visible result can
support an approval with `--reviewed-by` and `--strip-bom`.

A separate `retire-draft-reconcile --dry-run` remains independent. Stop on
`blocked` or `unclassified`. Review every `ref_reports` row before a
`format_drift_ready_for_review` approval. Treat `needs_content_change_review` as
a human content-review gate; use `--content-changed-ack` only when a named human
accepts the changed bytes. Success on either reconcile never authorizes the
other.

## Safety And Privacy

- Doctor, runtime-context, and AI start-here remain read-only.
- This patch changes no reconcile classifier, approval gate, receipt schema,
  archive write path, provider behavior, or credential access.
- The id grammar contains only shell-safe alphanumeric, underscore, and hyphen
  characters.
- No filename inference, local absolute path, provider URL, zet body, or secret
  enters the new command.
