# v0.3.86 - Object Storage Upload Evidence Audit

Date: 2026-06-17

Status: v0.3.86 read-only object-storage upload evidence audit checkpoint

v0.3.86 adds the read-only audit step after v0.3.85's local upload evidence
registration.

## Added

- `archive object-storage-upload-evidence-audit --receipt <json> --dry-run`.
- Aliases:

```text
object-storage-external-upload-evidence-audit
objet-storage-upload-evidence-audit
```

- Documentation: `wom-kit/docs/object-storage-upload-evidence-audit.md`.

## Behavior

The audit checks:

- upload evidence receipt schema and lifecycle action,
- archive id consistency,
- safe provider and store labels,
- non-secret privacy guards,
- closed no-provider action flags,
- `manifest_update_completed`,
- linked `object_storage` manifest locations,
- `declared_uploaded` availability,
- sha256 content-addressed key hints,
- receipt summary count vs linked manifest location count.

## Safety Boundary

This release remains read-only. It writes no audit receipt, reads no source
bytes, computes no local source-object hashes, calls no R2/S3/provider APIs,
performs no remote HEAD checks, uploads nothing, downloads nothing, syncs
nothing, creates no provider URLs, reads no secrets, drafts no zets, and mints
no zets.

The audit output also avoids echoing the receipt path, manifest path, object
ids, location records, provider URLs, bucket names, account ids, emails, tokens,
or secret values.
