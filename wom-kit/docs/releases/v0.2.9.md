# v0.2.9 Terminology Stabilization

`v0.2.9` stabilizes the product language after the first real minting implementation.

The rule is:

```text
preferred public language = mint
legacy compatibility language = promote / promotion
```

## Changed

- New archives now default to `ai_write_policy.canonical_requires: human_minting`.
- Doctor accepts both `human_minting` and legacy `human_promotion` without warnings.
- Zettel rules may now define `minting_rules`.
- Mint dry-runs prefer `minting_rules` and fall back to legacy `promotion_rules`.
- User-facing docs now describe minting as the normal path.

## Compatibility

The following remain available:

- `archive promote`
- MCP `promotion_check`
- canonical zettel `promotion` frontmatter
- `promotion_rules`
- `receipts/promotion/*.promotion.json`

This release does not remove any v0.2 compatibility surface.

## Migration

No private archive migration is required.

Existing archives with:

```yaml
ai_write_policy:
  canonical_requires: human_promotion
```

remain valid.

New archives should use:

```yaml
ai_write_policy:
  canonical_requires: human_minting
```

## Verification

```bash
cd wom-kit
python -m unittest discover -s tests
python cli/archive.py doctor examples/fake-life-archive --strict
```

Expected result:

```text
tests pass
doctor reports 0 errors and 0 warnings
```
