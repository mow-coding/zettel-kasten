# v0.3.156 - Operator Envelope Class Fields

v0.3.156 adds the first command-output retrofit for AI operator class fields.

## What Changed

- Added top-level `status_class`, `input_provenance_class`, and
  `secret_signal_class` fields to:
  - `archive operation-status-taxonomy --dry-run`,
  - `archive input-provenance-taxonomy --dry-run`,
  - `archive secret-signal-taxonomy --dry-run`,
  - `archive ai-response-contract --dry-run`.
- Added nested `operator_envelope` with schema
  `wom-kit/operator-envelope-classes/v0.1`.
- Documented the initial class values:
  - valid read-only safety commands use `status_class: preview`,
  - their archive root is `input_provenance_class: caller_supplied`,
  - their public safety vocabulary is `secret_signal_class: concept_word`.

## Safety Boundary

The retrofit does not expand what the covered commands read or write. They keep
their existing privacy guards: no archive body text, sample values, providers,
network checks, local absolute paths, tokens, secret values, or writes.

## Still Future

- Extending class fields to more JSON-producing commands.
- Using more specific input provenance classes for receipt/index/manifest-backed
  inputs.
- Adding item-level class fields for batch commands with mixed outcomes.
