# v0.3.85 - Object Storage Upload Evidence

Date: 2026-06-17

Status: v0.3.85 approval-gated object-storage upload evidence checkpoint

v0.3.85 addresses the client feedback #11 gap between an external upload ledger
and WOM's local manifest/receipt evidence.

## Added

- `archive object-storage-upload-evidence --ledger <jsonl> --dry-run|--approve`.
- Aliases:

```text
object-storage-external-upload-evidence
objet-storage-upload-evidence
```

- Documentation: `wom-kit/docs/object-storage-upload-evidence.md`.

## Behavior

Dry-run mode:

- reads UTF-8 JSONL upload evidence ledgers,
- accepts successful `uploaded`, `verified`, `succeeded`, `already_present`, or
  `ok` rows,
- validates sha256 and optional byte-size fields,
- matches successful objects against `objects/manifests/files.jsonl`,
- previews `object_storage` manifest location additions,
- echoes no ledger paths, row values, object ids, source filenames, local paths,
  provider URLs, bucket names, account ids, emails, tokens, or secret values.

Approved mode:

- requires `--reviewed-by`,
- requires a safe `--store-ref`,
- writes one non-secret receipt under
  `receipts/providers/object-storage-upload-evidence/`,
- updates existing manifest rows with safe `object_storage` locations marked
  `declared_uploaded`,
- records `byte_verification_by_wom_kit: false` and
  `provider_confirmation_by_wom_kit: false`.

## Safety Boundary

This release does not implement a live object-storage upload adapter. It does
not read source bytes, compute local source-object hashes, call R2/S3/provider
APIs, check remote availability, upload, download, sync, create provider URLs,
open password managers or keyrings, read environment variables, retrieve
secrets, draft zets, or mint zets.
