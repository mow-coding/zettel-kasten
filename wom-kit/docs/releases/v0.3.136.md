# v0.3.136 - Notion Recover One-Command Wrapper

Date: 2026-06-22

## Summary

v0.3.136 turns the v0.3.134 live Notion location fetch adapter and the
v0.3.135 beginner guide into a beginner-facing command:

```text
archive notion-recover
```

The command removes the developer-style page-id, credential-ref, approval
receipt path, and multi-command copy/paste chain from the normal human path.

## Added

- `archive notion-recover`, with aliases `archive notion-location-recover` and
  `archive notion-recovery`.
- Auto-selection of the reviewed Notion tree fixture that still has
  missing-location requests.
- Local confirmation text that states the command reads location links only.
- Hidden local token prompt for interactive runs when no local process value is
  already available.
- Internal approval receipt handoff to the existing approval-gated live adapter.
- Sanitized fetch output plus a merge-preview handoff sentence:
  "Please tidy and merge the recovered Notion locations into the reviewed list."

## Safety Boundary

The wrapper preserves the existing local security boundary:

- human approval happens locally,
- the token stays outside chat,
- AI receives no secret value,
- approval receipt paths are not part of the user copy/paste flow,
- page titles, page bodies, comments, media bytes, signed file URLs, and raw
  provider responses remain outside this flow.

The lower-level crawl, approval, fetch, and merge-plan commands remain
available for power users and automation.

## Verification

- CLI tests cover dry-run auto-scope behavior.
- CLI tests cover approved approval->fetch->merge-preview chaining.
- Tests assert that token values, env names, receipt paths, provider URLs, raw
  refs, page titles, page bodies, and local absolute paths are not echoed.
- Documentation updates cover README links, the beginner setup manual, the
  capability matrix, and the public documentation map.
