# v0.3.65 - Version Pin Discovery From Archive Roots

Date: 2026-06-16

## Summary

v0.3.65 tightens the version truth-source flow. `archive version <root>
--format json` and runtime-context `wom_kit_version` can now find a
project-local installed-version pin when the inspected root is the archive
folder and the pin lives in the parent project folder.

## What Changed

- Added parent-project pin discovery for archive roots containing `archive.yml`.
- Added safe logical reporting for checked pin locations through
  `project_pin.checked_locations`.
- Added `project_pin.pin_root` so AI runtimes can distinguish
  `inspection_root` from `parent_of_archive`.
- Preserved existing pin behavior for `.zettel-kasten/source/installed-version.txt`.
- Normalized UTF-8 BOM-prefixed installed-version files.

## Safety

This release remains read-only:

- writes no files,
- calls no providers,
- reads no secrets, keyrings, vaults, mailboxes, or source documents,
- echoes no local absolute paths by default.

Use `--no-redact-local-paths` only for trusted local debugging.
