# v0.3.92 - Connection Edge Review Summary

Date: 2026-06-17

## Added

- `review_summary` output for read-only
  `archive connection-edge-intelligence-plan --dry-run`.
- Text output counters for human-review-required candidates,
  durable-write human approvals, and auto-writable candidates.
- Extra human review queue fields for ambiguity and parsimony signals.

## Why

Client feedback showed that `ambiguous_count: 0` could be read as "safe to
write", even when every candidate still needed human review before a durable
edge write.

This release keeps `ambiguous_count` narrow and adds explicit review counters
so an AI runtime and a human reviewer can see the difference between "not
ambiguous" and "approved for writing".

## Behavior

- `classification_summary.ambiguous_count` remains the semantic or
  medium-confidence ambiguity count.
- `classification_summary.human_review_required_count` reports candidate-level
  review flags.
- `review_summary.durable_write_human_approval_required_count` reports how many
  candidates still need human approval before any durable edge write.
- `review_summary.auto_writable_count` remains `0`.

## Safety

The updated path writes no candidate records, zets, edges, receipts, or object
manifests, calls no providers or LLMs, reads no body text or object bytes, and
echoes no provider URLs, local paths, account ids, emails, tokens, or secret
values.
