# Release v0.2.38

Date: 2026-05-25

## Summary

v0.2.38 adds a read-only Foreign Block Attestation Review Candidate Plan.

The new planner prepares a safe human-review candidate from a recorded quarantine decision only when that decision is `eligible_for_attestation_review` and its outcome is `prepare_attestation_review_candidate`.

## Added

- `archive attestation-review-candidate <archive-root> --case-id <safe-case-id> --dry-run --format json`
- Optional replay/context flags:
  - `--expected-decision eligible_for_attestation_review`
  - `--expected-outcome prepare_attestation_review_candidate`
  - `--prospective-attestor <safe-actor-id>`
  - `--review-scope identity|source_refs|header_hashes|prompt_boundary|full_human_review`
  - `--review-note <operator-note>`
- Read-only MCP `foreign_block_attestation_review_candidate_plan`
- Candidate output with safe evidence summaries, missing human checks, disallowed actions, and next safe actions.

## Safety

- Writes nothing.
- Keeps `trust_state: untrusted_foreign`.
- Keeps `candidate_status: planned_not_recorded` for valid candidates.
- Returns `candidate_status: blocked_not_planned` and `attestation_review_candidate: null` when blockers prevent planning.
- Keeps `attestation_status: not_created`.
- Keeps `would_change: []`.
- Does not read original foreign bodies, source documents, provider payloads, or objet bodies.
- Does not calculate new full hashes.
- Does not create trust, import, acceptance, attestation, signature, minting, sharing, provider sync, ZET transport, or full-auto behavior.

## Compatibility

No private archive migration is required.

Existing v0.2.37 quarantine decisions remain valid. The new planner only reads existing sanitized quarantine case, quarantine receipt, decision record, and decision receipt metadata.
