# v0.3.14 - Project Intake Unpack Queue

Release date: 2026-06-14

v0.3.14 adds the first practical "unpacking boxes" layer to the manual
project-intake flow.

## Added

- CLI `archive project-intake-unpack-queue --staged-folder <folder> --dry-run`.
- MCP `project_intake_unpack_queue` with the same read-only behavior.
- Opaque `item-0001` style refs for top-level staged items, plus coarse item
  kind, safe extension, and size-bucket hints.
- A next-human-turn prompt that asks which item ref the human wants to unpack
  next and what kind of action they intend: inspect, preserve, draft from,
  defer, mark as noise after review, split, or stop.

## Boundaries

- The queue exposes no staged entry names or local paths.
- It reads no file bodies, calculates no content hashes, and does not recurse
  into the staged tree.
- It does not classify items automatically, run source-intake, capture objets,
  create drafts, mint zets, call providers, upload, sync, or clean.
- The human must still map the opaque item ref to the local staged item and run
  the later per-item commands explicitly.

## Upgrade

No archive migration is required for v0.3.13 users.
