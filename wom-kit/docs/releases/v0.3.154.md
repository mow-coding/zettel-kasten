# v0.3.154 - Secret Signal Taxonomy

v0.3.154 adds a read-only taxonomy for secret and sensitive-signal handling.

## What Changed

- Added `archive secret-signal-taxonomy`.
- Added aliases `archive secret-taxonomy` and
  `archive sensitive-signal-taxonomy`.
- Added a machine-readable taxonomy for:
  - `concept_word`,
  - `safe_reference`,
  - `credential_reference`,
  - `secret_value_pattern`,
  - `private_locator`,
  - `account_identifier`,
  - `unknown_sensitive_context`.
- Marked concept words, safe references, and credential references as not
  blocking by themselves.
- Marked secret-like values, private locators, account identifiers, and unknown
  sensitive context as blocking classes for public outputs.

## Safety Boundary

The command:

- reads no archive body text,
- accepts no sample values,
- echoes no sample values,
- calls no providers,
- checks no network,
- echoes no local absolute paths, tokens, or secret values,
- writes nothing.

## Still Future
