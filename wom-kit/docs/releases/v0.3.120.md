# v0.3.120 - Retire/Validate Edge Receipt Index

Date: 2026-06-20

## What Changed

v0.3.119 moved large mint and retired-draft cleanup runs away from fragile
per-item shell loops. v0.3.120 closes the next real-use performance gap:
per-item edge receipt rescans during retired-draft cleanup and validation.

This release adds a source-zettel-path edge receipt index for target SHA
evolution checks. The index lets WOM-kit answer this question quickly:

```text
Did this canonical zet's SHA change only because approved zettel-edge receipts
were written after minting or retiring?
```

## Batch Retire Impact

`archive retire-draft-batch --plan <json> --approve` now:

- builds one edge receipt index for the batch,
- reuses that index for every item whose canonical target SHA evolved through
  approved edge writes,
- reuses the verified retirement plan for approved writes,
- performs O(1) source, canonical target, mint receipt, and snapshot SHA replay
  checks before deleting the inbox draft,
- still writes the existing per-item retired-draft receipts and one batch
  receipt.

This keeps the v0.3.119 fork-safety work, but removes the next expensive
per-item rescan layer for post-edge canonical zets.

## Validate Impact

`archive validate` and `archive doctor` now use a lazy Doctor-level edge receipt cache
for mint and retired-draft target SHA evolution checks.

That means multiple mint receipts and retired-draft receipts can share one edge
receipt scan instead of each receipt rereading `receipts/edges/*.zettel-edge.json`.

## Compatibility

No archive migration is required.

Existing commands remain valid:

- `mint-zet`
- `mint-zet-batch`
- `retire-draft`
- `retire-draft-batch`
- `doctor`
- `validate`

## Safety Boundary

This release does not:

- bypass human `--reviewed-by` approval,
- remove SHA/path replay checks before deleting drafts,
- accept arbitrary target SHA drift,
- skip schema validation,
- add `validate --since`,
- add progress logging or ETA output,
- add MCP write tools,
- add a worker, queue, daemon, or scheduler,
- call providers,
- read unrelated source files,
- echo zettel body text.

The optimization is a local cache/index improvement only. It keeps the existing
receipt evidence model.
