# v0.3.50 - IMAP Mailbox Adapter Audit Plan

Date: 2026-06-16

v0.3.50 adds a read-only preview for the non-secret audit receipt a future IMAP
adapter should write after an approved mailbox operation.

The release still does not connect to mail providers. It only defines the safe
receipt shape before any live IMAP adapter exists.

## Added

- `archive imap-mailbox-adapter-audit-plan --dry-run`.
- Alias `archive imap-mailbox-adapter-audit --dry-run`.
- Alias `archive mailbox-adapter-audit-plan --dry-run`.
- MCP tool `imap_mailbox_adapter_audit_plan`.
- New documentation at `wom-kit/docs/imap-mailbox-adapter-audit-plan.md`.
- Capability matrix and public documentation map entries for the IMAP adapter
  audit preview boundary.

## Verify

```bash
archive imap-mailbox-adapter-audit-plan <archive-root> \
  --adapter-id local-imap \
  --source-id imap:gmail-personal \
  --provider gmail \
  --account-ref imap:account:gmail-personal \
  --username-ref env:WOM_GMAIL_USERNAME \
  --auth-mode oauth_token_ref \
  --oauth-token-ref keyring:gmail-oauth \
  --mailbox-ref imap:mailbox:inbox \
  --credential-id cred:gmail-mail-access \
  --operation header_metadata_scan \
  --selection-rule newest_first \
  --selector-id mail-selection:recent-inbox \
  --result-status not_run \
  --dry-run \
  --format json
```

The JSON should include `lifecycle_action: imap_mailbox_adapter_audit_plan`,
`audit_state: preview_ready`, and a proposed path under
`receipts/imap/adapter-audits/`.

## Safety

The planner remains read-only. It:

- writes no audit receipts, selection receipts, approval receipts, or adapter
  manifests,
- executes no live adapters,
- opens no IMAP connection,
- attempts no login,
- selects no mailbox,
- searches no mailbox,
- lists no candidate messages,
- reads no IMAP UIDs or Message-ID values,
- reads no headers, bodies, or attachments,
- creates no derived text,
- retrieves no secrets,
- starts no OAuth,
- calls no providers,
- echoes no email addresses, username values, exact account refs, exact
  credential refs, exact mailbox refs, IMAP host values, provider URLs, message
  ids, subjects, sender or recipient values, attachment names, approval receipt
  paths, selection receipt paths, local absolute paths, tokens, or secret values.

No archive migration is required.
