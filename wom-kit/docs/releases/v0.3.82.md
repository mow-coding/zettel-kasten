# v0.3.82 - Zettel Edge Write

Date: 2026-06-17

Status: v0.3.82 approval-gated zettel edge write checkpoint

v0.3.82 continues the #10 connection-import feedback path by adding the first
approval-gated writer for one durable WOM typed edge. It turns a human-reviewed
`from zet + target + edge type` decision into a source zettel frontmatter edge
and a non-secret receipt.

## Added

- CLI `archive zettel-edge --from-zettel <zet> --target <zet-or-objet> --edge-type <type> --dry-run|--approve`.
- CLI aliases `link-zettel-edge` and `write-zettel-edge`.
- Receipt writes under `receipts/edges/*.zettel-edge.json`.
- Public doc `docs/zettel-edge-write.md`.

## Behavior

The command writes exactly one reviewed edge from one source zet to one verified
target:

- existing `zet_<id>` target,
- manifested `sha256:<64hex>` object target,
- manifested `objet:sha256:<64hex>` object target.

`--edge-type` must already be defined in `zettel-kasten/types.yml`. Duplicate
`type + target` edges on the same source zettel are blocked.

## Safety Boundary

This release still does not implement bulk candidate import, real Notion export
parsing, provider calls, or an MCP write tool.

The workflow remains local and approval-gated: no Notion call, OAuth, real
export read, comment read, media download, candidate record write, object
manifest update, provider URL, local path, raw export path, page title, comment
body, account id, email, token, or secret value is produced. The command does
not echo zettel body text or zettel titles.
