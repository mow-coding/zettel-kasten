# v0.3.200 - Object-storage manifest reconcile receipt schema

v0.3.200 follows v0.3.199 by hardening the audit receipt that
`object-storage-wom-location-reconcile --approve` writes.

No archive migration is required.

## What Changed

The object-storage manifest reconcile audit receipt now has a public JSON schema:

```text
wom-kit/schemas/object-storage-manifest-reconcile-receipt.schema.json
```

`archive doctor` now validates receipts under:

```text
receipts/providers/object-storage-manifest-reconciles/
```

Doctor checks:

- schema value
- action
- `dry_run: false`
- `reviewed_by`
- archive-relative `receipt_path`
- archive-relative updated execution receipt refs
- positive planned/applied update counts
- privacy guards showing that object ids, object keys, provider URLs, bucket
  names, credential refs, and local absolute paths were not echoed

## Safety

The command behavior from v0.3.199 is unchanged:

- no provider calls
- no credential reads
- no object byte reads
- no uploads or downloads
- no remote availability checks
- no object id or remote key echo in candidate output

This release only fixes the receipt contract around the already-shipped repair
surface.
