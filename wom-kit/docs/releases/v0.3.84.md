# v0.3.84 - AI Response Concept Guide CLI

Date: 2026-06-17

Status: v0.3.84 read-only AI response concept guide CLI checkpoint

v0.3.84 continues the client feedback #8 path by turning the AI response concept
guide from documentation-only guidance into a read-only CLI surface an AI
runtime can call before answering beginner intake questions.

## Added

- `archive ai-response-concept-guide --topic all --dry-run`.
- Aliases `archive ai-concept-guide` and `archive wom-concept-guide`.
- Topic filters:

```text
all
sha256_identity
manifest_vs_zet
three_layers
```

## Behavior

The command returns:

- beginner-facing answer cards for sha256 object identity vs location,
- manifest vs zet address-book guidance,
- `objet -> derived text -> zet` layer explanations,
- Korean and English beginner scripts for the core sha256/R2 confusion,
- safe routing hints for the next dry-run command,
- overclaim guardrails for upload, availability, URL, derived-text, and manifest
  proof claims.

This gives terminal-capable AI runtimes a standard response package for
questions like:

```text
Do I need to upload to R2 first?
If the file location changes, will the zet path break?
Is this like a Google Drive URL?
```

## Safety Boundary

This release remains read-only. It does not read source bytes, read derived-text
bodies, write object manifests, write derived-text records, write receipts,
draft zets, mint zets, upload objects, call providers, read secrets, echo source
filenames, echo local paths, echo provider URLs, echo account ids, echo emails,
echo tokens, or echo secret values.
