# WOM-kit v0.3.20

Date: 2026-06-14

v0.3.20 adds the first credential reference contract for WOM-kit.

## Added

- CLI `archive credential-ref-plan --dry-run`.
- MCP `credential_ref_plan`.
- `wom-kit/docs/credential-store-contract.md`.
- Tests for CLI behavior, MCP structured output, dry-run-only behavior,
  allowed-root enforcement, raw-secret redaction, and documentation coverage.

## Behavior

The plan validates refs such as:

- `env:OPENAI_API_KEY`,
- `keyring:openai-api-key`,
- `keyring:naver-app-password`,
- `keyring:gmail-oauth-token`,
- `secret:ocr-provider-api-key`,
- `wallet:future-local-secret`.

It recognizes credential kinds for mail usernames, mail app passwords, mail
OAuth tokens, OpenAI API keys, OCR API keys, provider API keys, object-storage
tokens, backup passwords, and generic secrets.

## Boundaries

This release does not store passwords, store OAuth tokens, store OpenAI API
keys, store OCR API keys, write to Windows Credential Manager, write to macOS
Keychain, write to Linux keyrings, read from any keyring, read environment
variables, start OAuth, call OpenAI, call OCR providers, call mail providers,
call object-storage providers, upload files, download files, draft zets, or
mint zets.

If a user accidentally passes a raw key, token, password, email, URL, or local
path where a ref is expected, the plan blocks and does not echo that raw value
back in structured output.
