# v0.3.116 - Mint Source Resolve Fast Path

Date: 2026-06-19

## What Changed

This release closes a remaining mint scale gap found after v0.3.114.

v0.3.114 moved mint duplicate checks onto the generated index when
`db/archive-index.sqlite` is current. Real archive profiling then showed another
archive-wide read path: `resolve_zettel_path` could still scan and parse every
zettel just to resolve a standard draft id such as:

```text
inbox/<zettel_id>.md
```

v0.3.116 adds a direct standard-path fast path. When a safe zettel id maps to
`inbox/<zettel_id>.md` or `zettels/<zettel_id>.md`, and that file's frontmatter
`id` matches the requested id, WOM-kit resolves the path directly before using
the legacy archive-wide id scan fallback.

## Mint Impact

For generated-index-backed mint loops, `archive mint-zet --zettel-id ...
--dry-run|--approve` no longer reparses every zettel just to find the standard
inbox draft path.

The existing compatibility fallback remains:

- non-standard filenames can still be found by the legacy id scan;
- direct files whose frontmatter id does not match are not trusted blindly;
- duplicate semantics do not change.

## Safety Boundary

This release does not:

- skip frontmatter id verification for direct paths,
- remove the legacy id scan fallback,
- weaken duplicate checks,
- trust stale generated indexes,
- rewrite receipts,
- rewrite zettel bodies,
- remove inbox drafts,
- call providers,
- add UI behavior,
- add MCP write tools.
