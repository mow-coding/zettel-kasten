# v0.3.134 - Notion Ancestor Live Structure Fetch

Date: 2026-06-22

## Summary

v0.3.134 opens the first approval-gated local Notion live structure adapter.

`archive notion-ancestor-fetch-adapter-run --dry-run|--approve` now consumes the
scoped ancestor crawl queue, verifies a matching credential access approval
receipt and policy check, reads a Notion token from an approved `env:` ref
inside the local CLI process, calls the Notion API for parent-chain structure
metadata, and writes a sanitized `notion_ancestor_result_fixture` plus a
non-secret execution receipt.

## Added

- CLI-only `archive notion-ancestor-fetch-adapter-run`, with aliases
  `archive notion-ancestor-fetch-run` and `archive notion-ancestor-live-fetch`.
- Merge-compatible sanitized ancestor fixture writes under `workbench/`.
- Non-secret execution receipts under `receipts/notion/ancestor-fetches/`.
- `notion` as an explicit credential provider label for Notion API-key refs.
- Documentation for the implemented run command:
  `wom-kit/docs/notion-ancestor-fetch-adapter-run.md`.

## Safety Boundary

The live run still does not:

- expose an MCP live provider-call tool,
- start OAuth,
- open a browser session,
- read OS keyrings or password managers,
- echo exact credential refs, environment variable names, or token values,
- store raw provider responses,
- read page titles, page bodies, comments, or media bytes,
- refresh signed file URLs,
- hash or preserve media bytes,
- write zets, mint pages, write edges, or update object manifests.

## Verification

- Targeted CLI tests cover dry-run no-write behavior, approval receipt
  verification, fake-provider live execution, sanitized fixture writes,
  non-secret receipt writes, and redaction of token/ref/title/provider URL
  material.
- The older execution-contract test remains green, confirming the contract
  command itself is still read-only.
