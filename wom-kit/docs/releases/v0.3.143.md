# v0.3.143 - Tiro OS Credential Store Read

v0.3.143 removes the last `env:`-only blocker from the Tiro lossless recovery
fetch path for Windows users who already have a local Tiro credential stored by
another desktop/MCP flow.

## What Changed

- `archive tiro-lossless-recovery-fetch-run` now accepts approved
  `keyring:` and `credential-manager:` refs in addition to `env:` refs.
- Approved Windows runs can read a Tiro token from Windows Credential Manager
  without adding a new dependency.
- Dry-run still reads no credential value, opens no OS credential store, calls
  no provider, and writes no files.
- The Windows reader first tries an exact generic credential target match. If no
  exact target exists, it may auto-detect exactly one matching generic
  credential from the safe ref label. Multiple matches block without echoing
  target names.
- The credential value can be a raw bearer/API token or a JSON-shaped OAuth
  blob containing `access_token`, `accessToken`, `bearerToken`, `apiKey`,
  `token`, `secret`, or `value`.
- Output and receipts still echo no exact credential refs, env var names, OS
  credential target names, tokens, provider URLs, account ids, emails, meeting
  titles, transcript text, participant names, or raw provider responses.

## Command Shape

The recovery chain remains:

```text
archive tiro-lossless-recovery-plan
archive tiro-lossless-recovery-fetch-run
archive tiro-lossless-recovery-capture
```

```powershell
archive tiro-lossless-recovery-fetch-run <archive-root> `
  --credential-ref keyring:<safe-tiro-label> `
  --workspace-guid <workspace-guid> `
  --output workbench/tiro-lossless-recovery.live.json `
  --dry-run `
  --format json
```

```powershell
archive tiro-lossless-recovery-fetch-run <archive-root> `
  --credential-ref credential-manager:<safe-tiro-label> `
  --workspace-guid <workspace-guid> `
  --output workbench/tiro-lossless-recovery.live.json `
  --approve `
  --reviewed-by human:<id> `
  --format json
```

## Upgrade Notes

No archive migration is required.

Existing `env:` Tiro fetch commands keep working. Windows users can now point
the fetch command at an approved OS credential-store ref instead of copying a
token into an environment variable.
