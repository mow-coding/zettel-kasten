# v0.3.155 - AI Response Contract

v0.3.155 adds a read-only AI response contract for WOM operators.

## What Changed

- Added `archive ai-response-contract`.
- Added aliases `archive response-contract` and
  `archive operator-response-contract`.
- Added a machine-readable answer contract with required sections:
  - `operation_outcome`,
  - `evidence_basis`,
  - `privacy_boundary`,
  - `approval_boundary`,
  - `remaining_work`.
- Added optional `conversation_status_board` guidance.
- Explicitly states that a compact conversational status board is allowed and a
  web UI is not required.
- Ties the answer contract to operation status, input provenance, secret-signal
  handling, approval handoff audit, and archive status-board checks.

## Safety Boundary

The command:

- reads no archive body text,
- accepts no sample values,
- echoes no sample values,
- calls no providers,
- checks no network,
- echoes no zettel body text, source values, local absolute paths, tokens, or secret values,
- writes nothing.

## Still Future

- Adding command-specific `status_class`, `input_provenance_class`, and

- Auditing write commands so every human-facing summary cites the relevant

- Adding a reusable renderer for compact conversational status-board summaries.
