# v0.3.58 - Runtime Canonical Entry Points

Released: 2026-06-16

This release adds read-only canonical entrypoint metadata to runtime context.

## Added

- `runtime-context` field `canonical_entrypoints`.
- `archive.yml` as the explicit start-here file for WOM archive identity and
  root policy.
- Archive-relative role/status reporting for local agent instructions, identity
  context, source bindings, provider metadata, canonical zets, draft inbox,
  object manifests, derived-text manifests, saved views, and schema context.
- Documentation and tests for the runtime entrypoint map.

## Safety Boundary

The new check reads no file bodies, writes no files, calls no providers, reads
no secrets, and echoes no local absolute paths by default.

## Still Future Work

This release does not add migration enforcement, automatic project repair, live
IMAP execution, provider sync, credential retrieval, or full export
disambiguation.
