# v0.3.91 - Generated Index Health

Date: 2026-06-17

## Added

- CLI `archive index-health <archive-root> --dry-run`.
- MCP `index_health`.
- Public docs for generated-index drift checks.

## Why

Real-use feedback included stale-index confusion: `view-zets` and related
read surfaces can depend on an old `db/archive-index.sqlite` while
`list-zettels` reads live files.

This release adds a read-only preflight to check that mismatch before trusting
indexed navigation results.

## Behavior

- Compares live zettel paths against indexed zettel paths.
- Compares live/indexed `id`, `status`, and `kind`.
- Flags zettel files modified after the index file.
- Reports sample archive-relative paths for drift buckets.
- Blocks clearly when the generated index is missing.

## Safety

The new path writes nothing, rebuilds no index, edits no zettels, reads no
object bytes, calls no providers, and echoes no zettel body text, zettel titles,
absolute local paths, provider URLs, account ids, emails, tokens, or secret
values.
