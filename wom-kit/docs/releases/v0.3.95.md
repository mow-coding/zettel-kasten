# v0.3.95 - Mint Checklist Guidance

Date: 2026-06-17

## Added

- `mint_checklist_guidance` in read-only `archive mint-zettel --dry-run` and
  `archive mint-zet --dry-run` output.
- Beginner quickstart guidance for marking human-reviewed mint checklist items.
- Text output guidance when required checklist items are still missing.

## Why

Phase 3 feedback showed that `mint-zet` blockers such as `one_clear_purpose`
and `sensitive_content_reviewed` were correct but under-explained. A beginner
operator could see the blocker without knowing the safe frontmatter path for a
human-reviewed checklist decision.

## Behavior

- Reports missing required checklist ids.
- Reports checklist ids that still need explicit human review.
- Names `mint.checklist` as the preferred frontmatter path for new drafts.
- Keeps `promotion.checklist` as a legacy accepted compatibility path.
- Includes a boolean-only frontmatter example with no body text or private
  metadata.

## Safety

This release does not loosen minting. Dry-run writes nothing, real minting still
requires `--approve` and `--reviewed-by`, and machine-enforced safety checks
such as provider URL/object-id and edge-type checks still run at mint time.
