# v0.2.2 Text Provenance And Public History Hygiene

Date: 2026-05-23
Status: pre-release

## Summary

`v0.2.2` clarifies text provenance and cleans up public repository history.

The important design clarification:

```text
original editable text and OCR/AI-derived text are both stored,
but they do not have the same authority.
```

## What Changed

- Added `wom-kit/docs/text-provenance-hierarchy.md`.
- Clarified source object storage policy.
- Clarified object manifest boundaries for derived text.
- Updated README to mention provenance-aware derived text records.
- Prepared public history hygiene so local/private-looking examples are not retained as normal public refs.

## Compatibility

Compatible with `v0.2.1`.

No implemented schema migration is required.

This is currently a documentation/model clarification. Future schema work may add explicit derived-text records.

## Text Provenance Rule

Use this authority ladder:

```text
L0 original source object
L1 born-digital editable text
L2 parser-extracted text
L3 OCR / speech-to-text / AI transcription
L4 human-reviewed derived text
L5 minted zet
```

OCR and AI transcription should record:

- source object id,
- derivation kind,
- tool/model name,
- tool/model version when available,
- confidence when available,
- review status,
- creation time.
