# v0.3.59 - Derived Text Completeness Signal

Released: 2026-06-16

This release adds a manifest-scoped completeness signal to derived-text
coverage.

## Added

- `completeness_signal` in `archive derive-text coverage <archive-root>
  --dry-run --format json`.
- Explicit distinction between manifested objet derived-text coverage and full
  external workspace/mailbox/cloud-drive mirror completion.
- Public documentation and tests for the new signal.

## Safety Boundary

The signal reads no source file bodies, scans no external workspaces, calls no
providers, reads no secrets, writes no files, and echoes no local absolute
paths.

## Still Future Work

This release does not create full mirror manifests, run provider exports, scan
local folders, or prove full external account coverage.
