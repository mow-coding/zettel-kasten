# v0.2.37 Foreign Block Decision Outcome Plan

Release date: 2026-05-25

## Summary

v0.2.37 adds a read-only outcome planner for one recorded foreign block quarantine decision.

The new `quarantine-decision-outcome` command answers what the next safe non-mutating path is after a decision has been recorded and reviewed.

## Added

- `archive quarantine-decision-outcome <archive-root> --case-id <safe-case-id> --dry-run --format json`
- Optional replay/context flags:
  - `--expected-decision`
  - `--reviewer`
  - `--review-note`
- Read-only MCP `foreign_block_decision_outcome_plan`.
- New documentation: `wom-kit/docs/foreign-block-decision-outcome-plan.md`.

## Outcome Routing

- `keep_quarantined` stays isolated.
- `reject_and_keep_record` preserves quarantine and decision records.
- `needs_more_review` asks for more human review context.
- `eligible_for_attestation_review` becomes `prepare_attestation_review_candidate`.

`eligible_for_attestation_review` is still not trust and does not create an attestation in v0.2.37.

## Safety

Before returning a plan, WOM-kit re-reads:

- the current quarantine case,
- the original quarantine receipt,
- the recorded quarantine decision,
- the quarantine decision receipt.

Missing, contradictory, unsafe, or stale state blocks the plan. The command writes nothing and returns `would_change: []`.

## Non-Goals

v0.2.37 does not implement:

- decision outcome apply or acceptance,
- quarantine decision trust,
- foreign block import,
- foreign attestation writes,
- minting from foreign blocks,
- anchoring,
- delegation,
- real ZET transport,
- signing,
- payment,
- staking,
- consensus,
- blockchain,
- provider sync,
- OCR,
- LLM classification,
- full-auto execution.

MCP remains read-only for this workflow and exposes no decision outcome write/apply/accept/import/trust/attest/mint/full-auto tool.
