# v0.3.11 - Prehashed Multi-Ledger Registration

Release date: 2026-06-14

v0.3.11 is a compatible fast-follow to v0.3.10. It improves the external
prehashed objet ledger bridge for already-hashed source-export stores.

## Added

- CLI `archive prehashed-objet-ledger` now accepts repeated `--ledger`
  arguments.
- Multi-ledger dry-runs and approvals dedupe sha256 values across all provided
  ledgers in one pass.
- Prehashed ledger results now include `ledger_file_count`,
  `ledger_file_sha256es`, `skipped_row_count`, and `skipped_reasons`.

## Changed

- Rows with null or empty `sha256` are counted as skipped rows. This supports
  aid-dedup style ledgers where a row represents a file already accounted for in
  another ledger.
- Malformed non-empty sha values remain invalid and still block approval.
- README, capability matrix, and Notion source-export examples now document the
  multi-ledger path.

## Boundaries

- WOM-kit still does not read blob bytes, copy objects, upload, sync, call
  providers, draft, mint, or clean in `prehashed-objet-ledger`.
- MCP remains read-only through `prehashed_objet_ledger_preview`.
- `objet-capture` still verifies bytes itself from staged files; prehashed
  ledger registration is a separate manifest-registration path for externally
  verified stores.

## Upgrade

No archive migration is required for v0.3.10 users.
