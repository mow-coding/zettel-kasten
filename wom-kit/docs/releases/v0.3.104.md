# v0.3.104 - Notion Import Material Clue Audit

Date: 2026-06-17

## Summary

v0.3.104 adds a read-only audit for imported Notion zettels whose provider
locators may have been removed from body text.

The new command is:

```powershell
archive notion-objet-import-clue-audit <archive-root> --dry-run --format json
```

It can also use explicit source maps and download/retrieval ledgers:

```powershell
archive notion-objet-import-clue-audit <archive-root> --source-map source-maps/notion-export.jsonl --ledger receipts/import/notion-download-ledger.jsonl --dry-run --format json
```

MCP exposes the same read-only audit as:

```text
notion_objet_import_clue_audit
```

## User-Facing Meaning

v0.3.103 can propose material-link candidates from source maps and ledgers.
v0.3.104 adds the surrounding audit question:

```text
After import removed provider locators from the body, did each imported zet keep
a safe object ref, recoverable source-map clue, or nothing?
```

The audit classifies imported Notion zettels into preserved, recoverable, and
missing-material-clue groups before any edge write.

## Safety Boundary

This release adds no writer and no live provider adapter. The audit:

- writes nothing,
- reads no zettel body text,
- rewrites no body text,
- writes no edges or receipts,
- reads no object bytes,
- calls no providers,
- creates no presigned URLs,
- echoes no provider URLs, provider locator text, page titles, frontmatter
  values, local absolute paths, account ids, emails, tokens, or secret values.

## Migration

No archive migration is required.

Future import adapters should preserve a safe `source_refs` object id, a
planned/reviewed `embed` candidate, or a source-map/ledger join row whenever
they omit provider locators from zettel bodies.
