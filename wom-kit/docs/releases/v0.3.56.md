# v0.3.56 - IMAP Adapter Audit Receipt Write

Date: 2026-06-16

v0.3.56 adds a local, approval-gated IMAP adapter audit receipt writer.

The release still does not connect to mail providers or read mail. It only
writes one reviewed, non-secret audit receipt under the archive when explicitly
approved.

## Added

- CLI `archive imap-mailbox-adapter-audit-write --dry-run|--approve`.
- Alias `archive mailbox-adapter-audit-write`.
- Local receipt writes under `receipts/imap/adapter-audits/`.
- Replay blocking when the same audit receipt already exists.
- New documentation at `wom-kit/docs/imap-mailbox-adapter-audit-write.md`.

## Verify

```bash
archive imap-mailbox-adapter-audit-write <archive-root> \
  --adapter-id local-imap \
  --source-id imap:gmail-personal \
  --provider gmail \
  --account-ref imap:account:gmail-personal \
  --username-ref env:WOM_GMAIL_USERNAME \
  --auth-mode oauth_token_ref \
  --oauth-token-ref keyring:gmail-oauth \
  --mailbox-ref imap:mailbox:inbox \
  --credential-id cred:gmail-mail-access \
  --operation header_metadata_scan \
  --selection-rule newest_first \
  --selector-id mail-selection:recent-inbox \
  --result-status not_run \
  --reviewed-by person:me \
  --dry-run \
  --format json
```

Expected dry-run shape:

```text
lifecycle_action: imap_mailbox_adapter_audit_write_plan
audit_state: preview_ready
files_written: []
would_change: [receipts/imap/adapter-audits/...]
closed_actions.imap_connection_opened: false
closed_actions.message_headers_read: false
```

## Safety

The audit receipt writer:

- writes no files unless `--approve --reviewed-by <actor>` is used,
- writes only one archive-internal non-secret audit receipt,
- opens no IMAP connection,
- attempts no login,
- selects or searches no mailbox,
- lists no candidate messages,
- reads no IMAP UIDs or Message-ID values,
- reads no headers, bodies, or attachments,
- creates no derived text,
- retrieves no secrets,
- starts no OAuth,
- calls no providers,
- exposes no MCP write tool,
- echoes no email addresses, username values, exact account refs, exact
  credential refs, exact mailbox refs, IMAP host values, provider URLs, message
  ids, subjects, sender or recipient values, attachment names, approval receipt
  paths, selection receipt paths, local absolute paths, tokens, or secret
  values.

No archive migration is required.
