# v0.3.193 - Doctor local profile secret-safety progress

v0.3.193 adds progress inside the `local-profile-secret-safety` doctor stage.

The previous release made full doctor progress manageable for large archives.
That let operators reach a later quiet stage. This release makes that stage
observable too.

## Local Profile Secret-Safety Drilldown

`doctor --progress` can now emit:

- `checking gitignore secret patterns`
- `walking archive files`
- `checked archive files files=<count> content=<count> profiles=<count> skipped_dirs=<count>`
- `still checking local profile secret safety files=<count> content=<count> profiles=<count> skipped_dirs=<count>`
- `still scanning secret content <archive-relative-path> bytes=<count>`
- `local profile secret safety summary checked_files=<count> content_scanned=<count> local_profiles=<count> skipped_dirs=<count>`

The messages are content-free. They include counts and archive-relative paths,
but never secret values.

## Chunked Secret Content Scan

Eligible config/text files are now scanned in chunks instead of being loaded as
one full string. This keeps the same diagnostic intent while giving long reads a
heartbeat path.

## Compatibility

Additive; **no migration required**.

Result JSON, receipts, manifests, and archive files are unchanged. Diagnostics
are unchanged except for the same secret-safety findings the stage already
reported when it found a problem.
