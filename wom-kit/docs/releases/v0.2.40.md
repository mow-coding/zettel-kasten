# v0.2.40 Release Notes

Date: 2026-05-26

## Summary

v0.2.40 adds a read-only review index for recorded foreign block attestation review candidates.

The new index lets an operator list v0.2.39 candidate records, validate their matching receipts, and re-check the upstream quarantine case, quarantine receipt, decision record, and decision receipt before any later human review path.

## Added

- CLI `archive attestation-candidate-review <archive-root> --format json`.
- Optional `--case-id`, `--review-scope`, and `--include-receipts` filters.
- Read-only MCP `foreign_block_attestation_review_candidate_index`.
- Candidate record and candidate receipt consistency checks.
- Upstream quarantine case, original quarantine receipt, decision record, and decision receipt consistency checks.
- Case-level projection and count fields for deterministic review output.

## Safety Boundary

The index is read-only:

- writes nothing,
- keeps `dry_run: true`,
- keeps `trust_state: untrusted_foreign`,
- returns `would_change: []`,
- never reads original foreign artifacts, source payloads, objet bodies, provider URLs, or private local files,
- never creates trust, import, attestation, signatures, minting, sharing, provider calls, ZET transport, acceptance, or apply behavior.

Filters affect displayed candidates only. WOM-kit still validates every discovered candidate record before setting top-level `ok`.

## Compatibility

No private archive migration is required.

Archives without recorded candidates return an empty index with a warning, not an error.

MCP remains read-only and exposes no candidate review approve/write/apply/trust/import/attest/sign/mint/full-auto tool.
