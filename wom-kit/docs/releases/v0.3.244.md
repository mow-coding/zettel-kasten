# v0.3.244 - Approval-Gated Agent Skill Host Lifecycle

v0.3.244 closes the local activation gap between an installed WOM-kit command
and a discoverable `wom-archive` Agent Skill.

## What Changed

- Added read-only `archive runtime-skill-status`.
- Added preview-first, approval-gated `archive runtime-skill-install` for first
  install and verified managed update.
- Added preview-first, approval-gated `archive runtime-skill-uninstall` for an
  unchanged WOM-kit-managed target.
- Added current Codex user and repository scope resolution through
  `$HOME/.agents/skills` and `<repo>/.agents/skills`.
- Added explicit custom-host target support without guessing another product's
  configuration path.
- Added a path-free, self-checking ownership manifest with relative file names,
  byte counts, SHA-256 values, package/source identity, install time, and
  reviewer id.
- Added exact operation-plan digests, exclusive locking, staged copy, managed
  update backup, outside-active-root uninstall, and post-write verification.
- Extended clean-wheel verification to install, verify, and uninstall the skill
  in a disposable host directory using the installed `archive` entrypoint.

## Safety Boundary

- Python wheel installation still writes no AI-host configuration.
- Host writes require a separate dry-run, reviewer, and exact plan digest.
- Unmanaged, invalid, symlinked, or human-edited targets fail closed.
- Local paths are redacted by default and file bodies are never echoed.
- No archive, zet, objet, relation, or receipt is read or written.
- No network, provider, model, database, object store, credential store, MCP
  write, generated graph, or index is used.

## Verification

Release verification covers direct installer lifecycle tests, stale-plan,
collision, drift, unmanaged-target, path-redaction, and scope checks; the full
WOM-kit suite; package-resource parity; clean-wheel install/host-skill
lifecycle/onboarding/strict-Doctor checks; public privacy hygiene; and release
readiness.

## Upgrade

No archive migration is required. Update WOM-kit, run
`archive runtime-skill-install --dry-run`, review the result, then approve the
exact plan. Restart Codex only if it does not discover the installed skill.
