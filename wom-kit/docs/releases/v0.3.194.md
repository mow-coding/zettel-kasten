# v0.3.194 - Doctor result capture and diagnostic guidance

v0.3.194 makes large `archive doctor` runs easier for AI operators to capture
and report without overflowing a terminal.

## Full Result File

`archive doctor` now accepts:

```text
archive doctor <archive-root> --strict --summary --output logs/doctor-result.json
```

`--output` writes the full diagnostics JSON array to the selected
archive-relative file. Stdout prints only a compact summary,
so a large strict run can keep the complete result without flooding the chat or
terminal.

## Summary and Severity Filters

New stdout controls:

- `--summary`
- `--errors-only`
- `--diagnostic-level ERROR,WARN`

These controls only affect what is printed to stdout. The exit code still uses
the full diagnostics set: any ERROR fails, and `--strict` also fails on WARN.

## Safer Repair Hints

Doctor diagnostics now include clearer next-action hints for:

- invalid `provenance.creation_mode` enum values;
- object-storage execution receipts that have no matching `wom_uploaded`
  manifest location;
- UTF-8 BOM zettel warnings.

The hints route operators toward doctor result capture, schema-safe frontmatter
correction, remint-reconcile dry-runs, and upload/adopt/reconcile workflows
instead of manual receipt edits.

## Compatibility

Additive; **no migration required**.

Default `archive doctor` behavior is unchanged unless the new output/filter
options are used. Result JSON, receipts, manifests, and archive files are
unchanged.
