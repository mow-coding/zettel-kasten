# v0.2.45 Release Notes

Date: 2026-05-26

## Summary

v0.2.45 adds the ZET publication surface baseline.

This is a documentation and sanitized-example release. It explains how WOM can stay no-UI at the core while a user selects an outside surface for publication or projection. WordPress is included only as a familiar example surface.

## Added

- `wom-kit/docs/zet-publication-surface-baseline.md`
- `wom-kit/examples/zet-publication-surface/README.md`
- `wom-kit/examples/zet-publication-surface/zet-publication-envelope.example.json`
- `wom-kit/examples/zet-publication-surface/wordpress-post.safe-html.example.html`
- `wom-kit/examples/zet-publication-surface/wordpress-title.example.txt`

## Clarified

- A zet remains canonical archive memory.
- Posting is not minting.
- A surface locator is not the canonical zet identity.
- WordPress can be a selected projection surface, but it is not the WOM core.
- Future projection should pass through dry-run planning, a scope gate, human approval, and receipt preview before any provider-specific publisher exists.

## Compatibility

No private archive migration is required.

No CLI or MCP behavior changed in this release. Existing archives, commands, schemas, and tests remain compatible.

## Safety

v0.2.45 does not:

- call providers,
- publish to WordPress,
- implement projection-plan CLI/MCP,
- create projection receipts,
- trust, import, accept, attest, sign, mint, anchor, or transport anything,
- implement ZET transport,
- introduce Redis, backpropagation, model training, full-auto execution, payments, staking, consensus, blockchain, WOM coin, or token mechanics.
