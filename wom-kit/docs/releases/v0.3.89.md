# v0.3.89 - Notion Objet Locator Bridge

Date: 2026-06-17

## Added

- CLI `archive notion-objet-link-plan --path <zet.md>|--zettel-id <id>
  --dry-run`.
- MCP `notion_objet_link_plan`.
- Public docs for the Notion provider locator to manifested objet bridge.

## Why

Phase 3 real-use feedback showed that imported Notion zets can contain Notion
page mention or embed locators while `zettel-objet-links` only resolves stable
`sha256:<hex>` and `objet:sha256:<hex>` refs.

This release adds the missing read-only planning bridge before any body rewrite
or durable edge write.

## Behavior

- Scans one non-redacted zettel for Notion provider locator fingerprints.
- Matches locators against reviewed local metadata in
  `objects/manifests/files.jsonl`.
- Returns safe object candidates, match field names, store labels, resolver
  states, and suggested `objet:sha256:<hex>` refs for later human review.

## Safety

The new path writes nothing, rewrites no zettel body, calls no providers, starts
no OAuth, creates no provider or presigned URLs, reads no object bytes, and
echoes no zettel body text, frontmatter values, provider URLs, page titles,
absolute local paths, account ids, emails, tokens, or secret values.

Approval-gated Notion locator rewrite remains future work.
