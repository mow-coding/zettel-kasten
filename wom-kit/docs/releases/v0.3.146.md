# v0.3.146 - Archive Status Board

v0.3.146 adds a read-only archive status board for the real-use problem where
beginners cannot easily tell which zets are canonical, which drafts are still
active, which minted drafts are ready to retire, and which records need source
or derived-artifact metadata review.

## What Changed

- Added read-only `archive status-board`.
- Added aliases `archive archive-status-board` and `archive zet-status-board`.
- The board reports counts and limited safe examples for:
  - canonical zets,
  - active drafts,
  - minted drafts pending `retire-draft`,
  - canonical lifecycle metadata gaps,
  - mint receipt gaps,
  - missing `document_type`,
  - missing `audience`,
  - source metadata gaps,
  - derived artifact source/sync gaps.
- Added `--include-quality` to include body-inspecting quality blocker/warning
  counts using the v0.3.145 quality gate.
- Added next-action guidance for common review paths.

## Command Shape

```powershell
archive status-board <archive-root> `
  --dry-run `
  --format json
```

```powershell
archive status-board <archive-root> `
  --dry-run `
  --include-quality `
  --format json
```

## Safety Boundary

The status board:

- writes nothing,
- calls no providers,
- retires no drafts,
- edits no zets,
- syncs no reports,
- creates no revision candidates,
- cleans no scratch files,
- echoes no zet titles, body text, source-ref values, provider URLs, local
  absolute paths, tokens, or secret values.
