# v0.3.70 - IMAP Material Capture Execution Contract

Status: v0.3.70 read-only IMAP material capture execution contract checkpoint
Date: 2026-06-16

v0.3.70 adds the read-only execution-contract gate after a human-reviewed IMAP
material capture request is valid and before any future command reads selected
message bodies, attachment bytes, or mail-derived text.

## Added

- `archive imap-mailbox-material-capture-execution-contract <archive-root>
  --material-selection-receipt <archive-relative-json> --capture-action
  message_body_capture|attachment_capture|derived_text_capture --dry-run
  --format json`.
- Aliases:
  - `archive imap-material-capture-execution-contract`
  - `archive mailbox-material-capture-execution-contract`
- A public documentation page:
  - [IMAP Mailbox Material Capture Execution Contract](https://github.com/mow-coding/zettel-kasten/blob/main/wom-kit/docs/imap-mailbox-material-capture-execution-contract.md)

## Behavior

The command validates one non-secret material selection receipt through the
existing material capture request plan, then returns the future live-adapter
execution contract for the requested capture action.

The contract names the future execution mode, required future inputs, allowed
actions after implementation and separate approval, and a non-secret output
receipt shape.

## Safety Boundary

This release still does not implement message body capture, attachment capture,
or mail-derived text extraction.

The command reads no original execution receipt, opens no IMAP connection, reads
no environment variables, opens no keyring/password manager, reads no message
headers again, reads no message bodies, reads no attachments, creates no
derived text, writes no files, calls no providers, and echoes no material
selection receipt path, execution receipt path, candidate refs, subjects,
senders, recipients, local paths, tokens, or secret values.

## Verification

- CLI tests cover dry-run enforcement, ready execution-contract output,
  candidate-ref non-echoing, private-marker non-echoing, no writes, no material
  reads, and mismatched capture-action blocking.
- Capability matrix and public documentation tests cover the new command,
  documentation link, safety boundary, and release notes.
