# v0.3.218 - Reviewed Abstract Backfill Planning

v0.3.218 adds a read-only, exact-version planning gate for missing canonical
zet abstracts. It is additive and requires no archive migration.

## What Changed

- `read-zettel` now returns exact canonical file SHA-256 and decoded body
  SHA-256 for non-redacted zets. Redacted zet hashes remain suppressed.
- Added the private JSONL proposal contract and shipped JSON Schema.
- Added `zet-abstract-backfill-plan` (`abstract-backfill-plan`).
- Validates proposal shape, current canonical hash, identity/status, missing
  first-read state, safe bounded abstract text, and one-field-only insertion.
- Returns row indexes, safe counts, proposal/plan/candidate/abstract hashes, and
  blocker codes without returning ids, paths, titles, bodies, abstract text, or
  proposal filenames.
- Added content-free file/row progress and bounded 64 MiB, 1 MiB-line, and
  5,000-row limits.

## Honest Boundary

The planner reads selected canonical zet bodies to validate the exact candidate
bytes, but echoes none of that content. It calls no model or provider and writes
nothing.

This release does not approve or apply a revision. Green means candidates are
bound to current canonical bytes and ready for human review; it does not prove
the abstract is true, complete, understood, or approved. A transactional,
receipt-backed write remains a separate capability.
