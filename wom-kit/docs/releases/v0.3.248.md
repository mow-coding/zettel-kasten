# v0.3.248 - Canonical Revision Before Snapshot

v0.3.248 makes WOM's artifact-first doctrine concrete at the most dangerous
ordinary mutation boundary: immediately before a reviewed canonical zet is
replaced.

## What Changed

- `zet-revision-write --dry-run` now identifies the content-addressed object
  that approval will preserve, while still writing nothing.
- Approval writes the exact prior canonical bytes under
  `objects/sha256/<prefix>/<sha256>` before replacing the zet.
- It registers or reuses the matching local object-manifest record.
- New ordinary revision receipts use
  `wom-kit/zet-revision-receipt/v0.2` and bind a text-free
  `before_snapshot` descriptor to the prior file hash.
- The write-ahead lock carries the same descriptor, so an interrupted process
  can verify the old bytes before finishing a missing receipt.
- Handled failures restore the canonical zet and retain a verified snapshot for
  safe idempotent reuse.
- `zet-revision-receipt-audit` verifies each unique new snapshot and its
  manifest record. Missing, unsafe, or changed bytes block the audit.
- Existing v0.1 revision receipts and locks remain readable and recoverable as
  legacy hash-only history.

## Why It Matters

A hash proves which bytes existed, but cannot recreate them. A correction
system that keeps only before/after hashes can prove that memory changed while
still losing the memory that changed. This release keeps the previous artifact
itself, locally and content-addressed, before installing its reviewed successor.

## Safety Boundary

- Snapshot bytes are written only after the existing SHA-bound preview and
  explicit human approval gates.
- The final object path is never overwritten. Existing bytes must hash to the
  expected object id.
- The object store remains ignored by Git; only the text-free manifest record
  and revision receipt are ordinary archive metadata.
- No model, provider, credential store, database, remote object store, or
  network is called.
- This release does not claim remote backup completion. The local snapshot must
  still pass the archive's chosen object-storage backup workflow.
- No existing archive migration is required. v0.1 receipts remain valid, but
  they do not gain old bytes retroactively.

## Verification

Regression coverage includes no-write preview, exact-byte preservation,
manifest registration, idempotent replay, receipt-write rollback, interrupted
receipt recovery, destination collision refusal, v0.1 compatibility, and audit
failure for changed snapshot bytes. Release gates also cover the full WOM-kit
suite, package resources, clean-wheel installation, public privacy, and release
readiness.
