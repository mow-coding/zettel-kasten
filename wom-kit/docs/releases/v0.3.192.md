# v0.3.192 - Doctor progress volume control

v0.3.192 makes `doctor --progress` safer for large archives and AI-operated
terminals.

The previous release exposed the inside of target edge-receipt evolution. That
made the slow path visible, but it also made receipt-level progress too noisy
for long full-archive runs. This release keeps the diagnostic surface while
reducing default stderr volume.

## Compact Progress By Default

`doctor --progress` now uses compact stderr progress by default.

Compact progress keeps:

- stage `start` / `done` lines
- receipt milestones
- quiet-interval heartbeats
- key edge receipt index lifecycle events
- early counted-stage ETA warm-up

It no longer prints every receipt sub-step to stderr by default.

## Full Trace Opt-In

The previous detailed behavior remains available:

```text
archive doctor <archive-root> --strict --progress --progress-detail verbose
```

Use this when you are narrowing a single hard-to-localize failure and can afford
a larger progress stream.

## Full Progress JSONL Log

Operators can now write every progress event to a file while keeping stderr
compact:

```text
archive doctor <archive-root> --strict --progress --progress-log logs/doctor-progress.jsonl
```

The log option can also be used without `--progress` if an operator wants a
machine-readable trace file but no stderr progress stream.

## Compatibility

Additive; **no migration required**.

Result JSON, diagnostics, receipts, manifests, and archive files are unchanged.
Only optional progress reporting behavior changes.
