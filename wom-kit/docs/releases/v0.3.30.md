# WOM-kit v0.3.30

Date: 2026-06-15

v0.3.30 adds a read-only credential policy checker.

## Added

- CLI `archive credential-policy-check --dry-run`.
- CLI aliases `archive credential-access-policy-check --dry-run` and
  `archive secret-policy-check --dry-run`.
- MCP `credential_policy_check`.
- `wom-kit/docs/credential-policy-check.md`.

## Behavior

The checker evaluates a proposed credential use request and returns one of:

- `ready_after_approval_receipt`,
- `needs_human_review`,
- `denied_by_human_decision`,
- `denied_by_policy`,
- `blocked`.

It validates approval decision, store kind, adapter kind, adapter operation,
action kind, and non-echo policy before any future adapter can run.

## Boundaries

This release does not write approval receipts, run live adapters, open password
managers, open browser password stores, open OS keyrings, read environment
variables, read plaintext files, detect secret values, write vault entries,
write keyring entries, call providers, start OAuth, draft zets, or mint zets.

Even a passing policy result sets `live_execution_allowed_now` to `false`.

## Relationship To Previous Layers

The intended safe sequence is:

```text
credential-store-recommendation
-> credential-vault-onboarding-plan
-> credential-plaintext-migration-plan
-> credential-ref-plan
-> credential-ref-inventory
-> credential-access-broker-plan
-> credential-access-approval-plan
-> credential-policy-check
-> future adapter execution
-> credential-adapter-audit-plan
```

v0.3.30 creates the gate that future live adapters must pass. Live secret
storage, retrieval, migration, rotation, and provider use remain future work.
