# v0.3.187 - AI artifact lifecycle inventory

v0.3.187 adds a read-only AI artifact lifecycle inventory.

The motivating question is simple: when a human uses WOM with an AI operator,
the archive-adjacent workspace can accumulate AI-generated notes, plans,
summaries, report drafts, patch notes, and chat logs such as JSONL transcripts.
Those files should not silently sit forever in scratch folders, but they also
should not be copied raw into canonical zet bodies.

This release makes that middle state visible.

## AI artifact inventory

New command:

```bash
archive ai-artifact-inventory <archive-root> --dry-run --format json
```

Aliases:

```text
archive ai-artifact-status
archive ai-residue-inventory
```

The inventory scans only allowlisted AI artifact/scratch roots:

- `.wom-scratch/`
- `workbench/ai-scratch/`
- `staging/ai/inbox/`
- `staging/ai/reviewed/`

It does not perform a broad archive-wide sweep.

## Fate states

Each listed candidate receives an `artifact_ref`, an inferred `artifact_kind`,
and a `fate_state`.

Current fate states:

- `unreviewed_ai_artifact`
- `source_intake_recorded`

The inventory also recommends possible fates:

- preserve raw bytes as an objet,
- register derived text,
- distill to a draft zet,
- link to an existing zet,
- defer review,
- discard only with receipt evidence.

## JSONL chat logs

JSONL AI conversation logs are treated as possible raw evidence to preserve as
objets. The safety rule is not "discard chat logs." The rule is:

```text
raw AI artifact or chat log
-> objet/source evidence
-> derived or distilled text
-> draft zet
-> human review
-> canonical zet
```

In other words, preserve the raw log when it matters, but do not paste the raw
conversation directly into a canonical zet body.

## Privacy boundary

The command is read-only and metadata-only:

- reads no file bodies,
- calculates no content hashes,
- writes no files,
- deletes nothing,
- creates no zets,
- calls no providers,
- does not echo archive-relative paths by default.

Use `--show-relative-paths` only for local operator review when the operator
needs to pick a specific file.

## Tests

Added focused CLI coverage for:

- unreviewed AI artifact inventory without body/path echo;
- JSONL chat-log artifact kind detection;
- no-run-without-`--dry-run` blocking;
- `source_intake_recorded` detection after recording a matching AI artifact

## Compatibility
