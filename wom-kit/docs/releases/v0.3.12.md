# v0.3.12 - Project Intake Bulk Bridge

Release date: 2026-06-14

v0.3.12 is a compatible documentation follow-up to v0.3.11. It makes the
project-intake cookbook clearer for large already-hashed migrations.

## Changed

- `wom-kit/docs/project-intake-cookbook.md` now explains the two-layer path for
  large migrations:
  - register already-hashed raw object ledgers with
    `archive prehashed-objet-ledger --ledger ... --approve`;
  - use the project-intake cookbook only for the selected subset that should
    become human-visible drafts or zets.
- The cookbook now clarifies that `archive-objets/` is the recommended local
  staging root in the rehearsal commands, not a requirement to move an existing
  external content-addressed store.
- The source-intake-record to capture-selection handoff now uses explicit
  `$sourceIntakeReceipt` and `$selectionJson` variables for easier copy/paste
  rehearsal.

## Boundaries

- This release does not add automatic bulk import, provider sync, uploads,
  drafting, minting, or cleanup.
- `prehashed-objet-ledger` remains an explicit approval-gated manifest
  registration path for externally verified object stores.
- The project-intake cookbook remains a manual, human-reviewed command spine.

## Upgrade

No archive migration is required for v0.3.11 users.
