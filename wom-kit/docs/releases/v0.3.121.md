# v0.3.121 - Scoped Validate and Progress

Date: 2026-06-21

## What Changed

v0.3.120 removed the biggest per-receipt edge rescan from large mint and retire
validation flows. v0.3.121 adds the next real-use validation layer: operators
can now ask WOM-kit to validate only the batch or indexed facet slice they just
touched.

This is for the common large-archive loop:

```text
run a reviewed batch -> validate the touched artifacts -> continue the next batch
```

## Validate Since

`archive validate --since <batch-id-or-receipt>` accepts a mint batch,
retired-draft batch, or zettel-edge batch id/receipt reference.

The scoped run validates:

- the narrow global archive structure needed to interpret the archive,
- the batch receipt,
- touched mint, retired-draft, or zettel-edge receipts,
- touched zettels referenced by those receipts.

This avoids forcing every post-batch check to start with a full archive
validation pass.

## Validate Scope

`archive validate --scope <facet=value>` validates zettels that match indexed
facet rows in `db/archive-index.sqlite`.

The scope filter may be repeated. For example:

```powershell
archive validate <archive-root> --scope source_database=database_2_0 --progress
```

`--scope` requires a current generated index. If the index is missing or stale,
the command fails closed and asks the operator to run `archive index` first.

## Generated Index Cache

Generated index zettel rows now include:

- `file_size`
- `file_mtime_ns`
- `body_sha256`
- `approved_body_sha256`
- `forbidden_location_reference_found`

When a scoped validation run sees that a selected zettel's size and mtime still
match the index row, it can reuse the indexed frontmatter and body evidence
instead of rereading the unchanged body.

## Progress Output

`archive validate --progress` streams progress to stderr with stage names,
item counts, elapsed time, and ETA where an item total is known.

This is intentionally separate from JSON output, so scripts can still parse
stdout while humans watch the long run move.

## Compatibility

No archive migration is required.

Existing full validation remains valid:

```powershell
archive validate <archive-root>
archive validate <archive-root> --format json
```

Scoped validation is additive:

```powershell
archive validate <archive-root> --since <batch-id-or-receipt>
archive validate <archive-root> --scope <facet=value>
archive validate <archive-root> --progress
```

## Safety Boundary

Scoped validation is not a replacement for full archive validation.

This release does not:

- skip human approval gates,
- trust a stale generated index for `--scope`,
- perform archive migration,
- call providers,
- read source object bytes,
- echo zettel body text,
- add MCP write tools,
- add a worker, queue, daemon, or scheduler.

The optimization is local and rebuildable. It narrows validation work for recent
batch feedback loops while keeping periodic full `archive validate` as the broad
archive health check.
