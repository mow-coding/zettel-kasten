# v0.3.135 - Notion Recovery Beginner Guidance

Date: 2026-06-22

## Summary

v0.3.135 is a usability fast-follow to v0.3.134.

The live Notion ancestor structure fetch adapter exists, but a beginner should
not have to assemble the human approval and run steps from low-level vocabulary.
This release adds a read-only guide:

```text
archive beginner-setup-manual <archive-root> --topic notion_nested_recovery --dry-run
```

## Added

- `notion_nested_recovery` topic for `archive beginner-setup-manual`.
- Plain-language recovery terms:
  - ancestor / parent / child -> folder, shelf, upper page, and item location,
  - fetch / crawl -> ask Notion again for missing location links,
  - fixture / node -> review list and item,
  - merge -> put recovered locations back into the reviewed list,
  - untraceable -> items whose old location is still unknown.
- Human-step guide for:
  - reviewing the scoped missing-location queue,
  - preparing a private local env ref for the Notion token,
  - previewing and writing one-time credential approval,
  - previewing and running the approved live structure fetch,
  - handing the sanitized result fixture to `notion-ancestor-merge-plan`.

## Safety Boundary

The new guide is read-only. It does not:

- write approval receipts,
- read environment variables,
- call Notion,
- run the live fetch adapter,
- write ancestor result fixtures,
- read page titles, page bodies, comments, or media bytes,
- refresh signed file URLs,
- echo raw provider responses, exact env var names, account emails, provider
  URLs, token values, or secret values.

## Verification

- CLI tests cover the new guide topic, plain-language terms, guided step order,
  command handoff, no-write behavior, and privacy guards.
- Documentation tests cover the new topic in the beginner setup manual and
  capability matrix.
