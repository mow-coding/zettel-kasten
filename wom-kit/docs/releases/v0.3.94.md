# v0.3.94 - Projection Plan Surface Help

Date: 2026-06-17

## Added

- Clearer `archive projection-plan --help` text for supported projection
  surface kinds and projection formats.
- `projection_contract` in `projection-plan --dry-run --format json` output.
- Safe `surface.requested_surface_kind` and `surface.surface_status` fields.

## Why

Phase 3 feedback showed that `projection-plan` rejected values such as
`notion`, but the CLI help did not make the accepted surface enum obvious.
That made a beginner operator guess instead of choosing deliberately.

This release keeps the command read-only while making the boundary visible.

## Behavior

- Supported projection surfaces are:
  `generic_surface`, `private_workspace`, `rss_feed`, `static_site`, and
  `wordpress_private_blog`.
- `notion` is explained as a surface prototype, not a projection-plan surface;
  use `zet-surface-prototype --surface-kind notion` for that preview.
- Unsupported surfaces do not prevent a safe zet ref from being resolved, so
  blocked output can still include safe zet metadata and body hashes.
- Blocked JSON output still avoids echoing unsafe raw URLs or local paths.

## Safety

The command renders no body output, writes no projection records or receipts,
calls no providers, publishes nothing, runs no ZET transport, and reads no
credentials or secret values.
