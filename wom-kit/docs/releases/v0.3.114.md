# v0.3.114 - Mint Duplicate Scale And Retire Edge-Only Gap

Date: 2026-06-19

## What Changed

This release handles the eleventh real-use feedback sweep after v0.3.113.

Two mint lifecycle issues changed:

- mint duplicate checks can use the generated zettel index instead of rereading
  every canonical zet body for every mint;
- `retire-draft` now accepts the same approved edge-only target SHA history that
  `doctor` and `validate` already accepted.

## Mint Duplicate Scale

`archive mint-zet --dry-run|--approve` still performs the same duplicate
checks:

- proposed canonical path already exists,
- same zettel id,
- same title with the same body start,
- very similar body start.

When `db/archive-index.sqlite` is current, those comparisons now read canonical
id/title/body-start data from the generated index. The command returns
`duplicate_check` metadata showing whether it used the generated index or fell
back to the live scan.

Approved mint also upserts the newly minted canonical row into the generated
index when the fast path was used, so large mint loops can continue using the
index across many approved mints.

If the index is missing or stale, mint falls back to the existing live scan and
keeps the same duplicate semantics.

## Retire Edge-Only Gap

`retire-draft` now shares the same target SHA reconstruction helper as
`doctor` and `validate`.

If the current canonical zet differs from the mint receipt target SHA only
because approved post-receipt zettel-edge writes added frontmatter edges,
`retire-draft` can continue with a warning. Body edits, non-edge frontmatter
edits, missing edge receipts, or arbitrary SHA drift still block.

## Safety Boundary

This release does not:

- skip duplicate checks,
- weaken same-id or same-body duplicate detection,
- trust stale generated indexes,
- auto-rebuild the full generated index during dry-run,
- rewrite existing mint receipts,
- rewrite existing edge receipts,
- accept body edits as historical target SHA evolution,
- remove canonical zets,
- delete mint receipts or draft snapshots,
- call providers,
- add MCP write tools.
