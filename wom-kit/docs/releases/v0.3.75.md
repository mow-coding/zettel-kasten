# v0.3.75 - AI Response Concept Guide

Date: 2026-06-16

v0.3.75 adds a beginner-facing AI response guide for a real first-use confusion:
does WOM point at a file location like a cloud URL, or does it point at the
content identity of the source objet?

## Added

- Added `wom-kit/docs/ai-response-concept-guide.md`.
- The guide gives AI runtimes a standard explanation pattern for:
  - `sha256:<hex>` object identity vs a local/cloud location,
  - object manifests as catalogs of known objects and safe location labels,
  - `zet` records as human-authored memory that cites evidence,
  - the `objet -> derived text -> zet` layer split,
  - safe next-question routing during intake.
- Added short English and Korean beginner scripts that explain:
  - the sha256 value is the fingerprint of the bytes,
  - R2 or a local folder is only a shelf where a copy may live,
  - upload/availability must not be claimed without a receipt or future adapter
    verification.
- Linked the guide from:
  - `wom-kit/docs/public-documentation-map.md`,
  - `wom-kit/docs/public-documentation-map.ko.md`.

## Safety

- This is documented-only.
- It adds no command, MCP tool, provider call, object upload, source read,
  derived-text capture, draft, mint, transport, trust mutation, or write
  behavior.
- It explicitly tells AI runtimes not to overclaim remote availability, provider
  URLs, manifest proof, or derived-text authority.
