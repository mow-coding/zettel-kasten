# v0.3.93 - Saved View Facet Role Diagnostics

Date: 2026-06-17

## Added

- `facet_role_summary` and `facet_roles` in read-only
  `archive view-health <archive-root> --dry-run`.
- The same structured output through MCP `view_health`.
- Text output counts for navigation, internal, and unknown facet keys.

## Why

Phase 3 real-use feedback showed that imported archives can mix human
navigation facets and import-machine metadata in the same indexed facet table.
That makes AI navigation view repair harder, because useful axes such as
`subject` or `institution` sit next to internal keys such as `notion_status`,
`migration_batch`, or `contents`.

This release makes that noise visible without rewriting the archive.

## Behavior

- Classifies known navigation keys such as `domain`, `subject`,
  `institution`, `record_type`, `source_category`, `topic`, and `tags`.
- Classifies known internal/import keys and prefixes such as `notion_*`,
  `migration_*`, `import_*`, and `contents`.
- Leaves unfamiliar keys as `unknown` so a human can decide whether they are
  navigation axes or machine metadata.
- Adds next safe actions when saved views use internal/unknown keys, or when
  internal/unknown facet keys exist beside navigation keys.

## Safety

The update is read-only. It writes no view files, rewrites no zettel facets,
rebuilds no index, reads no zettel bodies or object bytes, calls no providers,
and echoes no zettel titles, absolute local paths, provider URLs, account ids,
emails, tokens, or secret values.
