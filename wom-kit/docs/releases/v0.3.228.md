# v0.3.228 - Actionable Full-Doctor Results And Current-Stage Progress

v0.3.228 makes one completed runtime-context full-Doctor run actionable. It
also fixes a stale compact heartbeat during local-profile secret-safety and
reduces repeated path-resolution work in that scan. It is additive and
requires no archive migration.

## What Changed

- Full runtime-context and AI start-here results now expose
  `doctor_findings`.
- ERROR/WARN severity and diagnostic-code counts are complete even when the
  detailed item list is bounded.
- Results retain up to 100 detailed findings and up to 20 deduplicated
  suggested commands, with explicit truncation flags.
- Each detailed item may carry severity, code, message, archive-relative path,
  hint, suggested command, and compatibility target.
- INFO findings remain count-only to keep the handoff surface compact.
- Outside absolute paths become `<local-path-redacted>` under the default
  redaction policy.
- Diagnostic text that still contains an outside local path or provider URL
  becomes `<sensitive-diagnostic-text-redacted>`.
- Compact `local-profile-secret-safety` heartbeat now reports the current
  `checked_files`, `content_scanned`, `local_profiles`, and `skipped_dirs`
  counts instead of a preserved edge-source aggregate.
- Ordinary files reuse the resolved boundary of their already-checked walked
  directory. Symlinks retain individual resolved-containment and ignored-target
  checks.

## Recovery Boundary

A v0.3.227 count-only result cannot truthfully reconstruct ERROR/WARN details
that were never stored. There is no honest zero-rerun decoder for that old
result. Run the complete command once after updating:

```text
archive runtime-context <archive-root> --full-doctor --progress --format json
```

The new result retains the actionable findings. Do not run a second broad
Doctor merely to identify the same findings. Review any suggested commands
before executing them; this release does not auto-repair archive data.

## Synthetic Benchmark

The public benchmark creates only a temporary synthetic archive:

```text
python tools/benchmark_local_profile_secret_safety.py --file-count 5000 --format json
```

On the release workstation, a 5,000-file before/after comparison reduced the
scan from 20.838 seconds to 11.644 seconds, about 44% less elapsed time and
1.79x throughput. A separate run of the shipped benchmark completed in
13.693172 seconds at about 365.219 files per second. These are synthetic local
workstation observations, not a promise for real archives; file sizes, storage,
antivirus, and machine load can change the result.

The benchmark reports `real_archive_read: false`,
`persistent_files_written: false`, `file_count`, checked/content counts,
`elapsed_seconds`, and throughput. It performs no provider, credential, or
network access.

## Safety And Privacy

This release changes result retention, progress routing, and read-only scan
implementation. It writes no archive file, receipt, manifest, zet, objet,
provider binding, credential, or repair. Detailed paths remain archive-relative
or redacted. Unknown or path-bearing compact progress payloads still fail the
fixed allowlist.
