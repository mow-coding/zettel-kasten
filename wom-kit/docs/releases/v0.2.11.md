# v0.2.11 Delegate Capability Contract

`v0.2.11` turns the post-`v0.2.10` delegation philosophy into a stronger dry-run contract.

Delegation is now modeled as an attestation-bound capability rather than a reusable public link.

## Added

- `archive delegate-zet --target-policy counterparty_bound|claimable_once`
- `claimable_once` delegate previews that can omit `--target-archive`
- `delegation_capability` preview fields
- `claim_binding` previews in attestation receipts and anchor metadata
- `settlement_condition: {mode: "none"}` as the explicit non-financial default
- MCP parity for `delegate_zet_check`

## Compatibility

Existing `counterparty_bound` delegate calls continue to work:

```bash
archive delegate-zet ./archive \
  --view view.example \
  --target-archive archive:team:example \
  --counterparty-id archive:team:example \
  --counterparty-fingerprint SHA256:example \
  --dry-run
```

New one-time claimable previews can defer the recipient until attestation:

```bash
archive delegate-zet ./archive \
  --view view.example \
  --target-policy claimable_once \
  --dry-run
```

`v0.2.10` delegate receipts that do not contain `delegation_capability` are treated as legacy `counterparty_bound` receipts.

## Not Implemented

This release does not implement:

- real delegate receipt writes,
- real claim registry,
- spent registry,
- revocation registry,
- P2P transport,
- relays,
- blockchain settlement,
- payment integration.

## Upgrade

No private archive migration is required.

```bash
git fetch --tags
git checkout v0.2.11
```

## 한국어 요약

`v0.2.11`은 `delegate`를 public link가 아니라 attestation-bound capability로 다루기 위한 dry-run 계약 패치입니다.

핵심 변경:

- `delegate-zet --target-policy counterparty_bound|claimable_once`
- `claimable_once`에서는 `--target-archive` 없이 preview 가능
- `delegation_capability`, `claim_binding`, `settlement_condition` preview 추가
- settlement는 아직 `mode: "none"`만 기록
- 실제 P2P, claim/spent registry, blockchain, payment는 아직 구현하지 않음
