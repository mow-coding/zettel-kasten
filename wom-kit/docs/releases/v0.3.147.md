# v0.3.147 - Derived Artifact Staleness

v0.3.147 adds a read-only freshness check for the real-use problem where a
zettel changes, but an external report or other derived artifact may still be
showing older information.

## What Changed

- Added read-only `archive derived-artifact-staleness`.
- Added aliases `archive report-staleness` and `archive artifact-staleness`.
- The check reports:
  - artifacts whose source zets are newer than `last_synced_at`,
  - artifacts missing `source_zettels`,
  - artifacts with unresolved source zettel refs,
  - artifacts with no sync timestamp.
- Added next-action guidance for stale report review and missing metadata
  cleanup.
- Added CLI tests covering stale detection, read-only behavior, and privacy-safe
  output.

## Command Shape

```powershell
archive derived-artifact-staleness <archive-root> `
  --dry-run `
  --format json
```

```powershell
archive report-staleness <archive-root> --dry-run
```

## Safety Boundary

The check:

- writes nothing,
- calls no providers,
- opens no external report body,
- edits no zets,
- syncs no reports,
- echoes no artifact refs, zet titles, zettel body text, provider URLs, local
  absolute paths, tokens, or secret values.
