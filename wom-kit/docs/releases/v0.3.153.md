# v0.3.153 - Approval Handoff Audit

v0.3.153 adds a read-only audit command for approval handoff records.

## What Changed

- Added `archive approval-handoff-audit`.
- Added aliases `archive handoff-audit` and
  `archive human-approval-handoff-audit`.
- The audit validates:
  - handoff record path boundary,
  - schema,
  - expected status,
  - expected operation kind,
  - `reviewed_by` presence for `approved_once`,
  - that the record itself did not execute an operation, read private material,
    or call a provider.
- The audit returns `future_operation_authorized: true` only for a matching
  `approved_once` handoff.

## Safety Boundary

The command:

- reads only the handoff metadata record,
- reads no private material,
- calls no providers,
- checks no network,
- executes no operation,
- echoes no target ref values,
- echoes no requested action values,
- echoes no local absolute paths, tokens, or secret values,
- writes nothing.

## Still Future

- Stale approval detection when target metadata changes.
- Command-specific enforcement for live fetch, write, publish, and credential
  operations.
