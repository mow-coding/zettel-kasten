# v0.3.79 - Connection Edge Vocabulary

Date: 2026-06-16

Status: v0.3.79 read-only connection edge vocabulary checkpoint

v0.3.79 continues the #10 connection-import feedback path by adding the
recommended edge vocabulary to the WOM model before any evidence parser or
approval-gated edge writer exists.

## Added

The base `zettel-kasten/types.yml` and fake archive types now define:

- `material`
- `derived`
- `semantic`
- `embed`
- `mention`
- `view_query`
- `comment_context`

## Changed

- `connection-import-plan` still maps Notion relation properties, synced block
  references, database view/filter snapshots, internal links, page mentions,
  comment context, and objet embeds to candidate WOM edge types.
- The planner now sees the recommended vocabulary as present in the fake archive
  instead of warning that every recommended type is missing.
- Dynamic database view/filter results still require reviewed static snapshots
  before durable edges can be written.

## Safety Boundary

This release still does not implement a Notion evidence parser or edge writer.

The workflow remains read-only: no Notion call, OAuth, export read, comment
read, media download, zet write, edge write, receipt write, object manifest
update, provider URL, local path, page title, comment body, account id, email,
token, or secret value is produced.
