# v0.3.35 - Derived Text Toolchain Doctor

Date: 2026-06-15

v0.3.35 adds a read-only doctor for the derived-text extraction toolchain.

## Added

- CLI `archive derive-text doctor --dry-run`.
- CLI alias `archive derive-text-doctor --dry-run`.
- Boolean tool readiness checks for:
  - Python modules: `docx`, `openpyxl`, `pptx`, `fitz`,
  - executables: `soffice`, `libreoffice`, `tesseract`, `hwp5txt`,
  - policy-dependent future routes such as local ASR.
- Family readiness output for text/markup, OOXML Word, spreadsheets,
  presentations, legacy Office, HWP/HWPX, PDF, image scan, and audio.

## Why

The coverage gate can say which textual objets still need derived text. The
toolchain recommendation can say which route to try. The doctor adds the next
safe preflight: whether the local environment appears ready for those routes.

## Privacy Boundary

The doctor output does not echo:

- executable paths,
- Python import paths,
- local absolute paths,
- source filenames,
- source bodies,
- provider URLs,
- usernames,
- secret values.

It reports only tool names, probe labels, booleans, route families, and missing
readiness categories.

## Closed Actions

The doctor does not:

- install tools,
- read source bytes,
- run OCR,
- run parsers,
- run ASR,
- call LLM vision,
- call providers,
- write derived text,
- write receipts,
- draft zets,
- mint zets.
