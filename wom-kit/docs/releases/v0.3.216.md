# v0.3.216 - one-process strict catalog pass

v0.3.216 removes repeated frontmatter scans from a complete CLI catalog pass
without introducing a persistent private-data cache. It is additive and
requires no archive migration.

## What Changed

- Added `zet-catalog-pass`, with `catalog-pass` and `zet-catalog-drain` aliases.
- Reused the existing strict continuation, process-memory snapshot, compact
  continuation, and completion-revalidation contracts in one invocation.
- Added complete-only `.wom-scratch/diagnostics/*.jsonl` output with header,
  page, and footer records.
- Added exclusive hidden partials, no-overwrite publication, handled-failure
  cleanup, a 256 MiB default output bound, and content-free progress.
- Updated AI runtime guidance to prefer the one-command pass for terminal CLI
  operation and to read/delete the private JSONL incrementally.

## Honest Boundary

The JSONL can contain private ids, titles, abstracts, facets, ties, and edges.
It contains no zet bodies and must not be committed. A forced process
termination can leave a hidden private partial; later runs report only its count
and do not auto-delete possible concurrent work.

The command persists no cache, map, index, goal, loop, archive record, receipt,
or external DB row. It calls no provider or model. Ordinary paged CLI and MCP
behavior are unchanged.

## Verification

- Multi-page fake-archive pass parses each frontmatter once and publishes every
  selected node exactly once.
- A snapshot change before the completing page blocks and leaves no complete
