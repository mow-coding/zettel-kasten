# v0.2.14 WOM Safe HTML Profile

Release date: 2026-05-23

Status: public pre-release

## Summary

`v0.2.14` is a compatible documentation/spec baseline patch.

It does not change CLI behavior, archive writes, schemas, MCP tools, or private archive layout. Its purpose is to make the product language and long-term document-format direction clear before the next implementation batch.

## What Changed

- `WOM` is defined as the whole system, worldview, and open infrastructure.
- `zet` is defined as the unit document minted inside a zettel-kasten.
- `ZET` is defined as the zettel-kasten-based communication layer that can become messenger, SNS/feed, or collaboration depending on relationship topology.
- `WOM Safe HTML Profile` is documented as the long-term canonical/interchange/rendering target for zets.
- Markdown remains the v0.2 authoring/import compatibility format.

## Why This Matters

The format question is not only Markdown versus HTML.

The current zettel-kasten runtime is AI/CLI/MCP first. That is good for local AI-assisted writing, but WOM is also meant to be open infrastructure that others can use to build custom zettel-kasten and `ZET`-based SaaS surfaces.

Future galleries, feeds, workspaces, media viewers, dashboards, and domain-specific tools need a web-native rendering target. The target cannot be arbitrary HTML, because archive memory must stay safe, deterministic, AI-readable, accessible, and provenance-preserving.

That is the role of WOM Safe HTML Profile.

## Compatibility

No private archive migration is required.

Existing Markdown-compatible zets remain valid in the v0.2 compatibility line.

The following are not implemented in this release:

- Markdown-to-WOM-Safe-HTML conversion,
- profile validator,
- HTML migration,
- UI,
- live sharing,
- P2P transport,
- external provider sync.
