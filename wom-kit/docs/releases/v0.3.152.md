# v0.3.152 - Input Provenance Taxonomy

v0.3.152 adds a read-only taxonomy for AI-visible input provenance.

## What Changed

- Added `archive input-provenance-taxonomy`.
- Added aliases `archive provenance-taxonomy` and `archive caller-input-taxonomy`.
- Added a machine-readable taxonomy for:
  - `tool_discovered`,
  - `receipt_verified`,
  - `human_selected`,
  - `caller_supplied`,
  - `ai_generated`,
  - `fixture_supplied`,
  - `environment_inferred`,
  - `unknown`.
- Marked caller-supplied, AI-generated, fixture-supplied, environment-inferred,
  and unknown inputs as unverified source truth.
- Added AI operator checks so supplied inputs are not described as tool-discovered
  or receipt-verified.

## Safety Boundary

The command:

- reads no archive body text,
- echoes no input values,
- echoes no source values,
- calls no providers,
- checks no network,
- echoes no local absolute paths, tokens, or secret values,
- writes nothing.
