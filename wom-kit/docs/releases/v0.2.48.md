# v0.2.48 Release Notes

Date: 2026-05-27

## Summary

v0.2.48 adds the ZET radio-frequency recommendation model baseline.

This is a documentation and sanitized-example release only. It does not add CLI, MCP, schema, provider, transport, ranking, or feed behavior.

## Added

- [ZET Radio-Frequency Recommendation Model](../zet-radio-frequency-recommendation-model.md).
- Sanitized non-executable selector shape example under `wom-kit/examples/zet-radio-frequency-recommendation/`.
- Public-safe work log for the v0.2.48 recommendation model baseline.

## Compatibility

No private archive migration is required.

No CLI or MCP behavior changed.

The release clarifies that:

- followed/neighbor feeds should come from explicit relationships, delegates, groups, and permissions,
- recommended/broadcast feeds may exist later, but should use user/node-owned and inspectable selector logic,
- a prompt-as-algorithm selector may be a prompt-like policy, rule bundle, config, code, or future signed policy artifact,
- the radio-frequency metaphor means a node chooses a channel/frequency-like scope to tune, not real radio hardware or real transport.

## Safety Boundary

v0.2.48 does not fetch recommendations, rank feeds, update neighbor feeds, call providers, publish to WordPress, write projection records, write receipts, run real ZET transport, create trust/import/acceptance/attestation/signature/minting changes, add background workers, train models, run payments/staking/consensus/blockchain, or enable full-auto behavior.
