# v0.3.15 - Project Intake Unpack Choice Receipt

Release date: 2026-06-14

v0.3.15 adds the next manual "unpacking boxes" step after the opaque unpack
queue: a reviewed receipt for one human-confirmed queue choice.

## Added

- CLI `archive project-intake-unpack-choice --choice <json> --receipt <receipt>
  --staged-folder <folder> --dry-run|--approve`.
- MCP `project_intake_unpack_choice` with the same service boundary.
- Schema `wom-kit/project-intake-unpack-choice/v0.1` for recording one
  `item-0001` style ref, one intended action, and `human_confirmed: true`.
- Approved receipts under
  `receipts/project-intake-unpack/*.project-intake-unpack-choice.json`.

## Boundaries

- Requires a completed project-intake decisions receipt before approval.
- Validates the current opaque queue and records a public-safe queue digest.
- Command output does not expose staged entry names, local paths, file bodies,
  or choice notes.
- It does not run source-intake, capture objets, create drafts, mint zets, call
  providers, upload, sync, or clean staged folders.

## Upgrade

No archive migration is required for v0.3.14 users.
