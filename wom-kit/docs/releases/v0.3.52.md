# v0.3.52 - IMAP Mailbox Adapter Manifest Schema

Date: 2026-06-16

v0.3.52 adds a JSON Schema for the non-secret IMAP mailbox adapter manifest
preview introduced in v0.3.51.

The release still does not connect to mail providers. It only makes the future
adapter declaration contract machine-checkable.

## Added

- `wom-kit/schemas/imap-mailbox-adapter-manifest.schema.json`.
- Schema validation inside `archive imap-mailbox-adapter-manifest-plan --dry-run`.
- Schema validation metadata inside MCP `imap_mailbox_adapter_manifest_plan`.
- Documentation and capability-matrix links for the schema baseline.

## Verify

```bash
archive imap-mailbox-adapter-manifest-plan <archive-root> \
  --adapter-id local-imap \
  --provider gmail \
  --provider naver \
  --operation header_metadata_scan \
  --selection-rule newest_first \
  --selection-rule unread_first \
  --dry-run \
  --format json
```

The JSON should include:

```text
schema_validation.ok: true
schema_validation.schema_name: imap-mailbox-adapter-manifest.schema.json
manifest_preview.schema: wom-kit/imap-mailbox-adapter-manifest/v0.1
```

## Safety

The planner remains read-only. It:

- writes no adapter manifests,
- executes no live adapters,
- opens no IMAP connection,
- attempts no login,
- selects no mailbox,
- searches no mailbox,
- lists no candidate messages,
- reads no IMAP UIDs or Message-ID values,
- reads no headers, bodies, or attachments,
- creates no derived text,
- retrieves no secrets,
- starts no OAuth,
- calls no providers,
- echoes no email addresses, username values, exact account refs, exact
  credential refs, exact mailbox refs, IMAP host values, provider URLs, message
  ids, subjects, sender or recipient values, attachment names, approval receipt
  paths, selection receipt paths, local absolute paths, tokens, or secret values.

No archive migration is required.
