## WOM Context Recommendation

For a beginner local offline KeePassXC 2.7.x vault, WOM recommends:

```text
Database format: KDBX 4.0
Encryption algorithm: AES-256
Key derivation function: Argon2d
KDF tuning: leave KeePassXC automatic or recommended values alone
```

The Argon2d recommendation is scoped to a local offline KDBX file threat model,
where the main risk is a stolen database being attacked offline with GPU/ASIC
hardware. Generic Argon2id advice is common for broader server-side password
hashing discussions, but this first-vault setup is not that context.

Workplace, school, or regulated-environment policy still wins when present.

## Safety

The manual remains read-only. It:

- opens no KeePassXC window,
- creates no database,
- records no database path,
- reads no master password,
- reads or writes no secret values,
- reads no environment variables,
- calls no providers,
- installs no tools,
- writes no files.

No archive migration is required.
