# v0.3.131 - Notion Ancestor Fetch Execution Subject Boundary

Release date: 2026-06-22
Status: prerelease

v0.3.131 clarifies the one remaining ambiguity in the Notion ancestor recovery
loop: who runs the future live fetch.

The answer is: a future WOM local credential-bounded adapter process, after
human scope review and credential approval. The AI chat runtime is not the live
fetch subject, and client-supplied ancestor fixtures are only sanitized
safe-origin fallback input. This release does not ask clients or client-side AI
to hand-roll provider crawling.

The contract now reports `execution_actor_contract`:

- current live fetch execution subject: `none_contract_preview_only`,
- intended future execution subject:
  `future_wom_local_credential_bounded_adapter_process`,
- AI chat runtime role: plan, review, and verify only,
- client fixture role: sanitized safe-origin fallback input only,
- no AI hand-rolled provider crawl,
- no client hand-rolled provider crawl requirement,
- no credential values in AI context.

## Still Closed

This release still does not add live Notion transport. It does not call
providers, start OAuth, open Notion connections, retrieve credential values,
read page titles or bodies, read comments, download media, write queue files,
write ancestor result fixtures, write zets, mint pages, write edges, write
receipts, update object manifests, or echo exact credential refs, provider URLs,
local paths, account ids, emails, tokens, or secret values.
