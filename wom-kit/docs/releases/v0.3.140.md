# v0.3.140 - Tiro Live Lossless Recovery Fetch

v0.3.140 turns the v0.3.139 Tiro recovery contract into the first live,
approval-gated fetch path.

## What Changed

- Added `archive tiro-lossless-recovery-fetch-run`.
- Added alias `archive tiro-recovery-fetch-run`.
- Added local `env:` credential execution for the live Tiro REST adapter.
- Added raw recovery bundle writes under `workbench/`.
- Added non-secret fetch receipts under `receipts/tiro/lossless-fetches/`.
- Updated the Tiro lossless recovery documentation, capability matrix, README,
  changelog, and tests.

## Tiro Fetch Flow

The intended sequence is now:

```powershell
archive tiro-lossless-recovery-plan <archive-root> --credential-ref env:WOM_TIRO_API_KEY --dry-run --format json
archive tiro-lossless-recovery-fetch-run <archive-root> --credential-ref env:WOM_TIRO_API_KEY --workspace-guid <workspace-guid> --output workbench/tiro-lossless-recovery.live.json --dry-run --format json
archive tiro-lossless-recovery-fetch-run <archive-root> --credential-ref env:WOM_TIRO_API_KEY --workspace-guid <workspace-guid> --output workbench/tiro-lossless-recovery.live.json --approve --reviewed-by human:<id> --format json
archive tiro-lossless-recovery-capture <archive-root> --bundle workbench/tiro-lossless-recovery.live.json --dry-run --format json
archive tiro-lossless-recovery-capture <archive-root> --bundle workbench/tiro-lossless-recovery.live.json --approve --reviewed-by human:<id> --format json
```

Dry-run fetch mode writes nothing, reads no credential value, and calls no
provider. Approved fetch mode reads the named local environment variable, calls
Tiro REST endpoints, writes the raw bundle privately, and writes a non-secret
fetch receipt.

## Privacy Boundary

The raw bundle is private archive material and may contain meeting titles,
transcript text, participant names, generated documents, wiki data, and provider
URLs. That is intentional: lossless recovery must preserve the official source
data before AI correction or enrichment.

Command output and receipts do not echo the credential ref, environment
variable name, token, meeting title, transcript text, participant name, provider
URL, raw provider response, or secret value.

## Remaining Gaps

This release still keeps several layers separate:

- keyring, vault, wallet, or OS password-manager credential reads,
- original audio byte retrieval when no official REST endpoint is confirmed,
- AI enrichment writes,
- derived-text capture,
- zet drafting,
- minting.

When audio bytes cannot be fetched, the bundle records an
`audio_original_bytes` gap instead of pretending the recovery is complete.

## Upgrade Notes

No archive migration is required.

Existing v0.3.139 users can keep using `tiro-lossless-recovery-capture` for a
reviewed local export. New users who have a Tiro API token in a local
environment variable can now use `tiro-lossless-recovery-fetch-run` to generate
the raw bundle first.
