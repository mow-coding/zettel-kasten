# v0.2.20 GitHub Profile Repository Setup Planner

Status: public pre-release patch

Date: 2026-05-25

## Summary

`v0.2.20` adds a dry-run-first GitHub repository setup planner for WOM profiles.

The planner helps an AI runtime avoid assuming the current/default archive when a user asks to set up a repository for a specific profile. It proposes safe local metadata and manual steps, but it does not create or connect any GitHub repository.

## Added

- `archive github-repo <archive-root> --dry-run --format json`
- Default private repo naming:

```text
zettel-kasten-<profile_slug>
```

- Safety gates for:
  - ASCII profile slugs,
  - `zettel-kasten-` repository prefix,
  - GitHub owner/account references,
  - path-like, URL-like, email-like, and secret-like values.
- `--approve --reviewed-by` local-only approval for:
  - `provider-bindings.yml`,
  - provider setup receipts under `receipts/providers/`.
- Optional ignored local profile/account hints through `--write-local-profile`.
- Read-only MCP `github_repository_setup_plan`.
- Rollback protection for approved local metadata writes, so `provider-bindings.yml` is not left modified if the matching setup receipt or optional ignored local profile cannot be written.

## Compatibility

No private archive migration is required.

Existing CLI and MCP commands remain compatible.

Approved CLI mode writes only local archive metadata. It does not create repositories or configure Git.

Approved metadata writes are guarded so a failed receipt/local-profile write does not leave a GitHub provider binding without its receipt.

## Non-Goals

This release does not:

- create GitHub repositories,
- run OAuth,
- call GitHub APIs,
- run `gh`,
- run `git remote`,
- push,
- sync,
- add MCP GitHub apply/create/connect/push/sync tools,
- rename WOM-kit,
- change WOM/zet/ZET philosophy.
