# v0.3.142 - Notion OAuth Connection Preflight

v0.3.142 moves the Notion "Connect Notion" path one step closer to a usable
beginner flow without opening the live OAuth gate too early.

This release adds the secret-blind local OAuth runtime contract as a preflight,
not live OAuth execution.

## What Changed

- Added `archive notion-oauth-connection-preflight`.
- Added aliases `archive notion-oauth-preflight` and
  `archive notion-connect-oauth-preflight`.
- Added safe ref validation for the future OAuth client id, client secret,
  optional state store, and token store.
- Added local loopback callback URI validation.
- Rejected plain env storage for future access/refresh tokens.
- Added `wom-kit/docs/notion-oauth-connection-preflight.md`.
- Updated the Notion connection plan, capability matrix, README links,
  changelog, public documentation maps, and tests.

## Contract

The intended flow is still:

```text
Connect Notion -> human approves once in browser -> trusted local runtime stores tokens -> AI receives only sanitized results
```

The new command checks that the future local runtime can stay secret-blind:

- AI does not see client secrets.
- AI does not see authorization codes.
- AI does not see access or refresh tokens.
- AI does not see exact credential refs or redirect URIs.
- Token exchange and token storage belong to a trusted local runtime, not the
  chat runtime.

## Still Future

- browser OAuth authorization,
- managed local callback server,
- authorization URL construction,
- OAuth token exchange,
- keyring/vault token storage,
- Notion recovery from stored OAuth tokens,
- UI button surface.

## Upgrade Notes

No archive migration is required.

Existing `archive notion-recover` env/file token handoff remains available as a
power-user fallback. v0.3.142 only adds the read-only local readiness contract
for the future OAuth path.
