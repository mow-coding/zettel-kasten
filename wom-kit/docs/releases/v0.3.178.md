# v0.3.178 - operator progress for doctor and adopt

v0.3.178 adds opt-in progress output for two long-running operator surfaces:
`archive doctor` and `archive object-storage-adopt-existing`. It is additive: no migration and
no default output change unless `--progress` is passed.

## The problem this fixes

The basoon v0.3.176 verification letter reported that large `doctor --strict` and
`object-storage-adopt-existing --key-map` runs could stay silent for many minutes while CPU was
active. Even when a command is correctly working, a beginner operator cannot tell whether it is
alive, which stage is active, or whether they should wait or stop.

## The fix

### `archive doctor --progress`

`Doctor` already had internal stage-progress plumbing. The CLI now exposes it with:

```bash
archive doctor <archive-root> --strict --progress
```

Progress goes to stderr as stage start/done heartbeats such as:

```text
[doctor] zettels: start elapsed=...
[doctor] zettels: done elapsed=...
```

The normal text and JSON diagnostic payloads are unchanged.

### `object-storage-adopt-existing --progress`

`object-storage-adopt-existing` now accepts a progress callback at the service layer and exposes
it in the CLI:

```bash
archive object-storage-adopt-existing <archive-root> ... --approve --progress --format json
```

Progress is emitted for:

- `adopt-plan` while WOM resolves object ids to remote keys;
- `adopt-declare` for declared, non-gating adopts;
- `adopt-verify` for verified presence-only HEAD checks.

For large loops, reporting is throttled to the first object, every 100th object, and the final
object, so a 19k-object run gives a useful heartbeat without flooding the terminal.

## Privacy

Progress output contains only fixed stage labels, counts, elapsed time, and ETA. It does not
print object ids, remote keys, bucket names, provider URLs, exact credential refs, tokens,
secret values, provider response bodies, or local absolute paths.

## Tests

Added focused coverage for:

- `doctor --progress` emitting stage start/done lines;
- `object-storage-adopt-existing` progress callbacks emitting plan and verified-HEAD stages;
- adopt progress events carrying no object id or prefix value.

## Compatibility

Additive; **no migration required**. Without `--progress`, output remains as before.
