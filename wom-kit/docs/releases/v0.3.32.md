# v0.3.32 - KeePassXC Command Preflight

Date: 2026-06-15

v0.3.32 adds a read-only KeePassXC command preflight.

## Added

- CLI `archive credential-keepassxc-command-plan --dry-run`.
- CLI aliases:
  - `archive keepassxc-command-plan --dry-run`
  - `archive credential-keepassxc-write-plan --dry-run`
- MCP `credential_keepassxc_command_plan`.
- `wom-kit/docs/credential-keepassxc-command-plan.md`.

## What It Does

The command verifies a written credential access approval receipt through
`credential_policy_check`, then returns a safe command shape:

```text
keepassxc-cli add --password-prompt <database.kdbx selected by human outside WOM> <entry>
```

The plan requires:

- `--approval-receipt <path>`,
- `--entry-label <safe-label>`,
- `decision=approve_once` in the receipt,
- `store_kind=password_manager`,
- `adapter_kind=keepassxc_cli`,
- a matching credential id, action kind, consumer, and archive id.

## What It Still Does Not Do

This release does not:

- run `keepassxc-cli`,
- open KeePassXC,
- open any password manager,
- store a `.kdbx` path,
- read a database password,
- read a plaintext secret file,
- detect secret values,
- pipe secrets to stdin,
- put secret values in argv,
- write a vault entry,
- write a keyring entry,
- call providers,
- start OAuth,
- write files.

`live_execution_allowed_now` remains `false`.

## Safety Chain

```text
credential-store-recommendation
-> credential-vault-onboarding-plan
-> credential-plaintext-migration-plan
-> credential-ref-plan
-> credential-ref-inventory
-> credential-access-broker-plan
-> credential-access-approval --approve
-> credential-policy-check --approval-receipt <path>
-> credential-keepassxc-command-plan --approval-receipt <path>
-> future live KeePassXC adapter execution
-> credential-adapter-audit-plan
```

This is still a preflight layer, not a vault adapter.
