# v0.3.126 - Notion Local Tree Fixture And Ancestor Merge

Release date: 2026-06-22
Status: prerelease

v0.3.126 closes more of the local Notion nested-tree recovery loop after
v0.3.125. WOM-kit can now build a nested-tree fixture preview from reviewed
block mirror metadata, and it can merge sanitized ancestor result nodes into a
tree preview before immediately replanning recovery.

## Added

```powershell
archive notion-block-mirror-tree-fixture-plan <archive-root> `
  --mirror workbench/notion-block-mirror.sample.json `
  --source notion `
  --dry-run `
  --format json
```

MCP:

```text
notion_block_mirror_tree_fixture_plan
```

This command derives safe `node_ref`, `parent_ref`, `content_class`, and
`source_status` values from reviewed block mirror metadata without reading page
titles or bodies. It returns both `nested_tree_fixture_preview` and
`nested_tree_plan_preview`.

```powershell
archive notion-ancestor-merge-plan <archive-root> `
  --tree workbench/notion-nested-tree.sample.json `
  --ancestors workbench/notion-ancestor-result.sample.json `
  --source notion `
  --dry-run `
  --format json
```

MCP:

```text
notion_ancestor_merge_plan
```

This command merges sanitized ancestor result nodes into a tree preview and
returns `nested_tree_plan_after_merge`, so a previously untraceable leaf can
move from `hold_queue` into `recovery_queue` or `structure_skip_queue` when the
parent chain is closed.

## Still Closed

This release still does not add live Notion deep-crawl transport. It does not
call providers, start OAuth, open Notion connections, read real export
directories, read page titles or bodies, read comments, download media, write
or merge fixture files, write zets, mint pages, write edges, write receipts,
update object manifests, or echo provider URLs, local paths, account ids,
emails, tokens, or secret values.
