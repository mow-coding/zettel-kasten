# v0.3.130 - Notion Ancestor Fetch Adapter Execution Contract

Release date: 2026-06-22
Status: prerelease

v0.3.130 adds the read-only execution contract for the remaining Notion ancestor
fetch adapter gap. It does not implement live Notion transport. It fixes the
contract a future credential-bounded adapter must satisfy before it can sit
between `notion-ancestor-crawl-plan` and `notion-ancestor-merge-plan`.

## Added

```powershell
archive notion-ancestor-fetch-adapter-execution-contract <archive-root> `
  --tree workbench/notion-nested-tree.sample.json `
  --source notion `
  --scope-generation-id DB1 `
  --credential-ref env:wom_notion_readonly `
  --dry-run `
  --format json
```

Aliases:

```text
notion-ancestor-fetch-execution-contract
notion-ancestor-crawl-adapter-execution-contract
```

MCP:

```text
notion_ancestor_fetch_adapter_execution_contract
```

The contract reports:

- credential ref presence without echoing the exact ref,
- scoped crawl request counts,
- future adapter input fields,
- sanitized ancestor-result fixture output fields,
- the next merge/replan command,
- closed actions and privacy guards.

## Still Closed

This release still does not add live Notion transport. It does not call
providers, start OAuth, open Notion connections, retrieve credential values,
read page titles or bodies, read comments, download media, write queue files,
write ancestor result fixtures, write zets, mint pages, write edges, write
receipts, update object manifests, or echo exact credential refs, provider URLs,
local paths, account ids, emails, tokens, or secret values.
