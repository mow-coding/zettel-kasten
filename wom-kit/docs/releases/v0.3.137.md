# v0.3.137 - Tiro Import Plan And Source Mirror Drift Check

Date: 2026-06-22

## Summary

v0.3.137 adds the first read-only WOM-kit planning gate for Tiro-style meeting
transcripts:

```text
archive tiro-import-plan
```

It also strengthens `archive version` so project-local WOM-kit source mirrors
can be checked against their actual package version, installed-version pin, and
latest fetched semver tag.

## Added

- CLI `archive tiro-import-plan <archive-root> --manifest <archive-relative-json>
  --dry-run --format json`.
- MCP `tiro_import_plan` for the same dry-run-only planning surface.
- Public fake archive manifest
  `workbench/tiro-meeting.sample.json`.
- Public documentation for the Tiro manifest shape, output contract, and safety
  boundary.
- `archive version` JSON field `project_source_mirror`, including source
  version, pyproject version, installed pin alignment, head tag, latest fetched
  tag, and source-mirror drift state when available.

## Safety Boundary

The Tiro planner remains read-only:

- no Tiro API call,
- no OAuth flow,
- no audio byte read,
- no ASR,
- no derived-text write,
- no draft,
- no mint,
- no meeting title, participant display name, transcript text, source URL,
  audio filename, local path, account id, email, token, or secret echoed.

The version check remains read-only and redacts local absolute paths by default.
It does not repair or update a project mirror automatically.

## Verification

- CLI tests cover Tiro plan structure and no transcript/title/link echo.
- MCP tests cover the read-only Tiro tool and allowed-root enforcement.
- CLI tests cover project source mirror drift, including latest fetched tag
  detection when local Git is available.
- Documentation and capability-matrix tests cover the new public command and
  version-truth source language.
