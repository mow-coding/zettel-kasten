# v0.3.72 - IMAP Material Capture Approval Audit

Status: v0.3.72 read-only IMAP material capture approval audit checkpoint
Date: 2026-06-16

v0.3.72 adds the read-only audit gate after an IMAP material capture approval
receipt exists and before any future command reads selected message bodies,
attachment bytes, or mail-derived text.

## Added

- `archive imap-mailbox-material-capture-approval-audit <archive-root>
  --material-selection-receipt <archive-relative-json> --approval-receipt
  <archive-relative-json> --capture-action
  message_body_capture|attachment_capture|derived_text_capture
  --expected-decision needs_review|approve_once|deny --dry-run
  --format json`.
- Aliases:
  - `archive imap-material-capture-approval-audit`
  - `archive mailbox-material-capture-approval-audit`
- A public documentation page:
  - [IMAP Mailbox Material Capture Approval Audit](https://github.com/mow-coding/zettel-kasten/blob/main/wom-kit/docs/imap-mailbox-material-capture-approval-audit.md)

## Behavior

The command validates one non-secret material selection receipt, reads one
non-secret material capture approval receipt, and verifies that the approval
receipt matches the selected receipt SHA-256, selected indexes, selected count,
candidate pool count, selection mode, capture action, expected decision,
future-adapter action flags, redaction flags, and closed-action flags.

## Safety Boundary

This release still does not implement message body capture, attachment capture,
or mail-derived text extraction.

The command reads no original execution receipt, opens no IMAP connection, reads
no environment variables, opens no keyring/password manager, reads no message
headers again, reads no message bodies, reads no attachments, creates no
derived text, writes no files, calls no providers, and echoes no approval
receipt path, material selection receipt path, execution receipt path,
candidate refs, subjects, senders, recipients, local paths, tokens, or secret
values.

## Verification

- CLI tests cover dry-run enforcement, approval receipt verification, expected
  decision mismatch blocking, candidate-ref non-echoing, private-marker
  non-echoing, no writes, and no material reads.
- Capability matrix and public documentation tests cover the new command,
  documentation link, safety boundary, and release notes.
