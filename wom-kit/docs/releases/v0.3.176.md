# v0.3.176 - content-free reconcile body-diff diagnostic (DX-only)

v0.3.176 adds a **content-free body-diff diagnostic** to the two reconcile plans so a client can
self-diagnose WHICH kind of sub-BOM difference triggered a `content_change` verdict — without
echoing any body bytes. It is **additive** and a **STRICT CLASSIFICATION NO-OP**: no migration,
no behavior change, and the classifier is byte-identical with and without it.

## The problem this fixes

The basoon 5th operator letter (§4) reported that a client ran
`remint-reconcile --strip-bom --dry-run` on a zet (`source_outline`) and got
`drift_class=content_change` + `body_changed:true` + ack required, even though they saw no
obvious post-BOM text edit.

The classifier is **correct** and must NOT change. `bytes_normalized_for_content_compare`
deliberately folds ONLY a single leading UTF-8 BOM and CRLF/CR→LF (the v0.3.167 locked "zero
Unicode normalization, no whitespace collapse" honesty). If two bodies still differ AFTER that
normalization, the residual is beyond BOM — trailing whitespace, a final-newline difference, an
NFC-vs-NFD Unicode-normalization difference, or a real content edit — and `content_change` is the
HONEST classification. Folding more would launder a real edit to `format_drift`, a cardinal sin.
So there is **no classifier change**.

What was missing was a way for the client to see WHICH kind of sub-BOM residual they hit, without
being handed the body bytes.

## The fix (DX only)

A new pure helper `_reconcile_body_diff_diagnostic(canonical_body, snapshot_body)` decorates the
reconcile plan with a `body_diff_diagnostic` dict. It reuses the LOCKED primitive
`bytes_normalized_for_content_compare` READ-ONLY on the SAME two forms the drift_class
body-identity proof compared, so the reported offset/delta explains the EXACT residual the
classifier folded down to. It emits ONLY non-content facts:

| field | type | meaning |
|---|---|---|
| `differs` | bool | `false` only on the defensive equal-after-normalization sentinel; else `true` |
| `category` | str | `final_newline_only` \| `trailing_whitespace_only` \| `unicode_normalization_only` \| `content_difference` (or `none` on the sentinel) |
| `first_differing_byte_offset` | int | first differing byte index in the normalized form (or `min_len` when one form is a strict prefix of the other); `-1` on the sentinel |
| `normalized_length_delta` | int | `len(canonical_norm) - len(snapshot_norm)` in bytes |
| `canonical_form` | str | `nfc` \| `nfd` \| `mixed` — set ONLY for `unicode_normalization_only`, else `n/a` |
| `snapshot_form` | str | same enum; `n/a` unless `unicode_normalization_only` |

No field ever holds a body substring, slice, `repr`, or character sample.

### Honesty rule (anti-laundering)

The category check is ORDERED (most specific first: final newline → trailing whitespace →
Unicode normalization) and a non-`content_difference` category is reported **ONLY when its
transform makes the two normalized bodies EQUAL TO EACH OTHER** (mutual, symmetric full
reconciliation — never tested against one side). If any residual survives the transform, that
category does NOT apply and we fall through to the honest terminal label `content_difference`. So
a diff that mixes whitespace or Unicode-normalization drift with a real content edit is **never**
mislabeled as benign. The `trailing_whitespace_only` transform is per-line `rstrip()` ONLY — never
internal, never leading, never `split()/join()` collapse, never unicode-whitespace folding — so an
internal-whitespace or leading-indentation edit stays `content_difference`.

## Strict classification no-op

- The diagnostic is computed **AFTER** the drift_class predicate and only added to the
  already-built output dict, exactly like the existing v0.3.172 strip-BOM preview. It never reads
  or influences `drift_class` / `classification_basis` / `content_change_ack_required`.
- `bytes_normalized_for_content_compare` and the drift_class predicate are byte-identical with and
  without this change (pinned by a byte-for-byte primitive test and a classification-baseline test).
- The key is emitted **only** when an anchor granted a body comparison AND the bodies changed
  (`body_changed is True`). It is **ABSENT** on Tier C / no-snapshot (`body_changed` None) and on
  `format_drift` (`body_changed` False), so it never shows a misleading residual.
- The helper is never called with `None` (the caller guards on `body_changed is True and
  snapshot_body is not None`), and it carries a defensive equal-after-normalization sentinel
  (`differs:false`, no offsets/text) for any future skew.

## Retire parity

`retire_draft_reconcile_plan` shares the same content-identity primitive. The inner remint plan's
already-computed diagnostic is threaded up through `_retire_reconcile_content_anchor_class` (now
returning `{drift_class, body_diff_diagnostic}`) and merged into the retire plan the same way, so
the retire plan carries the **identical** `body_diff_diagnostic` dict the remint plan produced for
the same zettel — no body symbols are introduced into retire scope, and the retire apply path is
unaffected (it uses only the `drift_class` string).

## CLI

Both text printers (`command_remint_reconcile`, `command_retire_draft_reconcile`) gain ONE
content-free summary line:

```
- body diff: category=<label> offset=<int> length_delta=<int> [forms=<form>/<form>]
```

(the `forms=` suffix appears only for `unicode_normalization_only`). It prints only when the key
is present and `differs` is true, and echoes only numbers and fixed labels — never body text.
JSON consumers see the new key only when it is present.

## Tests

Twenty-three tests (T1–T23) via the injected fixture and direct helper unit tests: category
correctness for `final_newline_only`, `trailing_whitespace_only`, `unicode_normalization_only`
(NFC vs NFD), and `content_difference`; the anti-laundering attacks (mixed trailing-ws + content,
inner content + trailing-ws, internal-whitespace edit, leading-indentation edit, added blank line,
NFD + trailing-ws, NFD + content edit — all `content_difference`); classification invariance
(drift_class / basis / ack byte-identical to a v0.3.175 baseline); the guards (Tier C / no-snapshot,
`format_drift`, unreadable snapshot, defensive sentinel — all omit or no-op without crashing); the
content-free guarantees (no body substring in the serialized diagnostic or the CLI printer line,
exact locked key set); the primitive byte-for-byte pin; retire parity (identical dict; omitted on
no body change); and offset semantics (prefix / min-length offset, integer type, signed delta).

## Compatibility

Additive; **no migration required**. No classification or behavior change. Existing receipts and
manifests are unaffected. The full test suite, `check_public_privacy`, and
`check_release_readiness` stay green.

## Upgrade

See `UPGRADE.md`. All changes are additive; no migration is required. The only new surface is the
content-free `body_diff_diagnostic` decoration on the reconcile plan output and the matching
one-line CLI summary; it never alters classification or the content-change ack gate.
