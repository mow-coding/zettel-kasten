# v0.3.8 - Human-Guided Intake Bridges And ZET Surface Prototypes

Release date: 2026-06-14

v0.3.8 is a compatible fast-follow to v0.3.7. It tightens the human-guided
project intake loop, adds safe bridges from reviewed source-intake plans toward
local capture selection, supports explicit external prehashed objet ledger
registration, and records the first read-only ZET surface prototypes for
WordPress, Joplin, Notion, and Obsidian.

## Added

- CLI `archive project-intake-staging-guide --dry-run` and MCP
  `project_intake_staging_guide` for local objet-store staging path guidance.
- CLI `archive project-intake-session-guide --dry-run` and MCP
  `project_intake_session_guide` for a one-screen next-step guide in the
  human-guided intake loop.
- CLI `archive project-intake-next-question --dry-run` and MCP
  `project_intake_next_question` so an AI runtime can ask one human-review
  question at a time.
- CLI `archive project-intake-decision-template --dry-run` and MCP
  `project_intake_decision_template` for the next answer's JSON shape without
  filling answer values.
- CLI `archive project-intake-item-plan --dry-run` and MCP
  `project_intake_item_plan` for one human-selected file's next source-intake
  route.
- CLI `archive source-intake-record --dry-run|--approve` for recording a
  reviewed, redacted source-intake dry-run plan under `receipts/sources/`.
- CLI `archive objet-capture-selection --dry-run|--approve` for building a
  reviewed `objet-capture --selection` manifest from one staged file and one
  recorded source-intake plan.
- CLI `archive prehashed-objet-ledger --dry-run|--approve` for explicit
  manifest registration of already-hashed external content-addressed ledgers.
  MCP remains read-only through `prehashed_objet_ledger_preview`.
- CLI `archive zet-surface-prototype --dry-run` and MCP
  `zet_surface_prototype_plan` for WordPress, Joplin, Notion, and Obsidian
  surface prototype previews.

## Changed

- The project-intake documentation now shows the reviewed path through
  `source-intake-record`, `objet-capture-selection`, and then explicit
  `objet-capture`.
- The capability matrix and README now distinguish human artifact planning from
  ZET surface prototype planning.
- Prehashed external ledger registration is documented as a separate metadata
  registration path, not as a shortcut for byte-verifying `objet-capture`.

## Boundaries

- Project intake remains human-guided. The new guide and question/template
  surfaces do not classify whole folders automatically.
- The project-intake session guide writes nothing, echoes no decision values,
  reads no file bodies, captures nothing, drafts nothing, mints nothing, uploads
  nothing, cleans nothing, and authorizes no automatic execution.
- Source-intake recording stores only a reviewed metadata plan; it does not read
  source file bodies, calculate source content hashes, capture objets, draft,
  mint, upload, or clean.
- Objet capture selection writes only a reviewed selection manifest. It does
  not run capture, copy object bytes, append object manifest records, draft,
  mint, upload, or clean staged originals.
- Prehashed objet ledger approval appends external manifest records and writes a
  receipt without reading blob bytes, copying objects, uploading, drafting,
  minting, or claiming that `objet-capture` can skip byte verification.
- ZET surface prototypes are read-only. They do not call providers, request
  tokens, write notes, write vault files, publish posts, create projection
  receipts, mint zets, sync, clean, or run ZET transport.

## Upgrade

No archive migration is required for v0.3.7 users.

To start a human-guided intake session:

```text
archive project-intake-session-guide <archive-root> --project-slug <project-slug> --dry-run
```

To continue from a reviewed project-intake receipt:

```text
archive project-intake-session-guide <archive-root> --receipt <receipt> --dry-run
```

To preview a ZET surface prototype:

```text
archive zet-surface-prototype <archive-root> --surface-kind obsidian --surface-ref obsidian:vault:review --dry-run --format json
```
