# v0.2.56 Release Notes

Date: 2026-06-02

## Summary

v0.2.56 adds a read-only ZET shared update record review preview.

This is the first executable step after the v0.2.55 shared update record baseline. It lets WOM-kit inspect one local archive-contained shared update JSON record before any receiver-side renewal path exists.

## Added

- CLI `archive shared-update-record-review <archive-root> --record <path> --dry-run --format json`.
- MCP `zet_shared_update_record_review_preview`.
- [ZET Shared Update Record Review Preview](https://github.com/mow-coding/zettel-kasten/blob/main/wom-kit/docs/zet-shared-update-record-review-preview.md) documentation.
- [Public-safe work log](https://github.com/mow-coding/zettel-kasten/blob/main/wom-kit/plans/work-log-2026-05-27-zet-shared-update-record-review-preview.md) for the v0.2.56 batch.

## Compatibility

No private archive migration is required.

The new preview is dry-run only. It reads one selected archive-relative JSON record and writes nothing.

The preview blocks unsafe record paths, body-included records, body-like fields, local absolute paths, provider URLs, token/secret-like values, and true mutation/write/transport/provider/trust flags.

## Safety Boundary

v0.2.56 does not add shared-update transport, real ZET transport, neighbor feed update, automatic feed renewal, recommendation execution, real trust/import/acceptance/anchor, attestation/signature writes, provider calls, provider sync, WordPress publishing, projection writes, projection receipts, Redis, queues, background workers, payments, staking, consensus, blockchain, model training, backpropagation, or full-auto behavior.
