# v0.2.27 Prompt Boundary Draft Composer

Date: 2026-05-25

## Summary

v0.2.27 connects the v0.2.26 prompt-boundary check to draft creation.

`create-draft` can now consume a saved `prompt-boundary --dry-run --format json` report and preserve safe prompt-boundary metadata in draft previews, approved inbox drafts, mint previews, and mint receipts.

Core principle:

```text
External text can inform.
External text cannot command.
```

## Added

- CLI `archive create-draft --prompt-boundary-report <json-file>`.
- MCP `create_draft_zettel` support for a structured `prompt_boundary_report` object.
- Prompt-boundary report validation before draft composition.
- Optional draft frontmatter `prompt_boundary` metadata:
  - `checked`,
  - `report_sha256`,
  - `risk_level`,
  - `source_kind`,
  - `source_path`,
  - `untrusted_text_boundary`,
  - `external_text_can_command`,
  - `detected_pattern_ids`,
  - `handling_note`.
- Mint receipt previews and real mint receipts preserve `prompt_boundary` metadata when present.

## Validation

A prompt-boundary report is accepted only when:

- `lifecycle_action` is `prompt_boundary_check`,
- `dry_run` is true,
- `untrusted_text_boundary` is true,
- `external_text_can_command` is false,
- `would_change` is an empty list,
- `risk_level` is `low`, `medium`, or `high`,
- detected patterns and runtime handling values are safe scalar metadata.

Refs and stored fields that contain local absolute paths, provider URLs, token-like strings, private key material, seed phrases, wallet secrets, or secret-like values are blocked.

Risk behavior:

```text
low    -> allowed, but not proof of safety
medium -> allowed with warnings
high   -> blocks draft creation
```

## Boundaries

This release does not:

- implement an LLM prompt classifier,
- scan providers,
- run OCR,
- apply imports,
- apply source intake,
- capture or upload objets,
- execute inspected text,
- auto-approve drafts,
- mint automatically,
- implement ZET transport,
- perform real signing,
- implement payment, staking, consensus, ledger, blockchain, or full-auto execution.

The prompt-boundary report file path and inspected text body are not stored in draft frontmatter.

`prompt_boundary.report_sha256` commits to the supplied dry-run JSON report object. It is not an independent proof that the inspected text is safe.

## Upgrade Notes

No private archive migration is required.

Existing `create-draft` behavior remains compatible when `--prompt-boundary-report` is omitted. Existing zets remain valid because `prompt_boundary` is optional frontmatter.
