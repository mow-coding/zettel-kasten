# WOM-kit v0.3.23

Date: 2026-06-15

v0.3.23 adds a read-only credential access broker planner.

## Added

- CLI `archive credential-access-broker-plan --dry-run`.
- CLI aliases `archive credential-broker-plan --dry-run` and
  `archive secret-access-broker-plan --dry-run`.
- MCP `credential_access_broker_plan`.
- `wom-kit/docs/credential-access-broker-plan.md`.

## Behavior

The planner defines a future local broker request shape for:

- mail source reads,
- model API calls,
- OCR API calls,
- object-storage requests,
- CLI token auth,
- browser login fill,
- plaintext secret migration planning.

It can accept a safe `credential_ref`, but it does not echo the exact ref value
back. It reports only the ref store and prefix, such as `secret:` or
`keyring:`.

## Broker Boundary

The intended future flow is:

```text
AI requests a credential capability by credential id, purpose, provider, and action
-> WOM checks policy and human approval
-> a local adapter talks to the real vault/keyring/platform surface
-> the adapter uses the secret for the approved action
-> the secret value is not returned to chat
```

v0.3.23 does not implement that adapter. It only defines the planning contract.

## Boundaries

This release does not open password managers, open browser password stores,
open OS keyrings, read environment variables, read plaintext secret notes,
write vault entries, call providers, draft zets, or mint zets.
