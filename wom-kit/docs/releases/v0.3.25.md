# WOM-kit v0.3.25

Date: 2026-06-15

v0.3.25 adds a read-only credential adapter readiness planner.

## Added

- CLI `archive credential-adapter-readiness-plan --dry-run`.
- CLI aliases `archive credential-adapter-plan --dry-run` and
  `archive secret-adapter-readiness --dry-run`.
- MCP `credential_adapter_readiness_plan`.
- `wom-kit/docs/credential-adapter-readiness-plan.md`.

## Behavior

The planner previews the contract a future local credential adapter must
satisfy before using a real password manager, OS keyring, browser/platform
password manager, developer secret manager, environment injection surface, or
future wallet.

It can plan adapter kinds such as:

- KeePassXC-style local CLI adapters,
- Bitwarden/1Password-style CLI adapters,
- Windows Credential Manager,
- macOS Keychain,
- Linux Secret Service,
- browser/platform password managers,
- developer secret managers,
- environment injection,
- future wallet/local custody.

It can plan future operations such as:

- resolving a credential for one approved action,
- writing a new secret,
- rotating a secret,
- plaintext secret migration,
- browser login fill,
- metadata-only listing.

## Boundaries

This release does not implement live adapter execution.

It does not open password managers, browser password stores, OS keyrings,
environment variables, plaintext secret files, provider APIs, approval receipt
writers, or audit receipt writers.

The exact credential ref value and the secret value are never returned.

## Relationship To Earlier Releases

v0.3.25 builds on:

- v0.3.21 credential ref inventory,
- v0.3.22 credential store recommendations,
- v0.3.23 credential access broker planning,
- v0.3.24 credential access approval receipt preview.

The new planner does not bypass any of those layers. It requires the same
policy-gated, human-approved, non-echoing boundary before any future adapter can
touch a real credential.
