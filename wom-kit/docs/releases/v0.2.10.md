# v0.2.10 ZET Sharing Dry-Run Lifecycle

`v0.2.10` turns the `mint -> delegate -> attest -> anchor` philosophy into the first dry-run CLI and MCP contract.

This release does not implement real sharing, P2P transport, feeds, external sending, or foreign zet import.

## Added

- `archive delegate-zet --dry-run`
- `archive attest-zet --dry-run`
- `archive anchor-zet --dry-run`
- MCP `delegate_zet_check`
- MCP `attest_zet_check`
- MCP `anchor_zet_check`
- `schemas/delegate-receipt.schema.json`
- `schemas/attestation-receipt.schema.json`
- `schemas/anchor-metadata.schema.json`

## Lifecycle Meaning

```text
mint
  issue a zet into private archive memory

delegate
  give a scoped capability around a minted zet

attest
  verify a delegated foreign zet and record distributed witness evidence

anchor
  preview placing an attested foreign zet into local meaning
```

## Compatibility

Existing compatibility surfaces remain:

- `archive share --dry-run`
- MCP `share_check`

`share --dry-run` remains the older scope/trust gate surface. `delegate-zet --dry-run` is the preferred product-facing surface for the new lifecycle language.

## Migration

No private archive migration is required.

New commands are dry-run only and do not write receipts, metadata, workpacks, zettels, or source files.
