# v0.3.223 - Full-Doctor Receipt Phase And Callback Coalescing

v0.3.223 improves the optional full-Doctor progress path after the fast/default
start-here split. It is additive and requires no archive migration.

## What Changed

- Added a fixed content-free mint-receipt phase to count-bearing heartbeat.
- Covers receipt checks, source/target/snapshot refs, file hash, target
  frontmatter, mint link, target edge evolution, and edge-receipt index work.
- Unknown detail produces no phase; path-bearing source text is never copied.
- Coalesces generic same-count callbacks before shared formatter/lock work while
  still updating phase state for heartbeat.
- Leaves direct Doctor verbose output and private JSONL progress logs unchanged.

## Upgrade Guidance

Quick/default `ai-start-here` remains unchanged. When a complete archive health
check is needed:

```text
archive ai-start-here <archive-root> --dry-run --full-doctor --progress --format json
```

If one mint receipt holds the count, heartbeat can now show a phase such as
`file_hash` or `edge_receipt_index`. The phase is liveness evidence only. Full
Doctor can still take minutes and the phase does not prove eventual completion.

## Privacy And Work Boundary

The phase vocabulary is fixed in code. Raw receipt paths, zet paths/ids, object
refs, edge values, titles, bodies, abstracts, provider values, and secrets never
enter shared reporter output. No validation, hash, or file is skipped.
