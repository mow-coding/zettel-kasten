# v0.2.21 Object Storage / Objet Setup Planner

Status: public pre-release patch

Date: 2026-05-25

## Summary

`v0.2.21` adds a dry-run-first object storage / objet setup planner for WOM profiles and archives.

The planner prepares safe local metadata for where source/original objets will live. It helps an AI runtime propose bucket/container names, prefixes, provider binding metadata, local profile hints, setup receipt previews, and manual steps without touching any provider.

## Added

- `archive object-storage <archive-root> --dry-run --format json`
- Default bucket/container naming:

```text
zettel-kasten-<normalized-profile-slug>-objets
```

- Default objet prefix:

```text
archives/<archive_id>/objets/
```

- Safety gates for:
  - provider kind,
  - ASCII profile slugs,
  - conservative bucket/container names,
  - storage account references,
  - region labels,
  - endpoint references,
  - path-like, URL-like, email-like, and secret-like values.
- `--approve --reviewed-by` local-only approval for:
  - `provider-bindings.yml`,
  - provider setup receipts under `receipts/providers/`.
- Optional ignored local object storage account hints through `--write-local-profile`.
- Read-only MCP `object_storage_setup_plan`.
- Rollback protection for approved local metadata writes, so `provider-bindings.yml` is not left modified if the matching setup receipt or optional ignored local profile cannot be written.
- Local object-storage profile hints preserve multiple buckets for the same WOM profile by matching existing hints on provider kind plus bucket name.

## Compatibility

No private archive migration is required.

Existing CLI and MCP commands remain compatible.

Approved CLI mode writes only local archive metadata. It does not create buckets/containers or configure a provider.

Approved metadata writes are guarded so a failed receipt/local-profile write does not leave an object storage provider binding without its receipt.

## Non-Goals

This release does not:

- create buckets or containers,
- run OAuth,
- call provider APIs,
- upload files,
- sync files,
- copy source files,
- calculate object hashes,
- import source content,
- add MCP object storage apply/create/connect/upload/sync tools,
- rename WOM-kit,
- change WOM/zet/ZET philosophy.
