# WOM-kit v0.3.26

Date: 2026-06-15

v0.3.26 adds a read-only credential adapter manifest preview and schema.

## Added

- CLI `archive credential-adapter-manifest-plan --dry-run`.
- CLI aliases `archive credential-adapter-manifest --dry-run` and
  `archive secret-adapter-manifest-plan --dry-run`.
- MCP `credential_adapter_manifest_plan`.
- JSON schema `wom-kit/schemas/credential-adapter-manifest.schema.json`.
- `wom-kit/docs/credential-adapter-manifest-plan.md`.

## Behavior

The planner emits a non-secret `manifest_preview` for a future local credential
adapter. The preview includes:

- adapter id,
- adapter kind,
- adapter family,
- store kind,
- platform,
- consumer label,
- expected ref prefixes,
- supported operations,
- required policy, approval, local UI, non-echo, and audit boundaries,
- privacy contract,
- closed actions.

The preview validates against `credential-adapter-manifest.schema.json`.

## Boundaries

This release does not write adapter manifests, open password managers, open
browser password stores, open OS keyrings, read environment variables, read
plaintext secret files, write secrets, migrate secrets, call providers, write
approval receipts, or write audit receipts.

It does not include secret values, exact credential refs, local absolute paths,
provider account values, provider URLs, passwords, tokens, or API keys.

## Relationship To v0.3.25

v0.3.25 previews whether a future adapter operation is safe to shape.

v0.3.26 previews the non-secret manifest that a future adapter could later
declare before any approval-gated writer or live adapter exists.
