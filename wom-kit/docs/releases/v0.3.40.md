# v0.3.40 - Presigned URL Planning

Date: 2026-06-15

v0.3.40 adds a read-only presigned URL planning surface.

## Added

- CLI `archive presigned-url-plan <archive-root> --dry-run`.
- Aliases:
  - `archive object-presigned-url-plan`
  - `archive objet-presigned-url-plan`
- MCP tool:
  - `presigned_url_plan`
- New document:
  - `wom-kit/docs/presigned-url-plan.md`

## Behavior

The planner validates the safe shape of a future provider presigned URL request:

- `object_id`
- external `store_ref` label,
- operation,
- TTL,
- manifest resolution summary,
- local object-storage binding presence,
- future adapter requirements.

It reuses `resolve-objet-ref` so it reads local manifest metadata instead of
object bytes.

## Safety

The command and MCP tool are read-only. They:

- create no presigned URLs,
- call no provider APIs,
- retrieve no credential values,
- open no password manager, keyring, browser password store, or secret manager,
- read no object file bytes,
- hash no object file bytes,
- upload nothing,
- download nothing,
- prove no remote availability,
- echo no provider URLs,
- echo no local absolute paths,
- echo no exact credential refs,
- write no files or receipts.

This release still has no live object-storage provider adapter.
