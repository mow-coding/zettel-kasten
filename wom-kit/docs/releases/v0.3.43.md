# v0.3.43 - Manifest-Aware Object Storage Recommendations

Date: 2026-06-15

v0.3.43 makes object-storage recommendations aware of archive manifest
composition.

## Added

- New scenario:

```text
auto_from_manifest
```

- `archive object-storage-recommendation --scenario auto_from_manifest
  --dry-run --format json` now reads aggregate metadata from
  `objects/manifests/files.jsonl`.
- Output now includes:
  - `manifest_analysis`
  - `scenario_source`
  - `rough_cost_estimates`

## Behavior

The manifest analysis reports total size, sized record count, dominant content
class, content-class percentages, inferred scenario, and inference confidence.

Example: a media-heavy archive can be classified as `personal_low_ops` with a
signal that egress policy matters, while still asking the human to confirm
whether the archive is backup-only, frequently read, shared, or compliance-bound.

## Estimate Boundary

Rough cost fields are comparison hints only. They use a static 2026-06-15
public-pricing snapshot and do not call live pricing APIs. Humans must still
review official provider calculators/docs before signup or upload.

## Safety

The command remains read-only. It:

- writes no files,
- reads no object bytes,
- calls no providers,
- calls no pricing APIs,
- checks no bucket availability,
- creates no buckets,
- uploads nothing,
- downloads nothing,
- creates no presigned URLs,
- echoes no object filenames,
- echoes no local paths,
- echoes no provider account URLs,
- echoes no tokens or secret values.

No archive migration is required.
