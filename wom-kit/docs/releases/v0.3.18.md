# WOM-kit v0.3.18

Date: 2026-06-14

v0.3.18 adds a read-only zettel-level objet link preview for the reading-side
UX.

## Added

- CLI `archive zettel-objet-links --path <zet.md>|--zettel-id <id> --dry-run`.
- MCP `zettel_objet_links`.
- `wom-kit/docs/zettel-objet-links.md`.
- Tests for CLI output, MCP structured output, dry-run-only behavior, and
  privacy boundaries.

## Behavior

The preview scans one non-redacted zettel for:

- `sha256:<64 hex characters>`,
- `objet:sha256:<64 hex characters>`.

For each distinct objet ref, it reuses the v0.3.17 resolver and returns:

- occurrence position hints,
- resolver state,
- safe local archive-relative candidates,
- safe external store labels,
- command hints for `resolve-objet-ref`.

## Boundaries

This release does not open files, echo zettel body text, echo frontmatter
values, echo absolute local paths, echo provider URLs, read object bytes, hash
object bytes during link preview, call providers, create presigned URLs,
download objects, upload objects, prove remote availability, decide deletion
safety, draft, mint, clean, or run ZET transport.

Redacted zettels are blocked and do not expose link previews.
