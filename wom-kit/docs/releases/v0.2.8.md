# v0.2.8 Minting Lifecycle

`v0.2.8` turns the product-facing minting lifecycle into real CLI, schema, doctor, and MCP behavior.

The old `promote` command remains for compatibility, but the preferred public language is now `mint-zettel`.

## Added

- `archive mint-zettel --dry-run`
- `archive mint-zettel --approve --reviewed-by <id>`
- `schemas/mint-receipt.schema.json`
- `receipts/mint/<zettel_id>.mint.json`
- `receipts/mint/drafts/<zettel_id>.draft.md`
- canonical zettel `mint` frontmatter metadata
- read-only MCP tool `mint_zettel_check`

## Minting Flow

```text
draft zet
-> dry-run checklist and duplicate gate
-> human approval
-> canonical private zettel
-> mint receipt
-> mint-time draft snapshot
```

Minting is private archive issuance. It is not public posting and it does not share data with another archive.

## Safety Defaults

- The original `inbox/` draft stays in place.
- The exact draft text at mint time is copied to `receipts/mint/drafts/`.
- Real minting requires `--approve`.
- Real minting requires `--reviewed-by`.
- The first authority mode is `basic`.
- MCP exposes only `mint_zettel_check`; it cannot perform real minting.

## Doctor Validation

`archive doctor` now validates:

- mint receipt schema shape,
- receipt action and `authority_mode`,
- source, target, and snapshot paths,
- SHA-256 hashes for source, target, and snapshot,
- canonical zettel `mint.receipt_path` linkage.

Canonical zettels that only have legacy `promotion` metadata remain valid for the v0.2 line.

## Migration

No private archive migration is required.

Newly minted zettels should keep all three generated files:

```text
zettels/<draft-name>.md
receipts/mint/<zettel-id>.mint.json
receipts/mint/drafts/<zettel-id>.draft.md
```
