# Release Notes

This folder contains human-readable release notes for public compatibility checkpoints.

Each release note should answer:

- What changed?
- Who should upgrade?
- Is archive migration required?
- Are schemas or zettel rules changed?
- How can users verify the upgrade?
- What should remain private?

Release notes are intentionally more beginner-friendly than raw Git commit history.

For the intended meaning of the pre-1.0 version lines, see the
[WOM Product Roadmap](https://github.com/mow-coding/zettel-kasten/blob/main/wom-kit/docs/product-roadmap.md).
