# v0.2.12 Real Delegate Receipt Write

`v0.2.12` turns `delegate-zet` from preview-only into the first real receipt write in the zet sharing lifecycle.

The important change is narrow on purpose: a source archive can now write a local delegate receipt after the same dry-run gates pass. This does not send the zet anywhere and does not implement claim or spent registries.

## Added

- `archive delegate-zet --approve --reviewed-by <actor>`
- `receipts/delegate/*.delegate.json` writes with `dry_run: false`
- applied delegate receipt validation in `archive doctor`
- real capability nonce issuance for delegate receipts
- receipt-local claim/spent state that explicitly says registries are not implemented

## Compatibility

Dry-run remains the first step:

```bash
archive delegate-zet ./archive \
  --view view.example \
  --target-archive archive:team:example \
  --counterparty-id archive:team:example \
  --counterparty-fingerprint SHA256:example \
  --dry-run
```

Approved receipt write uses the same gates:

```bash
archive delegate-zet ./archive \
  --view view.example \
  --target-archive archive:team:example \
  --counterparty-id archive:team:example \
  --counterparty-fingerprint SHA256:example \
  --approve \
  --reviewed-by person:me
```

`claimable_once` delegation can also write a receipt without choosing the recipient archive yet:

```bash
archive delegate-zet ./archive \
  --view view.example \
  --target-policy claimable_once \
  --approve \
  --reviewed-by person:me
```

## Not Implemented

This release does not implement:

- real claim registry,
- spent registry,
- revocation registry,
- real attestation writes,
- real anchor writes,
- P2P transport,
- relays,
- blockchain settlement,
- payment integration,
- UI.

## Upgrade

No private archive migration is required.

```bash
git fetch --tags
git checkout v0.2.12
```
