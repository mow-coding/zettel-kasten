# v0.2.17 WOM Profile Registry Dry-Run

Release date: 2026-05-24

Status: public pre-release

## Summary

`v0.2.17` adds a read-only WOM profile registry layer before runtime context and draft work.

The purpose is to prevent an AI runtime from assuming the current/default archive is the target when the user names another profile, team, project, or archive.

## What Changed

- added `archive profile-list --registry <path> --format json`,
- added `archive profile-resolve --registry <path> --target <query> --format json`,
- added read-only MCP tools `wom_profile_list` and `wom_profile_resolve`,
- added local path redaction by default,
- added an example registry at `ai-archive-kit/templates/profiles/wom-profiles.example.yml`,
- added profile resolution states: `resolved`, `ambiguous`, `not_found`, and `token_missing`,
- added Unicode-normalized profile matching for labels and aliases,
- added blocker checks for registry version drift, duplicate profile ids, and raw token-like fields.

## Correct AI Runtime Flow

```text
1. Resolve requested WOM profile.
2. Confirm or switch target archive context.
3. Run runtime-context with expected archive id/type.
4. Run create-draft dry-run in a later release.
5. Ask human approval before writing.
6. Keep minting as a separate approval step.
```

## Security Boundary

The profile registry may store:

- profile ids,
- labels and aliases,
- node/archive ids,
- archive type,
- local archive root references,
- operator id,
- authority mode,
- token state,
- token references.

It must not store raw token values.

Registries with unsupported token fields or an unsupported registry version are blocked by the v0.2.17 reader.

The CLI and MCP outputs redact local paths by default. MCP ignores local path disclosure requests unless `AI_ARCHIVE_MCP_ALLOW_LOCAL_PATHS=1` is set on the MCP server.

## Non-Goals

This release does not implement:

- profile registration,
- token storage,
- create-draft dry-run,
- provider API sync,
- UI,
- real minting through MCP,
- MCP write/register/apply tools.

## Privacy

The example registry uses placeholder paths and fake `token_ref` values only.

Do not publish real local archive paths, raw tokens, private filenames, real source maps, private zets, receipts, or private AI conversations in this public repository.
