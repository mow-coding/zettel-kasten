# v0.2.15 WOM Safe HTML Profile Validator Dry-Run

Release date: 2026-05-23

Status: public pre-release

## Summary

`v0.2.15` adds the first concrete tool for the WOM Safe HTML Profile rollout that `v0.2.14` documented: a read-only CLI dry-run validator that previews whether a v0.2 Markdown-compatible zet is compatible with a future WOM Safe HTML Profile migration.

It does not change CLI mint output, archive writes, schemas, MCP tools, or the private archive layout.

## What Changed

- added `archive check-safe-html --path <zet> --dry-run` to `ai-archive-kit/cli/archive.py`,
- added the underlying read-only `check_safe_html_dry_run` service in `ai_archive_kit.archive_services`,
- added focused tests for safe Markdown, unsafe `<script>`, unsafe `javascript:` link, unsafe `<iframe>`/`<object>`/`<embed>` elements, inline event handler attributes, and no-write dry-run behavior,
- documented the validator inside `ai-archive-kit/docs/concepts/wom-safe-html-profile.md` and its Korean pair.

## What The Validator Does

It reads a draft or canonical zet and returns a structured JSON report with:

- `ok`,
- `lifecycle_action`: `check_safe_html`,
- `source_path`,
- `detected_format`: `markdown_compatible`,
- `proposed_profile`: `wom-safe-html/v0.1-draft`,
- `blockers`,
- `warnings`,
- `html_profile_preview`,
- `text_extraction_preview`,
- `source_reference_preview`.

It blocks zet bodies that contain:

- `<script>` elements,
- `<iframe>` elements,
- `<object>` elements,
- `<embed>` elements,
- `javascript:` URLs,
- inline event handler attributes such as `onclick=`, `onload=`, and other `on*=` attributes.

It allows normal Markdown-compatible text.

## What The Validator Does Not Do

- It does not write files.
- It does not convert Markdown to HTML.
- It does not change mint output.
- It does not migrate existing zets.
- It does not publish the full WOM Safe HTML Profile element/attribute allowlist; the profile remains a `wom-safe-html/v0.1-draft` baseline at this point.

## Compatibility

No private archive migration is required.

Existing Markdown-compatible zets remain valid in the v0.2 compatibility line.

The following are not implemented in this release:

- Markdown-to-WOM-Safe-HTML conversion,
- a finalized element/attribute allowlist,
- HTML migration,
- UI,
- live sharing,
- P2P transport,
- external provider sync.

## Example

```bash
python ai-archive-kit/cli/archive.py check-safe-html ai-archive-kit/examples/fake-life-archive \
  --path inbox/zet_20260519_draft_ai_lunch_note.md \
  --dry-run \
  --format json
```
