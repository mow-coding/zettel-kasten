# v0.2.16 WOM AI Runtime Context Layer

Release date: 2026-05-24

Status: public pre-release

## Summary

`v0.2.16` adds a read-only runtime context layer for terminal-capable AI runtimes such as Codex and Claude Code.

The goal is simple: before an AI creates a draft, runs a dry-run, or asks for mint approval, it can confirm which archive it is operating on and which safe actions are available.

## What Changed

- added `archive runtime-context <archive-root> --format json`,
- added `--expected-archive-id`, `--expected-type`, and `--strict`,
- added default local path redaction with `--redact-local-paths` and `--no-redact-local-paths`,
- added read-only MCP tool `archive_runtime_context`,
- added tests for deterministic redacted JSON, id/type mismatch behavior, no-write behavior, MCP allowed roots, and no MCP mint/apply exposure.

## JSON Output

The CLI returns:

- `ok`,
- `lifecycle_action: runtime_context`,
- `archive_id`,
- `archive_type`,
- `scope`,
- `principal`,
- `owner`,
- `ai_write_policy`,
- `paths`,
- `available_safe_actions`,
- `doctor_summary`,
- `blockers`,
- `warnings`,
- `redaction.local_paths_redacted`.

By default, `paths` are archive-relative:

```json
{
  "inbox": "inbox/",
  "zettels": "zettels/",
  "receipts": "receipts/",
  "object_manifest": "objects/manifests/files.jsonl"
}
```

## Behavior

- Archive id mismatch blocks.
- Archive type mismatch warns by default.
- Archive type mismatch blocks with `--strict`.
- Local absolute paths are redacted by default.
- MCP local path disclosure requires `AI_ARCHIVE_MCP_ALLOW_LOCAL_PATHS=1` on the MCP server and an explicit caller request.
- The command writes no files.
- The MCP tool follows existing `AI_ARCHIVE_MCP_ALLOWED_ROOTS` enforcement.

## Non-Goals

This release does not implement:

- create-draft dry-run,
- provider API sync,
- UI,
- real minting through MCP,
- any MCP apply tool,
- product philosophy or naming rule changes.

## Example

```bash
python ai-archive-kit/cli/archive.py runtime-context ai-archive-kit/examples/fake-life-archive \
  --expected-archive-id archive:personal:fake-life \
  --expected-type personal \
  --format json
```

## Verification

Recommended checks:

```bash
python -m unittest discover -s ai-archive-kit/tests
python ai-archive-kit/cli/archive.py doctor ai-archive-kit/examples/fake-life-archive --strict
git diff --check
```

## Privacy

The runtime context output redacts local absolute paths by default.

Do not publish real private zets, source maps, receipts, provider tokens, credentials, private filenames, local absolute paths, or private AI conversations in this public repository.
