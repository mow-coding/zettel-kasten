# v0.2.18 Profile-Aware Draft zet Creation Dry-Run

Status: public pre-release patch

Date: 2026-05-24

## Summary

`v0.2.18` adds the safe draft creation step that comes after profile resolution and runtime context.

The AI workflow is now:

```text
profile-resolve
-> runtime-context
-> create-draft dry-run
-> human draft approval
-> create inbox draft
-> mint only through separate approval
```

Natural-language requests such as "올려줘" should mean preview or create an `inbox/` draft zet. They must not automatically mint canonical archive memory.

## Added

- `archive create-draft --dry-run --format json`
- expected archive id/type checks for draft creation
- replay-safe `draft_id`, `created_at`, and `expected_body_sha256`
- `draft-approved-by` for inbox-draft-only approval
- optional provenance fields:
  - `creation_mode`
  - `assisted_by`
  - `supervised_by`
  - `derived_from`
  - `source_refs`
  - `local_ai_sessions`
  - `draft_creation`
- MCP `create_draft_zettel` dry-run support
- body hash mismatch blocking
- line-ending-normalized body hashes for cross-OS replay stability
- body sanity checks for empty or whitespace-only draft content
- deterministic timestamp validation for replay fields
- AI-assisted and AI-generated draft checks that require an identified assisting AI runtime
- unsafe local path/provider locator blocking for body and provenance refs
- mint receipt preservation for draft source refs, derived refs, and local AI sessions

## Compatibility

Existing `create-draft` commands keep working when the new flags are omitted.

Existing drafts remain valid. The new frontmatter fields are optional.

## Safety

- Dry-run writes nothing.
- Normal mode writes only to `inbox/`.
- Profile-bound AI writes require human draft approval and expected body hash replay.
- Body hashes normalize LF/CRLF line endings before comparison.
- AI-assisted and AI-generated drafts must identify the assisting AI runtime in provenance.
- Draft approval is not mint approval.
- MCP still exposes no real mint tool.
- Raw provider URLs, local absolute paths, and secret-like values are blocked from draft body/provenance refs.

## Non-Goals

This release does not add:

- profile registration,
- token storage,
- provider API sync,
- UI,
- real minting through MCP,
- real source import/copy/upload.
