# Contributing

Thank you for looking at `zettel-kasten` and `zet`.

This project is early and concept-heavy. Useful contributions include:

- pointing out broken assumptions,
- improving schemas,
- improving setup docs,
- adding tests,
- strengthening privacy and security boundaries,
- comparing the design with existing local-first, P2P, archive, or AI-agent systems.

## Ground Rules

- Do not submit real private archives, real credentials, real provider exports, or private AI conversations.
- Use fake examples when demonstrating behavior.
- Keep `zet` as a text-first object.
- Treat minting as private archive incorporation unless a sharing/export layer is explicitly involved.
- Keep provider integrations dry-run and approval-first.
